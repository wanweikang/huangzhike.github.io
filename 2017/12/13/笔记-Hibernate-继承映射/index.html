[笔记]-Hibernate-继承映射

2017-12-13

12-28重新整理更新Hibernate笔记

假设有接口Payment和实现类：CreditCardPayment，CashPayment。

interface Payment {
    Integer id;
    String amount;
    // ...
}
public class CreditCardPayment implements Payment {
    String creditCardType;
    // ...
}
public class CashPayment implements Article {
    String cashPaymentType;
    // ...
}

三种继承映射：

  • 每个类分层结构一张表
  • 每个子类一张表
  • 每个具体类一张表

每个类分层结构一张表

  • 只需要一张表。
  • 限制：由子类定义的字段，如CCTYPE，不能有非空NOT NULL约束。
create table xxx (
    PAYMENT_ID int not null auto_increment primary key,
    PAYMENT_TYPE varchar(10),
    -- 略
);
<class name="Payment" table="PAYMENT">
    <id name="id" type="long" column="PAYMENT_ID">
        <generator class="native"/>
    </id>
    <!-- 通过单表映射整个层次，在表中创建一个额外的discriminator列来标识具体是那个类 -->
    <!-- 用于辨别到底是哪一个类，CreditCardPayment或CashPayment -->
    <!-- Discriminator映射就是把一个表映射成不同的类，有不同的属性 -->
    <discriminator column="PAYMENT_TYPE" type="string"/>

    <property name="amount" column="AMOUNT"/>
    <!-- 子类 -->
    <!-- discriminator-value：鉴别是哪个类的一个值，表示这个值就是这个类，不写默认为类的全限定名 -->
    <!-- 做数据库修改时就不用关心PAYMENT_TYPE字段的值了，会自动的加CREDIT或CASH -->
    <subclass name="CreditCardPayment" discriminator-value="CREDIT">
        <property name="creditCardType" column="CCTYPE"/>
    </subclass>

    <subclass name="CashPayment" discriminator-value="CASH">
        <!-- ... -->
    </subclass>
</class>

如果用注解，大概是这样的：

@Entity
@Table(name = "xxx")
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
@DiscriminatorColumn(name = "PAYMENT_TYPE", discriminatorType = DiscriminatorType.STRING)
public class Payment {}
@Entity
@DiscriminatorValue(value = "CREDIT")
public class CreditCardPayment extends Payment {}
@Entity
@DiscriminatorValue(value = "CASH")
public class CashPayment extends Payment {}

每个子类一张表

  • 需要三张表。
  • 子类表通过主键关联到超类表(一对一关联)。
  • 表是根据类创建的，但是由外键相关，所以没有重复的列。
<class name="Payment" table="PAYMENT">
    <id name="id" type="long" column="PAYMENT_ID">
        <generator class="native"/>
    </id>
    <property name="amount" column="AMOUNT"/>

    <joined-subclass name="CreditCardPayment" table="CREDIT_PAYMENT">
        <key column="PAYMENT_ID"/>
        <property name="creditCardType" column="CCTYPE"/>
    </joined-subclass>
    <joined-subclass name="CashPayment" table="CASH_PAYMENT">
        <key column="PAYMENT_ID"/>
        <!-- ... -->
    </joined-subclass>
</class>

使用辨别标志Discriminator的方式

<class name="Payment" table="PAYMENT">
    <id name="id" type="long" column="PAYMENT_ID">
        <generator class="native"/>
    </id>
    <discriminator column="PAYMENT_TYPE" type="string"/>
    <property name="amount" column="AMOUNT"/>
    <subclass name="CreditCardPayment" discriminator-value="CREDIT">
        <join table="CREDIT_PAYMENT">
            <key column="PAYMENT_ID"/>
            <property name="creditCardType" column="CCTYPE"/>
        </join>
    </subclass>
    <subclass name="CashPayment" discriminator-value="CASH">
        <join table="CASH_PAYMENT">
            <key column="PAYMENT_ID"/>
            <!-- ... -->
        </join>
    </subclass>

</class>

混用每个类分层结构一张表和每个子类一张表

<class name="Payment" table="PAYMENT">
    <id name="id" type="long" column="PAYMENT_ID">
        <generator class="native"/>
    </id>
    <discriminator column="PAYMENT_TYPE" type="string"/>
    <property name="amount" column="AMOUNT"/>
    <subclass name="CreditCardPayment" discriminator-value="CREDIT">
        <join table="CREDIT_PAYMENT">
            <property name="creditCardType" column="CCTYPE"/>
        </join>
    </subclass>
    <subclass name="CashPayment" discriminator-value="CASH">
        <!-- ... -->
    </subclass>

</class>

对上述任何一种映射策略而言，指向根类Payment的关联是使用<many-to-one>进行映射的。

<many-to-one name="payment" column="PAYMENT_ID" class="Payment"/>

每个具体类一张表
第一种方法：<union-subclass>。

  • 两张与子类相关的表。
  • 每张表为对应类的所有属性（包括从超类继承的属性）定义相应字段，在子类表中添加了重复列。
  • 假若超类是抽象类，抽象类不对应表，用abstract="true"。
  • 假若不是抽象的，需要一个额外的表（这里默认是PAYMENT），来保存超类的实例。
  • 局限：
    • 如果一个属性在超类中做了映射，其字段名必须与所有子类表中定义的相同。
    • 不允许在联合子类union subclass的继承层次中使用标识生成器策略identity generator strategy。
<!-- abstract默认为false，设为true表示本类不对应表（类可以不是abstract的），这时会忽略table属性 -->
<class name="Payment">
    <id name="id" type="long" column="PAYMENT_ID">
        <!-- 每个具体类一张表时，主键生成策略不能是identity，因为在整个继承结构中，主键值不能重复 -->
        <generator class="sequence"/>
    </id>
    <property name="amount" column="AMOUNT"/>
    <!-- 指定子类，将父表的列添加到此表中 -->
    <union-subclass name="CreditCardPayment" table="CREDIT_PAYMENT">
        <property name="creditCardType" column="CCTYPE"/>
    </union-subclass>
    <union-subclass name="CashPayment" table="CASH_PAYMENT">
        <!-- ... -->
    </union-subclass>
</class>

第二种方法：隐式多态。

  • 没有明确的提及Payment。
  • Payment的属性在每个子类中都进行了映射。
  • 缺陷：执行多态查询时(polymorphic queries)无法生成带UNION的SQL语句。
<!-- 每个子类都在各自独立的元素<class>中映射 -->
<class name="CreditCardPayment" table="CREDIT_PAYMENT">
    <id name="id" type="long" column="CREDIT_PAYMENT_ID">
        <generator class="native"/>
    </id>
    <property name="amount" column="CREDIT_AMOUNT"/>
</class>

<class name="CashPayment" table="CASH_PAYMENT">
    <id name="id" type="long" column="CASH_PAYMENT_ID">
        <generator class="native"/>
    </id>
    <property name="amount" column="CASH_AMOUNT"/>
</class>

对于这种映射策略而言，通常用<any>来实现到Payment的多态关联映射。

<any name="payment" meta-type="string" id-type="long">
    <meta-value value="CREDIT" class="CreditCardPayment"/>
    <meta-value value="CASH" class="CashPayment"/>
    <column name="PAYMENT_CLASS"/>
    <column name="PAYMENT_ID"/>
</any>

第四种稍有不同的多态映射策略：隐式多态(implicit polymorphism)

对于同一个继承层次内的不同分支，可以采用不同的映射策略，然后用隐式多态来完成跨越整个层次的多态。

在同一个<class>根元素下，不支持混合了元素<subclass>、<joined-subclass>和<union-subclass>的映射。

在同一个<class>根元素下，可以混合使用“每个类分层结构一张表”和“每个子类一张表”这两种映射策略，这是通过结合元素<subclass>和<join>来实现的。

在多个映射文件中，可以直接在<hibernate-mapping>根下定义<subclass>、<joined-subclass>和<union-subclass>。

可以仅加入一个新的映射文件来扩展类层次。

<!-- 在每个映射文件里，超类必须在子类之前定义 -->
<hibernate-mapping>
    <subclass name="DomesticCat" extends="Cat" discriminator-value="D">
        <property name="name" type="string"/>
    </subclass>
</hibernate-mapping>

隐式多态和其他继承映射混合使用

<class name="CreditCardPayment" table="CREDIT_PAYMENT">
    <id name="id" type="long" column="CREDIT_PAYMENT_ID">
        <generator class="native"/>
    </id>
    <discriminator column="CREDIT_CARD" type="string"/>
    <property name="amount" column="CREDIT_AMOUNT"/>
    <!-- ... -->
    <subclass name="MasterCardPayment" discriminator-value="MDC"/>
    <subclass name="VisaPayment" discriminator-value="VISA"/>
</class>

<class name="NonelectronicTransaction" table="NONELECTRONIC_TXN">
    <id name="id" type="long" column="TXN_ID">
        <generator class="native"/>
    </id>
    <!-- ... -->
    <joined-subclass name="CashPayment" table="CASH_PAYMENT">
        <key column="PAYMENT_ID"/>
        <property name="amount" column="CASH_AMOUNT"/>
        <!-- ... -->
    </joined-subclass>

</class>
  • 没有明确的提到Payment。
  • 如果针对接口Payment执行查询，如from Payment，自动返回CreditCardPayment(和它的子类，因为它们也实现了接口Payment)、CashPayment和Chequepayment的实例，但不返回NonelectronicTransaction的实例。
