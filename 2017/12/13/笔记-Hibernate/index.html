Hibernate对象有三种状态：临时态(Transient)、 持久态(Persistent)、游离态(Detached)。

  • 临时状态：对象不在Session的缓存中，在数据库中没有与之对应的记录。进入临时状态：
    • 通过new语句创建新对象。
    • 执行delete()方法
      • 对于游离状态的对象，delete()方法删除数据库中对应的记录；
      • 对于持久化状态的对象，delete()方法删除数据库中对应的记录，并将其在Session缓存中删除。
  • 持久化状态：对象被持久化到Session对象被销毁之前的状态，对象在Session的缓存中，在数据库中有对应的记录。Session在清理缓存的时候，会根据持久化对象的属性变化更新数据库。进入持久化状态：
    • save()或saveOrUpdate()方法，使临时对象转变为持久化对象。
    • upda()或saveOrUpdate()方法，使游离对象转变为持久化对象。
    • load()或get()方法，返回的对象都是持久化对象。
    • find()方法，返回List集合中存放的都是持久化对象。
    • 在允许级联保存的情况下，Session在清理缓存时会把与持久化对象关联的临时对象转变为持久化对象。
  • 游离状态：从持久化对象的Session对象被销毁到该对象消失之前的状态，对象不在Session的缓存中，在数据库中有对应的记录。进入游离状态：
    • close()方法，将Session缓存清空，缓存中的所有持久化对象将转变成游离对象。
    • evict()方法，能从缓存中删除一个持久化对象，使之转变成游离对象。
  • 当一个对象被new了以后此对象处于临时态。
  • 对此对象执行session的save() 或者saveOrUpdate()方法后，此对象被放入session的一级缓存进入持久态。
  • 再对此对象执行evict()/close()/clear()的操作后此对象进入游离态）。
  • 游离态和临时态的对象由于没有被session管理会在适当的时机被垃圾回收回收。
  • session的get()/load()/find()/iternte()等方法从数据库里查询的到的对象，处于持久态。
  • 对数据库中的纪录进行update()/saveOrUpdate()/lock()操作后游离态的对象就过渡到持久态。
  • 处于持久态与游离态的对象在数据库中都有对应的记录。
  • 临时态与游离态的对象都可以被回收可是临时态的对象在数据库中没有对应的纪录，而游离态的对象在数据库中有对用的纪录。
  • 执行save()方法时并不会将对象存入数据库，在提交事务时，对象才被真正的保存到数据库中。
  • Hibernate在提交事务的时候，会将之前对对象做的操作一并提交，就算是在save()方法后的操作也一样。

Configuration

  • 将配置文件信息加载到内存，并创建SessionFactory对象，得到SessionFactory对象后，Configuration对象的使命就结束了。
  • 数据库连接：hibernate.properties 和 hibernate.cfg.xml。
  • 类映射设置：Java 类和数据库表格之间的联系。
  • 属性文件（hibernate.properties）：Configuration cfg = new Configuration();
  • Xml文件（hibernate.cfg.xml）：Configuration cfg = new Configuration().configure();

SessionFactory

  • 线程安全，由应用多个线程共享。
  • 重量级对象，通常在应用程序启动时创造。
  • 每个数据库需要一个 SessionFactory 对象使用单独的配置文件，使用多种数据库要创造多种 SessionFactory 对象。
  • 有二级缓存(可选)。

Session

  • 一个会话用于与数据库的物理连接。
  • 轻量级。
  • 非线程安全。避免多个线程共享同一个Session实例，为每一个请求分配一个Session。
  • 有一级缓存。

Hibernate 4后，buildSessionFactory()被标记为过时，官方建议使用buildSessionFactory(ServiceRegistry sr)。
