附：

  • 不使用 Spring 的 Web 插件

每刷新一次页面（每提交一次请求），就 new 出一个新的 Spring 容器。

将 Spring 容器的创建放在 Servlet 初始化时，即 init()方法时执行。
Servlet 还是单例多线程的，一个业务只有一个 Servlet 实例，这样，Spring 容器就具有了唯一性。
但 LoginServlet 只有一个，还会有StudentServlet、TeacherServlet 等。
每个业务都有一个 Servlet，执行自己的 init()方法，创建一个 Spring 容器，这样 Spring 容器就又不唯一。

  • 使用 Spring 的 的 Web 插件
    一个 Web 应用，只有一个 ServletContext 对象，该对象是在 Web 应用装载时初始化的。
    若将 Spring 容器的创建放到 ServletContext 的初始化方法中，并将创建好的 Spring 容器作为 ServletContext 的属性放入其中，需要 Spring 容器，直接读取该属性值即可。
    ServletContext 对象生命周期与 Web 应用的相同，即放在其中的属性为全局属性。
    所以，放入 ServletContext 中的 Spring 容器，在整个应用的生命周期中，均可被访问。
    这样保证 Spring 容器在 Web 应用中的唯一性了。
    上述工作已经被封装在 spring-web-4.2.1.RELEASE.jar。

在 ServletContext 初始化时创建 Spring 容器，需要监听器接口 ServletContextListener 对 ServletContext 监听。
Spring 为该监听器接口定义了一个实现类 ContextLoaderListener，专门用于在 ServletContext 初始化时创建 Spring 容器。
在 web.xml 中注册ContextLoaderListener 监听器。
ContextLoaderListener 在创建 Spring 容器时，加载 Spring 配置文件。
默认的 Spring 配置文件位置与名称为：WEB-INF/applicationContext.xml。
一般会将该配置文件放置于项目的 classpath 下，即 src 下，需要在 web.xml <context-param/>中对 Spring 配置文件的位置及名称指定。
