[笔记]-Spring-监听器获取Spring配置文件

2017-12-05

12-27重新整理更新Spring笔记

直接加载beans.xml文件

// @Component 监听器是web层，是tomcat实例化的，不是Spring实例化的，不能放到Spring中
public class InitDataListener implements ServletContextListener {

    private ProductService productService = null;

    @Override
    public void contextDestroyed(ServletContextEvent event) {}

    @Override
    public void contextInitialized(ServletContextEvent event) {
        ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
        productService = (ProductService) context.getBean("productService");
        System.out.println(productService);
    }

}

web.xml

<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
<!-- 加载 Spring 容器 -->
<context-param>
    <!-- 读取其他 XML 配置文件 -->
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:beans.xml</param-value>
</context-param>
<listener>
    <listener-class>mmp.InitDataListener</listener-class>
</listener>

加载了两次配置文件，bean被实例化了两次，Spring通过自己的监听器已经加载过一次，没必要再加载一次。

从ServletContext中获取

public class InitDataListener implements ServletContextListener {

    private ProductService productService = null;

    @Override
    public void contextDestroyed(ServletContextEvent event) {}

    @Override
    public void contextInitialized(ServletContextEvent event) {

        // 项目在启动时，把Spring配置文件通过Spring的监听器加载，存储到ServletContext中，在ServletContext中获取即可
        ApplicationContext context = (ApplicationContext) event.getServletContext()
            .getAttribute(WebApplicationContext.ROOT_WEB_APPLICATION_CONTEXT_ATTRIBUTE);
        productService = (ProductService) context.getBean("productService");
        System.out.println(productService);
    }

}

通过Spring提供的工具类加载

public class InitDataListener implements ServletContextListener {

    private ProductService productService = null;

    @Override
    public void contextDestroyed(ServletContextEvent event) {}

    @Override
    public void contextInitialized(ServletContextEvent event) {

        WebApplicationContext context = WebApplicationContextUtils
            .getWebApplicationContext(event.getServletContext());
        productService = (ProductService) context.getBean("productService");
        System.out.println(productService);
    }
}

一般使用第三种方法来获取Spring的配置文件，从而获取相应的实例化bean。

不使用Spring的Web插件

public void doPost(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    // 每刷新一次（提交一次请求），就 new 出一个新的 Spring 容器
    ApplicationContext context = new ClassPathXmlApplicationContext(new String[] { "applicationContext.xml" });
    System.out.println(context);

    // 如果将创建Spring容器放在Servlet初始化（init()）时，Servlet是单例多线程的，只有一个Servlet实例，这样就唯一了
    // 但LoginServlet只有一个，还有StudentServlet、TeacherServlet等，执行自己的init()方法，这样Spring容器就又不唯一
}

使用Spring的Web插件

  • spring-web-4.2.1.RELEASE.jar
    • 一个Web应用，只有一个ServletContext对象，在Web应用装载时初始化
    • 将创建容器放到ServletContext的初始化方法中，并作为ServletContext的属性放入，需要Spring容器，直接读取该属性，保证唯一
  • 在web.xml中注册ContextLoaderListener监听器
    • ServletContext初始化时创建Spring容器，需要监听器接口ServletContextListener监听ServletContext
    • 该监听器接口实现类ContextLoaderListener专门用于在ServletContext初始化时创建Spring容器
    • ContextLoaderListener在创建容器时，加载Spring配置文件applicationContext.xml，也可以在<context-param/>中指定
public void doPost(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    // 获取Spring 容器，不是创建
    WebApplicationContext context = WebApplicationContextUtils
        .getRequiredWebApplicationContext(this.getServletContext());
    System.out.println(context);
}
