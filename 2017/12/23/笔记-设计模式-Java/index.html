单例模式

// 饿汉式单例：在单例类被加载时候，就实例化一个对象交给自己的引用
class Singleton {
    // 指向自己实例的私有静态引用
    private static Singleton singleton = new Singleton();
    // 私有构造方法
    private Singleton() {}
    // 以自己实例为返回值的静态公有方法
    public static Singleton getInstance() {
        return singleton;
    }
}

// 懒汉式单例：在调用取得实例方法的时候才会实例化对象
class Singleton {
    private static Singleton singleton;
    private Singleton() {}
    public static synchronized Singleton getInstance() {
        if (singleton == null) {
            singleton = new Singleton();
        }
        return singleton;
    }
}

Class c = Class.forName(Singleton.class.getName());
Constructor ct = c.getDeclaredConstructor();
ct.setAccessible(true);
// 每次运行都会产生新的单例对象
Singleton singleton = (Singleton)ct.newInstance();

  • 只能使用单例类提供的方法得到单例对象，不要使用反射，否则将会实例化一个新对象。

  • 不要做断开单例类对象与类中静态引用的危险操作。

  • 多线程使用单例使用共享资源时，注意线程安全问题。

工厂模式：简单工厂模式（静态工厂模式）、工厂方法模式、抽象工厂模式。

抽象工厂模式

// 产品接口
interface IProduct1 {
    public void show();
}

interface IProduct2 {
    public void show();
}
// 产品实现
class Product1 implements IProduct1 {
    public void show() {}
}

class Product2 implements IProduct2 {
    public void show() {}
}
// 工厂接口
interface IFactory {
    public IProduct1 createProduct1();
    public IProduct2 createProduct2();
}
// 工厂实现
class Factory implements IFactory {
    public IProduct1 createProduct1() {
        return new Product1();
    }

    public IProduct2 createProduct2() {
        return new Product2();
    }
}

public class Client {
    public static void main(String[] args) {
        IFactory factory = new Factory();
        factory.createProduct1().show();
        factory.createProduct2().show();
    }
}

  • 工厂方法模式提供的所有产品都是衍生自同一个接口或抽象类，而抽象工厂模式所提供的产品则是衍生自不同的接口或抽象类。

  • 对于抽象工厂模式，当减少一个方法使的提供的产品不再构成产品族之后，它就变成了工厂方法模式。

  • 扩展只需要再增加一个工厂类的实现就可以。

  • 简单工厂只有三个要素，没有工厂接口，并且得到产品的方法一般是静态的。

  • 工厂模式最终目的是解耦。

建造者模式

// 产品类
class Product {
    private String name;
    private String type;

    public void showProduct() {
        System.out.println("名称：" + name);
        System.out.println("型号：" + type);
    }

    // setters
}
// 抽象建造者
// 将建造的具体过程交与子类来实现，更容易扩展
abstract class Builder {
    // 建造产品
    public abstract void setPart(String arg1, String arg2);
    // 返回产品
    public abstract Product getProduct();
}
// 建造者
class ConcreteBuilder extends Builder {
    private Product product = new Product();
    // 实现方法
    public Product getProduct() {
        return product;
    }

    public void setPart(String arg1, String arg2) {
        product.setName(arg1);
        product.setType(arg2);
    }
}
// 导演类
public class Director {
    // 导演类一般不与产品类发生依赖关系，与导演类直接交互的是建造者类
    private Builder builder = new ConcreteBuilder();
    // 调用适当的建造者来组建产品
    public Product getAProduct() {
        builder.setPart("宝马汽车", "X7");
        return builder.getProduct();
    }

    public Product getBProduct() {
        builder.setPart("奥迪汽车", "Q5");
        return builder.getProduct();
    }
}

public class Client {
    public static void main(String[] args) {
        Director director = new Director();

        Product product1 = director.getAProduct();
        product1.showProduct();

        Product product2 = director.getBProduct();
        product2.showProduct();
    }
}

  • 建造者模式的封装性很好，一般产品类和建造者类比较稳定，主要的业务逻辑封装在导演类，容易扩展，实现一个新的建造者类就可以。

  • 建造者模式只比工厂模式多了一个导演类，一般用来创建复杂对象，将对象的创建过程独立出来组成导演类。

  • 工厂模式是将对象的全部创建过程封装在工厂类中，由工厂类向客户端提供最终的产品；

  • 而建造者模式中，建造者类一般只提供产品类中各个组件的建造，而将具体建造过程交付给导演类。

原型模式：主要用于对象的复制

// 实现Cloneable接口，才可以clone
class Prototype implements Cloneable {
    // 重写Object类的clone()方法，将protected改为public
    public Prototype clone() {
        Prototype prototype = null;
        try {
            prototype = (Prototype) super.clone();
        } catch (CloneNotSupportedException e) {
            e.printStackTrace();
        }
        return prototype;
    }
}

class ConcretePrototype extends Prototype {
    public void show() {
        System.out.println("原型模式实现类");
    }
}

public class Client {
    public static void main(String[] args) {
        ConcretePrototype cp = new ConcretePrototype();
        for (int i = 0; i < 10; i++) {
            ConcretePrototype clonecp = (ConcretePrototype) cp.clone();
            clonecp.show();
        }
    }
}

  • 原型模式创建对象比直接new一个对象性能好，需要重复地创建相似对象时可以考虑使用原型模式，比如需要在一个循环体内创建对象。

  • 使用原型模式复制对象不会调用类的构造方法。

    • 单例模式中，将构造方法的访问权限设置为private，但是clone方法无视构造方法，所以，单例模式与原型模式是冲突的。

  • Object类的clone方法只拷贝对象中的基本数据类型，这是浅拷贝。要实现深拷贝，必须将原型模式中的引用对象等另行拷贝：

public class Prototype implements Cloneable {
    private ArrayList list = new ArrayList();

    public Prototype clone() {
        Prototype prototype = null;
        try {
            prototype = (Prototype) super.clone();
            // 大部分的容器类都实现了Cloneable接口
            prototype.list = (ArrayList) this.list.clone();
        } catch (CloneNotSupportedException e) {
            e.printStackTrace();
        }
        return prototype;
    }
}

模版方法模式

购物一般需要三个步骤：登录、挑选商品、付款，对于登录与付款，每个人操作相同，但每个人挑选的商品是不同的。

某类的某个方法的实现，需要几个固定步骤，对于该类的不同对象，有些步骤是固定的，有些步骤是不同的，有些步骤是可变可不变的。

这种情况适合模板方法：定义一个操作中某种算法的框架，而将一些步骤延迟到子类中。

模板方法中：由一个抽象类和一个（或一组）实现类通过继承结构组成：

  • 模板方法：实现某种算法的方法步骤。
  • 步骤方法：完成模板方法的每个阶段性方法：
    • 抽象方法：子类必须实现。
    • 最终方法：子类不能重写的方法，所有子类执行都一样的步骤。
    • 钩子方法：父类给出了默认实现，但子类也可以重写。
abstract class ShoppingService {
    // 模版方法：抽象类声明并实现，大多会定义为final
    public void shopping() {
        login();
        choose();
        pay();
    }

    // 子类不能重写
    public final void login() {}

    // 子类必须实现
    public abstract void choose();

    // 钩子方法：子类可以重写
    public void pay() {}
}

class ShoseShoppingService extends ShoppingService {
    @Override
    public void choose() {}
}

class DressShoppingService extends ShoppingService {
    @Override
    public void choose() {}

    @Override
    public void pay() {}
}

中介者模式

abstract class AbstractColleague {
    protected int number;

    // getter setter

    // 中介者
    public abstract void setNumber(int number, AbstractMediator am);
}
// 同事类之间相互影响，相互依赖
class ColleagueA extends AbstractColleague {
    // 通过中介者消息传递
    public void setNumber(int number, AbstractMediator am) {
        this.number = number;
        am.AaffectB();
    }
}

class ColleagueB extends AbstractColleague {
    @Override
    public void setNumber(int number, AbstractMediator am) {
        this.number = number;
        am.BaffectA();
    }
}

// 抽象中介者
abstract class AbstractMediator {
    protected AbstractColleague A;
    protected AbstractColleague B;

    public AbstractMediator(AbstractColleague a, AbstractColleague b) {
        A = a;
        B = b;
    }
    // 各个同事类之间的通信
    public abstract void AaffectB();

    public abstract void BaffectA();
}
// 中介者实现类
class Mediator extends AbstractMediator {
    public Mediator(AbstractColleague a, AbstractColleague b) {
        super(a, b);
    }
    // 处理A对B的影响
    public void AaffectB() {
        int number = A.getNumber();
        B.setNumber(number * 100);
    }

    // 处理B对A的影响
    public void BaffectA() {
        int number = B.getNumber();
        A.setNumber(number / 100);
    }
}

public class Client {
    public static void main(String[] args) {
        AbstractColleague collA = new ColleagueA();
        AbstractColleague collB = new ColleagueB();
        AbstractMediator am = new Mediator(collA, collB);

        // 通过设置A影响B
        collA.setNumber(1000, am);

        // 通过设置B影响A
        collB.setNumber(1000, am);

    }
}

其实就是把处理对象关系的代码封装到中介类中，通过这个中介类来处理对象间的关系。
同事类之间的关系将变为星型结构，类的变动，只会影响类本身，以及中介者，减小了耦合。

观察者模式：如果一个对象的状态发生改变，与它相关的对象也要随之做出相应的变化。

// 被观察者：接口或抽象类
abstract class Subject {
    // 存放观察者对象，Vector线程安全，List不安全
    private Vector obs = new Vector();
    // 添加观察者对象
    public void addObserver(Observer obs) {
        this.obs.add(obs);
    }
    // 移除观察者对象
    public void delObserver(Observer obs) {
        this.obs.remove(obs);
    }
    // 依次调用观察者对象的对应方法
    protected void notifyObserver() {
        for (Observer o : obs) {
            o.update();
        }
    }

    public abstract void doSomething();
}
// 具体的观察者：定义被观察者对象状态发生变化时要处理的逻辑
class ConcreteSubject extends Subject {
    public void doSomething() {
        System.out.println("被观察者事件反生");
        this.notifyObserver();
    }
}
// 观察者
interface Observer {
    // update()：被观察者状态发生变化时，被触发
    public void update();
}
// 具体的被观察者：便于扩展，定义具体的业务逻辑
class ConcreteObserver1 implements Observer {
    public void update() {
        System.out.println("观察者1收到信息，并进行处理。");
    }
}

class ConcreteObserver2 implements Observer {
    public void update() {
        System.out.println("观察者2收到信息，并进行处理。");
    }
}

public class Client {
    public static void main(String[] args) {
        Subject sub = new ConcreteSubject();

        sub.addObserver(new ConcreteObserver1()); // 添加观察者1
        sub.addObserver(new ConcreteObserver2()); // 添加观察者2
        sub.doSomething();
    }
}

就是在Subject类中关联一下Observer类，并且在doSomething方法中遍历一下Observer的update方法就行了。

Java有Observer接口，以及实现类Observable，对观察者角色常进行了实现。

访问者模式

// 抽象元素类
abstract class Element {
    // 声明接受哪一类访问者访问
    public abstract void accept(IVisitor visitor);
    // 本身的业务逻辑
    public abstract void doSomething();
}
// 抽象访问者
interface IVisitor {
    // 定义哪些对象是可以被访问的
    public void visit(ConcreteElement1 el1);

    public void visit(ConcreteElement2 el2);
}
// 元素类
class ConcreteElement1 extends Element {
    public void doSomething() {
        System.out.println("这是元素1");
    }
    // 实现抽象元素类所声明的accept方法
    public void accept(IVisitor visitor) {
        visitor.visit(this);
    }
}

class ConcreteElement2 extends Element {
    public void doSomething() {
        System.out.println("这是元素2");
    }

    public void accept(IVisitor visitor) {
        visitor.visit(this);
    }
}
// 访问者
class Visitor implements IVisitor {
    // 实现抽象访问者所声明的方法，访问者访问到一个类后干什么
    public void visit(ConcreteElement1 el1) {
        el1.doSomething();
    }

    public void visit(ConcreteElement2 el2) {
        el2.doSomething();
    }
}
// 结构对象
class ObjectStruture {
    public static List getList() {
        // 元素的容器
        List list = new ArrayList();
        Random ran = new Random();
        for (int i = 0; i < 10; i++) {
            int a = ran.nextInt(100);
            if (a > 50) {
                list.add(new ConcreteElement1());
            } else {
                list.add(new ConcreteElement2());
            }
        }
        return list;
    }
}

public class Client {
    public static void main(String[] args) {
        List list = ObjectStruture.getList();
        for (Element e : list) {
            e.accept(new Visitor());
        }
    }
}

命令模式

// 调用者
class Invoker {
    private Command command;

    public void setCommand(Command command) {
        this.command = command;
    }
    // 调用命令
    public void action() {
        this.command.execute();
    }
}

// Command
abstract class Command {
    // 执行命令
    public abstract void execute();
}
// Command类实现类
class ConcreteCommand extends Command {
    private Receiver receiver;

    public ConcreteCommand(Receiver receiver) {
        this.receiver = receiver;
    }
    // 命令封装
    public void execute() {
        this.receiver.doSomething();
    }
}
// 接收者
class Receiver {
    // 接收并执行命令
    public void doSomething() {
        // 封装操作
        System.out.println("接受者-业务逻辑处理");
    }
}

public class Client {
    public static void main(String[] args) {
        Receiver receiver = new Receiver();
        Command command = new ConcreteCommand(receiver);
        // 客户端直接执行具体命令方式
        command.execute();
        // 客户端通过调用者来执行命令
        Invoker invoker = new Invoker();
        invoker.setCommand(command);
        // 调用者类==>命令类==>接收者类
        // 一条命令被分成了三步，命令的调用者与执行者分开
        invoker.action();
    }
}

责任链模式

public void test(int i, Request request) {
    if (i == 1) {
        Handler1.response(request);
    } else if (i == 2) {
        Handler2.response(request);
    } else if (i == 3) {
        Handler3.response(request);
    } else {
        Handler4.response(request);
    }
}
// 模拟判定条件
class Level {
    private int level = 0;

    public Level(int level) {
        this.level = level;
    };

    public boolean above(Level level) {
        if (this.level >= level.level) {
            return true;
        }
        return false;
    }
}
// 请求
class Request {
    Level level;

    public Request(Level level) {
        this.level = level;
    }

    public Level getLevel() {
        return level;
    }
}
// 响应
class Response {
}
// 抽象处理类
abstract class Handler {
    // 指向下一处理类
    private Handler nextHandler;
    // 处理请求的方法，把判定条件的部分放到处理类中
    public final Response handleRequest(Request request) {
        Response response = null;
        // 如果满足条件，则本处理类处理，否则nextHandler处理
        // 只有处理类的处理等级高于Request的等级才处理，否则交给下一个
        if (this.getHandlerLevel().above(request.getLevel())) {
            response = this.response(request);
        } else {
            if (this.nextHandler != null) {
                this.nextHandler.handleRequest(request);
            } else {
                System.out.println("-----没有合适的处理器-----");
            }
        }
        return response;
    }

    public void setNextHandler(Handler handler) {
        this.nextHandler = handler;
    }

    protected abstract Level getHandlerLevel();

    public abstract Response response(Request request);
}
// 具体处理类
class ConcreteHandler1 extends Handler {
    protected Level getHandlerLevel() {
        return new Level(1);
    }

    public Response response(Request request) {
        System.out.println("-----请求由处理器1进行处理-----");
        return null;
    }
}

class ConcreteHandler2 extends Handler {
    protected Level getHandlerLevel() {
        return new Level(3);
    }

    public Response response(Request request) {
        System.out.println("-----请求由处理器2进行处理-----");
        return null;
    }
}

class ConcreteHandler3 extends Handler {
    protected Level getHandlerLevel() {
        return new Level(5);
    }

    public Response response(Request request) {
        System.out.println("-----请求由处理器3进行处理-----");
        return null;
    }
}

public class Client {
    public static void main(String[] args) {
        Handler handler1 = new ConcreteHandler1();
        Handler handler2 = new ConcreteHandler2();
        Handler handler3 = new ConcreteHandler3();

        handler1.setNextHandler(handler2);
        handler2.setNextHandler(handler3);
        // 执行时将请求交给第一个处理类
        Response response = handler1.handleRequest(new Request(new Level(4)));
    }
}

interface IStrategy {

迭代器模式：使用最多的模式，Java语言已经完整地实现了迭代器模式。

// 抽象迭代器
interface Iterator {
    // 定义遍历元素所需要的方法

    // 取得下一个元素
    public Object next();
    // 判断是否遍历结束
    public boolean hasNext();
}
// 迭代器实现
class ConcreteIterator implements Iterator {
    private List list = new ArrayList();
    private int cursor = 0;

    public ConcreteIterator(List list) {
        this.list = list;
    }

    public boolean hasNext() {
        if (cursor == list.size()) {
            return false;
        }
        return true;
    }

    public Object next() {
        Object obj = null;
        if (this.hasNext()) {
            obj = this.list.get(cursor++);
        }
        return obj;
    }
}

// 抽象容器，如List接口
interface Aggregate {
    public void add(Object obj);

    public void remove(Object obj);
    // 提供一个iterator()方法
    public Iterator iterator();
}
// 具体容器，如ArrayList
class ConcreteAggregate implements Aggregate {
    private List list = new ArrayList();

    public void add(Object obj) {
        list.add(obj);
    }

    public Iterator iterator() {
        return new ConcreteIterator(list);
    }

    public void remove(Object obj) {
        list.remove(obj);
    }
}

public class Client {
    public static void main(String[] args) {
        Aggregate ag = new ConcreteAggregate();

        ag.add("小明");
        ag.add("小红");

        Iterator it = ag.iterator();
        while (it.hasNext()) {
            String str = (String) it.next();
            System.out.println(str);
        }
    }
}

备忘录模式：回滚，JDBC事务，文本编辑器的Ctrl+Z恢复等。

// 备忘录

class Memento {
    // 存储Originator类中state变量的临时状态
    private Map stateMap;
    // 存储发起人对象的内部状态
    public Memento(Map map) {
        this.stateMap = map;
    }

    // getter setter

class BeanUtils {
    public static Map backupProp(Object bean) {
        Map result = new HashMap();
        try {
            BeanInfo beanInfo = Introspector.getBeanInfo(bean.getClass());
            PropertyDescriptor[] descriptors = beanInfo.getPropertyDescriptors();
            for (PropertyDescriptor des : descriptors) {
                String fieldName = des.getName();
                Method getter = des.getReadMethod();
                Object fieldValue = getter.invoke(bean, new Object[] {});
                if (!fieldName.equalsIgnoreCase("class")) {
                    result.put(fieldName, fieldValue);
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
        return result;
    }

    public static void restoreProp(Object bean, Map propMap) {
        try {
            BeanInfo beanInfo = Introspector.getBeanInfo(bean.getClass());
            PropertyDescriptor[] descriptors = beanInfo.getPropertyDescriptors();
            for (PropertyDescriptor des : descriptors) {
                String fieldName = des.getName();
                if (propMap.containsKey(fieldName)) {
                    Method setter = des.getWriteMethod();
                    setter.invoke(bean, new Object[] { propMap.get(fieldName) });
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
