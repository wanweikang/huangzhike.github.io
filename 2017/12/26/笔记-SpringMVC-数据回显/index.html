数据回显：提交后，如果出现错误，将刚才提交的数据回显到刚才的提交页面。

将数据放在request域中，然后请求转发到表单页面，EL表达式将数据回显。

@RequestMapping("/editItemSubmit")
public String editItemSubmit(Model model, Integer id, ItemsCustom itemsCustom) {
    // 数据回显
    model.addAttribute("id", id); // 传入到request域
    // POJO
    model.addAttribute("item", itemsCustom); // 传入到request域
    return "editItem";
}

@ModelAttribute()，括号内传入要回显的POJO，指定POJO回显到页面在request中的key，还可以将方法的返回值传到页面。

// 访问该Controller任何处理方法前调用，将返回值放入Model
// 提取的公用方法，也可以去掉，但是要在处理器方法中处理
// itemsCustom表示最终将方法返回值放在request中的key
@ModelAttribute("itemsCustom")
public ItemsCustom getItemsCustom() {
    // new一个
    ItemsCustom itemsCustom = new ItemsCustom();
    // ...
    return itemsCustom;
}

@RequestMapping("/editItemSubmit")
// 根据请求参数填充覆盖itemsCustom对象
public String editItemSubmit(@ModelAttribute("itemsCustom") ItemsCustom itemsCustom) {
    // 不用在每个Controller方法通过Model将数据传到页面
    return "editItem";
}

@ModleAttribute()也是将数据填入到request域中，然后通过EL表达式取出，同model.addAttribute()和modelAndView.addObject()。

数据类型转换发生异常后，返回到表单页面，让用户重写。

  • 正常情况下，发生TypeMismatchException，系统自动跳转到 400 页面。

  • 要在发生类型转换异常后，跳转到指定页面，需要捕获异常，然后通过异常处理器跳转到指定页面。

  • 若仅仅是跳转，SimpleMappingExceptionResolver就可以，当返回到表单页面后，还将数据回显，让用户更正，需要自定义异常处理器。

数据回显原理：在异常处理器中，通过request.getParameter()获取用户输入的表单原始数据，放到ModelAndView的Model中，然后从要转向的页面中通过 EL 表达式取出。

@Controller
@RequestMapping("/test")
public class MyController {
    @RequestMapping(value = "/register.do")
    public String register(int age, Date birthday) {
        System.out.println("age = " + age);
        System.out.println("birthday = " + birthday);
        return "/WEB-INF/jsp/show.jsp";
    }

    @ExceptionHandler(TypeMismatchException.class)
    public ModelAndView exceptionResolver(Exception ex, HttpServletRequest request) {
        ModelAndView mv = new ModelAndView();
        mv.addObject("ex", ex);

        String age = request.getParameter("age");
        String birthday = request.getParameter("birthday");
        mv.addObject("age", age);
        mv.addObject("birthday", birthday);

        mv.setViewName("/index.jsp");
        return mv;
    }
}

年龄有误，日期正确，回显没问题。

日期有误，年龄正确，回显有问题。

  • 日期有误时，SimpleDateFormat.parse()方法抛出ParseException，非TypeMismatchException。

  • 实现的自定义类型转换器的 convert()方法不能抛出异常，对异常的处理必须为 try-catch。

  • ParseException发生时，convert()方法自己处理了异常，JVM不知道有异常，异常处理器也无法捕获到ParseException。

解决：当日期格式不符合要求时，手动一个类型匹配异常。

public class MyConverter implements Converter<String, Date> {
    @Override
    public Date convert(String source) {
        try {
            if (source != null && !"".equals(source.trim())) {
                SimpleDateFormat sdf = getSimpleDateFormat("yyyy-MM-dd");
                return sdf.parse(source);
            }
        } catch (ParseException e) {
        }
        return null;
    }

    private SimpleDateFormat getSimpleDateFormat(String source) {
        SimpleDateFormat sdf = new SimpleDateFormat();
        if (Pattern.matches("^\\d{4}-\\d{2}-\\d{2}$", source)) {
            sdf = new SimpleDateFormat("yyyy-MM-dd");
        } else if (Pattern.matches("^\\d{4}/\\d{2}/\\d{2}$", source)) {
            sdf = new SimpleDateFormat("yyyy/MM/dd");
        } else if (Pattern.matches("^\\d{4}\\d{2}\\d{2}$", source)) {
            sdf = new SimpleDateFormat("yyyyMMdd");
        } else {
            // 抛出
            throw new TypeMismatchException("", Date.class);
        }
        return sdf;
    }
}

TypeMismatchException类第一个参数为要匹配的值，第二个参数为要匹配的类型，两个参数的类型没有is-a关系时，该异常发生。

第一个参数确定为String类型，就写了个String常量，目标类型确定为Date类型。

可以从系统提示信息中查找用户输入数据，确定哪个数据出现了类型转换异常，一旦确定问题数据，异常信息就可以准确替换了。

// 在处理器中定义一个方法，根据发生类型转换异常的数据的不同而生成不同提示信息
public String changeExceptionMessage(Exception ex, String arg, String paramName) {
    String exMsg = ex.getMessage();
    // String 类的 contains()方法判断系统异常信息中是否存在用户输入数据
    if (exMsg.contains(arg)) {
        String newMsg = paramName + "【" + arg + "】 填写有误，请重新填写";
        return newMsg;
    }
    return null;
}

// 在异常处理方法中将新生成的异常信息写入到ModelAndView中的Model，通过EL表达式显示
@ExceptionHandler(TypeMismatchException.class)
public ModelAndView exceptionResolver(Exception ex, HttpServletRequest request) {
    ModelAndView mv = new ModelAndView();

    String age = request.getParameter("age");
    String birthday = request.getParameter("birthday");
    mv.addObject("age", age);
    mv.addObject("ageMsg", ageMsg);

    String ageMsg = changeExceptionMessage(ex, age, "年龄");
    String birthdayMsg = changeExceptionMessage(ex, birthday, "生日");
    mv.addObject("birthday", birthday);
    mv.addObject("birthdayMsg", birthdayMsg);

    mv.setViewName("/index.jsp");
    return mv;
}
