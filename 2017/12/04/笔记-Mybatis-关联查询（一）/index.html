[笔记]-Mybatis-关联查询（一）

2017-12-04 0:23

一对一关联：一个老师只教一个班

或者创建一个pojo继承包括查询字段较多的类。

一对多关联：根据classId查询对应的班级信息,包括学生,老师

自关联是指，自己即充当一方，又充当多方。
如，对于新闻栏目 NewsColumn，可以充当一方，即父栏目，也可以充当多方，即子栏目。
反映到 DB 表中，只有一张表，这张表中具有一个外键，表示该栏目的父栏目。
一级栏目没有父栏目，可以将其外键值设为 0，而子栏目则具有外键值。

以一对多方式处理：如在页面上点击父栏目，显示出其子栏目。
可以分为两种情况：

  • 查询出指定栏目的所有子孙栏目
  • 查询出指定栏目及其所有子孙栏目

查询出指定栏目的所有子孙栏目

查询出指定栏目及其所有子孙栏目

以多对一方式处理：如在网页上显示当前页面的站内位置。

association和collection还具备延迟加载的功能

在主配置文件的<properties/>与<typeAliases/>标签之间，添加<settings/>标签，完成全局参数设置。

延迟加载，是指在进行关联查询时，按照设置延迟规则推迟对关联对象的 select 查询。
MyBatis 的延迟加载只是对关联对象的查询有迟延设置。
关联对象的查询与主加载对象的查询必须是分别进行的 select 语句，不能是使用多表连接所进行的 select 查询。
