[笔记]-Log4J

2017-12-04 0:23

Log4j 由三部分组成：

  • loggers：采集日志信息。
  • appenders：发布日志信息。
  • layouts：格式化日志信息。

Log4j 里有两种类型的对象：

  • 核心对象：框架必需。
    • Logger 对象
    • Layout 对象
      • org.apache.log4j.HTMLLayout：网页布局，以 HTML 表格形式布局
      • org.apache.log4j.SimpleLayout：简单布局，包含日志信息的级别和信息字符串
      • org.apache.log4j.PatternLayout：匹配器布局，可以灵活地指定布局模式
    • Appender 对象
      • org.apache.log4j.ConsoleAppender：日志输出到控制台
      • org.apache.log4j.FileAppender：日志输出到文件
      • org.apache.log4j.RollingFileAppender：当日志文件大小到达指定尺寸的时候将产生一个新的日志文件
      • org.apache.log4j.DailyRollingFileAppender：每天产生一个日志文件
  • 支持对象：框架可选对象
    • Level 对象：日志信息的粒度和优先级。OFF、DEBUG、INFO、ERROR、WARN、FATAL、ALL。
      • Log4j 建议只使用四个级别，优先级从高到低分别是 ERROR、WARN、INFO、DEBUG。
    • Filter 对象：决定该条日志是否被记录。
    • ObjectRenderer 对象：为传入日志框架的不同对象提供字符串形式的表示。
    • LogManager 对象：管理日志框架，读取初始配置参数。

Log4j 的日志输出控制文件，由三部分组成：

  • 输出Appender：输出的位置，是控制台还是文件等。
  • 输出Layout：显示格式。
  • 输出Level：显示级别。

记录具体语句的日志可以：

debug：显示出所执行的 SQL 语句、参数值、对 DB 的影响条数等信息。
trace：显示出查询出的每条记录的每个字段名及值。
若日志对象使用根日志对象 rootLogger，则会输出太多的信息。
