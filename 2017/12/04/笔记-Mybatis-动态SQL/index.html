[笔记]-Mybatis-动态SQL

2017-12-04

12-29重新整理，修改了一些错误

  • if：条件判断
  • choose：(when, otherwize)，相当于switch，类似JSTL的choose
  • trim：对包含的内容加prefix或suffix，前缀或后缀
  • where：简化SQL语句中where条件判断，智能处理and，or，不必担心多余导致语法错误
  • set：更新
  • foreach：in语句查询

prefix和suffix：在包含的内容加上前缀或后缀。

prefixOverrides和suffixOverrides：属性中指定的内容将被移除，插入prefix和suffix属性中指定的内容。

trim代替where

<select id="xxx" parameterType="Blog" resultType="Blog">
    select * from t_blog 
    <trim prefix="where" prefixOverrides="and |or">
        <if test="title != null">
            title = #{title}
        </if>
        <if test="content != null">
            and content = #{content}
        </if>
        <if test="owner != null">
            or owner = #{owner}
        </if>
    </trim>
</select>

trim代替set

<update id="xxx" parameterType="User">
    UPDATE user
    <trim prefix="SET" suffixOverrides=",">
        <if test="username != null and username != '' ">
            username = #{username},
        </if>
        <if test="sex != null and sex != '' ">
            sex = #{sex},
        </if>
        <if test="birthday != null ">
            birthday = #{birthday},
        </if>
    </trim>
    WHERE user_id = #{user_id}
</update>

set主要用于更新。

update语句中没用if标签时，如果有一个参数为null，导致错误。

update语句中使用if标签时，如果前面的if没有执行，或导致逗号多余错误。

set标签动态前置SET关键字，同时消除无关的逗号。

用if+set标签后，如果某项为null则不更新，保持原值。

<update id="xxx" parameterType="Blog">
    update t_blog
    <set>
        <if test="title != null">title = #{title},</if>
        <if test="content != null">content = #{content},</if>
        <if test="owner != null">owner = #{owner}</if>
    </set>
    where id = #{id}
</update>

foreach的主要构建in条件，迭代一个集合。

  • item：遍历的对象。
  • index：迭代过程中，每次迭代到的位置。
  • open：开始遍历时拼接的SQL串。
  • close：结束遍历时拼接的SQL串。
  • separator：遍历的两个对象中需要拼接的SQL串。
  • collection：输入对象中的集合属性：
    • 传入的是单参数且是List。
    • 传入的是单参数且是array数组。
    • 传入多参数，需要封装成Map。

List

<select id="xxx" resultType="Blog">
    select * from t_blog where id in
    <foreach collection="list" index="index" item="item" open="(" separator="," close=")">
        #{item}
    </foreach>
</select>

动态SQL若出现大于号>、小于号<、大于等于号>=，小于等于号<=，最好转换为实体符号，否则可能会出现解析出错问题。

特别是小于号<，在XML中绝对不能出现。

// 提供CRUD对应的SQL语句
public class CategoryDynaSqlProvider {
    public String list() {
        return new SQL().SELECT("*").FROM("category_").toString();
    }

    public String get() {
        return new SQL().SELECT("*").FROM("category_").WHERE("id = #{id}").toString();
    }

    public String add() {
        return new SQL().INSERT_INTO("category_").VALUES("name", "#{name}").toString();
    }

    public String update() {
        return new SQL().UPDATE("category_").SET("name = #{name}").WHERE("id = #{id}").toString();
    }

    public String delete() {
        return new SQL().DELETE_FROM("category_").WHERE("id = #{id}").toString();
    }

}
