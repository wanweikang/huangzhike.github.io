[笔记]-Mybatis-动态SQL

2017-12-04 0:23
  • if 语句：简单的条件判断
  • choose：(when, otherwize) ,相当于 switch ,类似 JSTL 中的 choose
  • trim：对包含的内容加上 prefix,或 suffix，前缀，后缀
  • where：简化SQL语句中 where 条件判断的，能智能的处理 and or ,不必担心多余导致语法错误
  • set：主要用于更新
  • foreach：在实现 in 语句查询时特别有用

如果没有一个能匹配上 SQL 会变成：SELECT * FROM BLOG WHERE

如果<where/>包含的标签中有返回值的话，插入一个 where。
如果标签返回的内容是以 AND 或 OR 开头的，则剔除掉。
在<where/>中不需要考虑空格，MyBatis 会智能加上。

即所有的 when 和 otherwise 条件中，只有一个输出，所有的条件都不满足时输出 otherwise 中的内容。

prefix 和 suffix：在包含的内容加上前缀或后缀。
prefixOverrides 和 suffixOverrides：属性中指定的内容将被移除，插入 prefix 和 suffix 属性中指定的内容。

trim代替where

trim 代替 set

set 主要用于更新时。
当 update 语句中没有使用 if 标签时，如果有一个参数为 null，会导致错误。
当 update 语句中使用 if 标签时，如果前面的 if 没有执行，或导致逗号多余错误。
set 标签动态前置 SET 关键字，同时消除无关的逗号。
用 if + set 标签后，如果某项为 null 则不进行更新，而是保持数据库原值。

foreach 的主要用在构建 in 条件中，迭代一个集合。
item：遍历的对象。
index：迭代过程中，每次迭代到的位置。
open：开始遍历时拼接的sql串。
close：结束遍历时拼接的sql串。
separator：遍历的两个对象中需要拼接的sql串。
collection 输入对象中的集合属性：

  • 传入的是单参数且是 List。
  • 传入的是单参数且是 array 数组。
  • 传入多参数，需要封装成 Map。

SQL片段

动态 SQL 中若出现大于号（>）、小于号（<）、大于等于号（>=），小于等于号（<=），最好转换为实体符号。
否则可能会出现解析出错问题，特别是小于号（<），在 XML 中是绝对不能出现的。
