[笔记]-Mybatis-缓存&分页

2017-12-04

12-29重新整理更新Mybatis笔记

  • 一级缓存：作用域为Session，默认开启，session.flush()或session.close()后，该Session所有Cache清空。

  • 二级缓存：作用域为 Mapper(Namespace)，使用二级缓存时，实体类必须实现Serializable接口。

  • 从缓存读取数据的依据是id。

  • 增删改，无论是否提交，均会清空一级、二级查询缓存。

开启二级缓存

<!-- mybatis-config.xml -->
<settings>
    <setting name="cacheEnabled" value="true">
<settings/>
<mapper namespace="mmp.xxx.UserMapper">
<!-- 此mapper开启二级缓存 -->
<cache/>
<!-- 回收策略为先进先出，自动刷新时间60s，最多缓存512个引用对象，只读 -->
<!-- <cache eviction="FIFO" flushInterval="60000" size="512" readOnly="true"/> -->

  • select语句将会被缓存。

  • insert、update和delete语句会刷新缓存。

  • commit语句会清空缓存。

全局关闭二级缓存。

<settings>
    <setting name="cacheEnabled" value="false">
<settings/>

局部关闭二级缓存：针对某个<select/>查询，不使用二级缓存。

<select useCache="false"><select/>

毛用：若要使某个增、删或改操作不清空二级缓存，在其<insert/>或<delete/>或<update/>中添加属性 flushCache="false"。

二级缓存使用原则

  • 只能在一个命名空间下使用二级缓存
    • 二级缓存中基于namespace，多个namespace中若均存在对同一个表的操作，可能出现不一致。
  • 在单表上使用二级缓存
    • 如果一个表与其它表有关联关系，可能存在多个namespace对同一数据的操作。
  • 查询多于修改时使用二级缓存
    • 二级缓存的频繁刷新将降低性能。

二级缓存

  • 内置
  • 三方框架，如ehcache，实体类无需实现序列化接口。

classpath:ehcache.xml

<ehcache xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="../config/ehcache.xsd">
    <diskStore path=""/>
    <defaultCache 
        maxElementsInMemory="1000" 
        maxElementsOnDisk="10000000"
        eternal="false" 
        overflowToDisk="false" 
        timeToIdleSeconds="120"
        timeToLiveSeconds="120" 
        diskExpiryThreadIntervalSeconds="120"
        memoryStoreEvictionPolicy="LRU">
    </defaultCache>
</ehcache>

然后在XxxMapper.xml中添加ehcache的配置

<!-- 开启本mapper的namespace下的二级缓存，type：cache接口的实现类的类型 -->
<cache type="org.mybatis.caches.ehcache.EhcacheCache"/>

分页(pagination)

  • 逻辑分页：查询得到表中的1000条数据，然后分页显示。

  • 物理分页：先判断该选出这1000条的第几条到第几条，然后数据库查询出需要的100条返回。

  • 对于物理分页，Mysql利用limit、offset、pagesize实现，Oracle通过rownum来实现。

Mysql实现Mybatis物理分页，最简单方式：

<!-- parameterType是传入的参数类，或者map，包含了offset pagesize -->
<select id="listCategory" parameterType="Your_params" resultMap="Category">
    select * from category_
    <if test="offset != null and pagesize != null">
        limit #{offset}, #{pagesize}
    </if>
</select>

更通用的方式是插件

<!-- PageHelper 分页插件 -->
<plugins>
    <plugin interceptor="com.github.pagehelper.PageInterceptor"/>
</plugins>

注释掉limit，因为分页相关工作，由PageHelper做。

查询

PageHelper.offsetPage(0, 5); // 查出第一页的数据，每页5条

List<Category> cs = session.selectList("listCategory");

for (Category c : cs) {
    System.out.println(c.getName());
}
// 获取分页信息，包括总数以及其他的
PageInfo pageInfo = new PageInfo<>(cs);
System.out.println("总数： " + pageInfo.getTotal());
System.out.println(pageInfo);
