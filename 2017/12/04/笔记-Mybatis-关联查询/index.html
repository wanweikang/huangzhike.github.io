嵌套关联查询：先得到一个结果集，然后根据这个结果集的每一条记录进行关联查询。

假设嵌套查询就一个（resultMap内就一个association），查询1次，查询的结果集返回N条，关联查询语句会被执行N次，共需要访问数据库N+1次。

如果N比较大的话，数据库访问非常大。

可以延迟加载：<association>和<collection>。

<settings>
    <!-- 延迟加载 -->

    <setting name="aggressiveLazyLoading" value="false"/>

延迟加载：关联查询时，推迟对关联对象的查询。

关联对象的查询与主加载对象的查询必须是分别进行的select语句，不能是多表连接查询。

多表连接查询，其实是对多个表连接后形成的一张表的查询。

如果不调用class.getTeacher()，就不会去查询Teacher。

也就是把Class和Teacher的查询当做两步，然而不太合适Blog，因为看Blog同时都看Author，查询数据库两次。

嵌套结果查询一次查出结果。

自关联：自己既是一方，又是多方。

一个栏目，可以是一方，即父栏目，也可以是多方，即子栏目。表中有一个外键，表示该栏目的父栏目。

一对多：在页面上点击父栏目，显示出其子栏目。

查指定栏目的所有子孙栏目

public class NewsLabel {
    private Integer id;
    private Set<NewsLabel> children;
    // ...
}
<resultMap type="xxx.NewsLabel" id="NewsLabelMap">
    <!-- 即使字段名与属性名相同，也要写出映射关系 -->
    <id property="id" column="id"/>
    <!-- 第二步，根据第一次查询的结果中的id传给column="id"，递归查询所有下级栏目 -->
    <collection property="children" ofType="xxx.NewsLabel" select="selectChildren" column="id"/>
</resultMap>
<!-- 第一步 -->
<select id="selectChildren" resultMap="NewsLabelMap">
    <!-- 第一次pid动态参数来自调用方法传递的实参，递归后来自上一次的查询结果id -->
    select id from newslabel where pid = #{id}
</select>
