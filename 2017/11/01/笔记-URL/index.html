[笔记]-URL

2017-11-01

12-31重新整理

  • URI：标识某个资源
  • URL：标识互联网上某个资源

相对路径必须结合参照路径，有两种：以斜杠/开头的相对路径，不以斜杠/开头的相对路径。

  • 以斜杠/开头的相对路径分为：前台路径与后台路径。

    • 前台路径：浏览器解析执行的代码的路径，参照路径是服务器的根路径，如http://localhost:4000/。

      • html、css、js中的路径：<img src=""/>、<a href=""></a>、<form action=""></form>、background:img("")、window.location.href=""等。

    • 后台路径：服务器解析执行的代码的路径，参照路径是应用的根路径，如http://localhost:4000/primary/。

      • Java代码、jsp动态部分（Java代码块）、xml文件中的路径（xml文件被Java代码解析）等。

      • 特例：response.sendRedirect("");方法重定向时，是给浏览器用的，参照路径是服务器的根路径。

        • 若要使用重定向，要么写为无斜杠的相对路径：response.sendRedirect("show.jsp");，要么加上项目名称：response.sendRedirect(request.getContextPath + "/show.jsp");

  • 不以斜杠/开头的相对路径：无论是前台还是后台路径，其参照路径都是当前资源的访问路径，而不是保存路径。

/代表应用的根路径

this.getServletContext().getResourceAsStream("/form1.html");
this.getServletContext().getRealPath("/download/1.JPG"); // 获取服务器上的资源
// 客户端请求web资源，服务器跳转到另一个web资源，forward给服务器
request.getRequestDispatcher("/form1.html");
this.getServletContext().getRequestDispatcher("/index.jsp").forward(request, response);
<%@include file="/jspfragments/head.jspf" %>

/代表服务器的根路径

// 请求重定向，服务器发送URL给浏览器，浏览器拿到URL后，再请求服务器
response.sendRedirect("/JavaWeb/index.jsp");
// 灵活写法
response.sendRedirect(request.getContextPath() + "/index.jsp");
<!-- 超链接跳转页面 -->
<a href="/JavaWeb/index.jsp">跳转到首页</a>
<!-- 避免路径中出现项目名称 -->
<a href="${pageContext.request.contextPath}/index.jsp">跳转到首页</a>
<!-- 客户端使用超链接跳转，"/"给浏览器用，"/"代表服务器根目录 -->

<!-- form表单提交 -->
<form action="${pageContext.request.contextPath}/servlet/CheckServlet" method="post"></form>
<!-- 浏览器将form表单提交到服务器，"/"是给浏览器用，"/"代表服务器根目录 -->
<form action="/JavaWeb/servlet/CheckServlet" method="post"></form>

<!-- 绝对路径引用js脚本 -->
<script src="${pageContext.request.contextPath}/js/index.js"></script>
<!-- ${pageContext.request.contextPath}与request.getContextPath()效果一样 -->
<script src="<%= request.getContextPath() %>/js/login.js"></script>
<!-- 绝对路径的方式引用css样式 -->
<link rel="stylesheet" href="${pageContext.request.contextPath}/css/index.css" type="text/css"/>

如果/是给服务器用的，代表应用，如：http://localhost:4000/primary/。

如果/是给浏览器用的，代表服务器根目录，如http://localhost:4000/。
