通过扩展 JdbcDaoSupport，设置数据源，并且 JdbcTemplate 不再是必需的，只需要正确的数据源注入JdbcCustomerDAO。

使用 JDBC 模板，需要配置好数据源，默认的数据源 DriverManagerDataSource，DriverClassName 属性用于接收 DB 驱动。

为了便于维护，可以将数据库连接信息写入到属性文件中，一般都是放在 src 下。
从属性文件中读取数据时，需要在<property/>的 value 属性中使用${ }，
将在属性文件中定义的 key 括起来，以引用指定属性的值。
属性文件若要被 Spring 配置文件读取，必须在配置文件中注册。：

  • <bean/> 方式
    以 PropertyPlaceholderConfigurer 类的 bean 实例注册。location属性指定属性文件的位置。不常用。
  • <context> 方式
    在 Spring 配置文件头部加入 context 的约束，即修改配置文件头。
    <context:property-placeholder/>标签中有一个属性 location，指定属性文件的位置。

查询单个列名作为字符串。
在 JdbcTemplate query() 方法，需要手动转换返回的结果为目标对象类型，并传递一个对象数组作为参数。
在 SimpleJdbcTemplate类，它更简单。

查询行的总数。

简单对象查询方法有：

  • 查询结果为单个对象：pubic T queryForObject(String sql, Class<T> type, Object... args)
  • 查询结果为 List：pubic List<T> queryForList(String sql, Class<T> type, Object... args)

自定义对象查询方法有：

  • 查询结果为单个对象：pubic T queryForObject(String sql, RowMapper<T> m, Object... args)
  • 查询结果为 List：pubic List<T> query(String sql, RowMapper<T> m, Object... args)

RowMapper 接口将查询结果集中每一条记录包装为指定对象，该接口中有一个方法需要实现：
public Object mapRow(ResultSet rs, int rowNum)
参数 rowNum 表示总的结果集中当前行的行号，但参数 rs 并不表示总的结果集，而是表示 rowNum 所代表的当前行的记录所定义的结果集，仅仅是当前行的结果。
该方法体中一般就是实现将查询结果中当前行的数据包装为指定对象。

查询单行数据：两种方法

一. 自定义RowMapper

它传递给 queryForObject()方法，返回的结果将调用自定义 mapRow()方法的值匹配到属性。

二. BeanPropertyRowMapper

有一个方便 RowMapper 实现，BeanPropertyRowMapper，通过匹配行的名字的列值映射到一个属性。
只要确保这两个属性和列具有相同的名称，如属性“CUSTID’将匹配到列名为”CUSTID’或“CUST_ID”。
在SimpleJdbcTemplate类，需要使用“ParameterizedBeanPropertyRowMapper”代替“BeanPropertyRowMapper”。
