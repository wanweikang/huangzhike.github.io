实现初始化方法和销毁方法3种方式

  • 实现标识接口InitializingBean，DisposableBean

  • 设置bean属性Init-method，destroy-method

  • 使用注释配置后，调用@PostConstruct，@PreDestroy注解

InitializingBean和DisposableBean是两个标记接口，为Spring执行时bean的初始化和销毁某些行为时的有用方法。

package mmp.services;

public class CustomerService implements InitializingBean, DisposableBean {

    String message;

    // getter setter

    // Bean实现InitializingBean，运行afterPropertiesSet()，在所有bean属性被设置后
    public void afterPropertiesSet() throws Exception {
        // message属性设置后调用
        System.out.println("Init method after properties are set : " + message);
    }

    // Bean实现DisposableBean，运行destroy()，在Spring容器释放该bean后
    public void destroy() throws Exception {
        // 调用 applicationContext.close()之后
        System.out.println("Spring Container is destroy! Customer clean up");
    }

}
<bean id="customerService" class="mmp.services.CustomerService">
    <property name="message" value="msg"/>
</bean>

使用init-method和destroy-method在bean配置文件属性，替代InitializingBean和DisposableBean接口，减少耦合。

package mmp.services;

public class CustomerService {

    String message;

    // getter setter

    public void initIt() throws Exception {
        System.out.println("Init method after properties are set : " + message);
    }

    public void cleanUp() throws Exception {
        System.out.println("Spring Container is destroy! Customer clean up");
    }

}
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans-3.0.xsd"
    default-init-method="init" default-destroy-method="destroy">
    <!-- 如果有太多同名初始化或销毁方法的Bean，不需要在每个bean上声明，用上面的就行了 -->

    <bean id="customerService" class="mmp.services.CustomerService" 
        init-method="initIt" destroy-method="cleanUp">
        <property name="message" value="msg"/>
    </bean>
</beans>

使用@PostConstruct和@PreDestroy注解。该注解不属于Spring，它在J2EE库-common-annotations.jar。

package mmp.services;

public class CustomerService {

    String message;

    // getter setter

    @PostConstruct
    public void initIt() throws Exception {
        System.out.println("Init method after properties are set : " + message);
    }

    @PreDestroy
    public void cleanUp() throws Exception {
        System.out.println("Spring Container is destroy! Customer clean up");
    }

}

Bean后处理器是特殊的Bean，容器中所有Bean在初始化时，均会自动执行该类的两个方法，该Bean自动执行，无须id属性。

Bean初始化完毕有一个标志：某个方法将被执行。即当该方法被执行时，表示该Bean被初始化完毕。

Bean后处理器中两个方法，在这个方法之前之后执行。

package mmp;

public class HelloWorld {
    private String message;
    // getter setter
    public void init() {}
    public void destroy() {}
}

package mmp;

public class Init implements BeanPostProcessor {
    // 目标Bean初始化完前调用
    // 第一个参数：将初始化的Bean实例
    // 第二个参数：该Bean实例的id，没有id就是name
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("BeforeInitialization : " + beanName);
        // 返回增过的Bean对象
        return bean;
    }

    // 目标Bean初始化完后调用
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("AfterInitialization : " + beanName);
        return bean;
    }
}

  • 调用无参构造器，创建实例对象。

  • 调用参数的setter，为属性注入值。

  • 若Bean实现了BeanNameAware接口，执行接口方法setBeanName(String beanId)，使Bean类可以获取其在容器中的id名称。

  • 若Bean实现了BeanFactoryAware接口，则执行接口方法setBeanFactory(BeanFactoryfactory)，使Bean类可以获取BeanFactory对象。

  • 若定义并注册了Bean后处理器BeanPostProcessor，则执行接口方法postProcessBeforeInitialization()。

  • 若Bean实现了InitializingBean接口，则执行接口方法afterPropertiesSet()。

    • 在Bean的所有属性的set方法执行完毕后执行，是Bean初始化结束的标志，Bean实例化结束。

  • 若设置了init-method方法，则执行。

  • 若定义并注册了Bean后处理器BeanPostProcessor，则执行接口方法postProcessAfterInitialization()。

  • 执行业务方法。

  • 若Bean实现了DisposableBean接口，则执行接口方法destroy()。

  • 若设置了destroy-method方法，则执行。
