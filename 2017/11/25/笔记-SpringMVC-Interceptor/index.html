[笔记]-SpringMVC-Interceptor

2017-11-25 10:24

Interceptor拦截器拦截指定的用户请求，并进行相应的预处理与后处理。

当有多个拦截器时，形成拦截器链。执行顺序与其注册顺序一致。
当某一个拦截器的 preHandle()方法返回 true 并被执行时，向一个方法栈中放入该拦截器的 afterCompletion()方法。
只要有一个 preHandle()方法返回 false，则上部的执行链将被断开，
其后续的处理器方法与 postHandle()方法将无法执行。
无论执行链执行情况怎样，只要方法栈中有方法，即执行链中只要有 preHandle()方法返回 true，就会执行方法栈中 afterCompletion()方法。

注册

一、拦截所有URL

二、拦截匹配的URL

三、HandlerMapping上的拦截器

如果使用了<mvc:annotation-driven/>，会自动注册DefaultAnnotationHandlerMapping 与 AnnotationMethodHandlerAdapter 两个bean，没有机会再给它注入interceptors属性，无法指定拦截器。
当然可以人工配置上面的两个Bean，不使用 <mvc:annotation-driven/>，就可以给interceptors属性注入拦截器了。
