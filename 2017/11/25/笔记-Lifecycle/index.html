实现初始化方法和销毁方法3种方式

  • 实现标识接口 InitializingBean，DisposableBean
  • 设置bean属性 Init-method destroy-method
  • 使用注释配置后，调用@PostConstruct和@PreDestroy注解

InitializingBean和DisposableBean是两个标记接口，为Spring执行时bean的初始化和销毁某些行为时的有用方法。

使用 init-method 和 destroy-method 在bean 配置文件属性，替代 InitializingBean和DisposableBean接口。

initIt()方法被调用，消息属性设置后，在 context.close()调用后，执行 cleanUp()方法
建议在Bena配置文件使用init-method 和 destroy-methodbean ，而不是执行 InitializingBean 和 DisposableBean 接口，造成不必要的耦合代码。

如果有太多同名初始化或销毁方法的Bean，不需要在每个bean上声明。
用default-init-method和default-destroy-method

使用 @PostConstruct 和 @PreDestroy 注解来做同样的事情。
@PostConstruct和@PreDestroy 标注不属于 Spring，它是在J2EE库- common-annotations.jar。

启用@PostConstruct和@PreDestroy注解，要么注册“CommonAnnotationBeanPostProcessor”，要么在bean配置文件的<context:annotation-config/>指定

CommonAnnotationBeanPostProcessor

<context:annotation-config/>

Step1：调用无参构造器，创建实例对象。
Step2：调用参数的setter，为属性注入值。
Step3：若Bean实现了BeanNameAware接口，执行接口方法setBeanName(String beanId)，使Bean类可以获取其在容器中的id名称。
Step4：若Bean实现了BeanFactoryAware接口，则执行接口方法setBeanFactory(BeanFactoryfactory)，使Bean类可以获取到BeanFactory对象。
Step5：若定义并注册了Bean后处理器BeanPostProcessor，则执行接口方法postProcessBeforeInitialization()。
Step6：若Bean实现了InitializingBean 接口，则执行接口方法afterPropertiesSet ()。在Bean的所有属性的set方法执行完毕后执行，是Bean初始化结束的标志，Bean实例化结束。
Step7：若设置了init-method方法，则执行。
Step8：若定义并注册了Bean后处理器BeanPostProcessor，则执行接口方法postProcessAfterInitialization()。
Step9：执行业务方法。
Step10：若Bean实现了DisposableBean接口，则执行接口方法destroy()。
Step11：若设置了destroy-method方法，则执行。

Bean后处理器是一种特殊的Bean，容器中所有Bean在初始化时，均会自动执行该类的两个方法。
该Bean自动执行，故无须id属性。
Bean 初始化完毕有一个标志：一个方法将被执行。
即当该方法被执行时，表示该 Bean被初始化完毕。
Bean 后处理器中两个方法的执行，是在这个方法之前之后执行。
