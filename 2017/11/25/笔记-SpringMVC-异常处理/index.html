2017-11-25

12-27重新整理更新Spring笔记

SpringMVC常用异常处理有三种：

  • SimpleMappingExceptionResolver
  • 自定义异常处理器
  • 异常处理注解

SimpleMappingExceptionResolver可以将不同的异常映射到不同的jsp页面（通过exceptionMappings）。

<!-- 只需注册该异常处理器Bean即可 -->
<bean id="exceptionResolver" class="org.springframework.web.servlet.handler.SimpleMappingExceptionResolver">
    <!-- 指定默认的异常响应页面，没有在exceptionMappings里面找到时返回error的视图--> 
    <property name="defaultErrorView">
        <value>/error</value>
    </property>
    <!-- 发生异常时默认的HttpServletResponse的返回码-->
    <property name="defaultStatusCode" value="404"/>
    <!-- 发生异常时视图跟返回码的对应关系 -->
    <property name="statusCodes">
        <props>
            <!-- number视图对应的HttpServletResponse的返回码是500 -->
            <prop key="number">500</prop>
            <prop key="null">503</prop>
        </props>
    </property>
    <property name="exceptionMappings">
        <props>
            <!-- 抛出NumberFormatException时就返回number视图-->
            <prop key="NumberFormatException">number</prop>
        <prop key="mmp.SpringException">ExceptionPage</prop>
        </props>
    </property>
    <!-- 捕获到的异常对象，一般异常响应页面中使用 -->
    <property name="exceptionAttribute" value="ex"/>
</bean>

@Controller

SimpleMappingExceptionResolver异常处理器可以实现发生指定异常后的跳转，完成不了捕获到指定异常时，执行一些操作。

自定义异常处理器，需要实现HandlerExceptionResolver接口。

系统遇到异常，手动throws Exception向上抛出，dao抛给service、service给controller、controller抛给前端控制器，前端控制器调用全局异常处理器。

全局异常处理器（一个系统只有一个异常处理器）进行统一异常处理。

  • 如果该异常类型是系统自定义的异常，直接取出异常信息，在错误页面展示
  • 如果该异常类型不是系统自定义的异常，构造一个自定义的异常类型（信息为“未知错误”）
package mmp.exception;

public class CustomExceptionResolver implements HandlerExceptionResolver {
    @Override
    public ModelAndView resolveException(HttpServletRequest request, HttpServletResponse response, 
            Object handler, Exception ex) {

        CustomException customException;
        // 如果是程序中手动抛出的异常，在错误页面中显示自定义的异常信息
        if (ex instanceof CustomException) {
            customException = (CustomException) ex;
        } else {
            // 如果不是手动抛出异常说明是一个运行时异常，在错误页面只显示未知错误
            customException = new CustomException("未知错误");
        }

        String message = customException.getMessage();

        ModelAndView modelAndView = new ModelAndView();
        modelAndView.addObject("message", message);
        modelAndView.setViewName("error");

        return modelAndView;
    }
}
<!-- 只要实现HandlerExceptionResolver接口就是全局异常处理器 -->
<!-- 全局异常处理器只有一个，配置多个也没用 -->
<bean class="mmp.exception.CustomExceptionResolver"/>
  • 在Controller方法中抛出异常
public String editItems(Model model, @RequestParam(value = "id", required = true) Integer items_id) 
        throws Exception {

    // 调用service根据商品id查询商品信息
    ItemsCustom itemsCustom = itemsService.findItemsById(items_id);

    // 没有查询到商品，抛出异常
    if (itemsCustom == null) {
        throw new CustomException("修改的商品信息不存在!");
    }

    // 相当于modelAndView.addObject方法
    model.addAttribute("items", itemsCustom);

    return "items/editItems";
}
  • 在Service接口中抛出异常
public ItemsCustom findItemsById(Integer id) throws Exception {
    Items items = itemsMapper.selectByPrimaryKey(id);
    if (items == null) {
        throw new CustomException("修改的商品信息不存在!");
    }
    // 中间对商品信息进行业务处理
    ItemsCustom itemsCustom = null;
    // 将items的属性值拷贝到itemsCustom
    if (items != null) {
        itemsCustom = new ItemsCustom();
        BeanUtils.copyProperties(items, itemsCustom);
    }

    return itemsCustom;
}

@ExceptionHandler异常处理注解将一个方法指定为异常处理方法。
该注解value属性为一个Class<?>数组，所要匹配的异常。
一般将异常处理方法专门定义在一个Controller中，让其它Controller继承。

public class BaseController { 
    @ExceptionHandler
    public String exp(HttpServletRequest request, Exception ex) { 
        // 异常处理
    }
}
