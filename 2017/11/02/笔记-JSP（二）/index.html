[笔记]-JSP（二）

2017-11-02 14:26

JSP：Java Server Pages

  • JSP表达式：<%= %>，
    • 变量或表达式
    • 无分号，用于输出数据
  • JSP脚本片段:<% %>
    • 翻译到servlet的service方法中
    • 可以定义变量，不能添加访问修饰符
    • 可以编写语句，不能定义方法，不能定义静态语句块
    • 编写多行Java代码,多个脚本片段可以相互访问,单个片段可不完整
    • 只能出现Java代码，不能出现模板元素
  • JSP声明：<%! %>
    • 翻译到_jspService方法外
    • 定义JSP页面转换成的Servlet程序的静态代码块、成员变量和方法
    • JSP隐式对象的作用范围仅限于Servlet的_jspService方法，在JSP声明中不能使用这些隐式对象
  • JSP注释:<%-- 注释 --%>，浏览器看不到注释。
  • HTML注释：<!-- 注释 -->，浏览器看得到注释。
  • JSP指令：不直接产生输出，只是说明如何处理JSP页面。
    • <%@ page %>指令
    • <%@ include %>指令
    • <%@ taglib %>指令

errorPage属性的设置值必须使用相对路径，“/”开头表示相对于当前Web应用的根目录(不是站点根目录)，否则表示相对于当前页面。
如果设置了JSP页面的errorPage属性，在web.xml设置的错误处理将不对该页面起作用。

将error.jsp页面显式声明为错误处理页面后，可以在error.jsp页面中使用exception对象。

<%@ include file="relativeURL"%>，路径以/开头，代表当前web应用。

  • 注意：
    • 被引入的文件必须遵循JSP语法。
    • 被引入的文件可以使用任意的扩展名，也会按照JSP处理，建议使用.jspf（JSP fragments）。
    • include指令会把2个JSP翻译成一个servlet，这2个JSP页面的指令不能冲突（pageEncoding和导包除外）。

<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

  • JSP标签
    • <jsp:forward />
    • <jsp:include />
    • <jsp:param />

<jsp:include>标签把另外一个资源的输出内容插入当前JSP的输出内容中，在JSP执行时引入，动态引入。
如果包含的是非*.jsp文件，则只是静态包含进来。

<jsp:include>标签是动态引入，涉及的JSP被翻译成不同的servlet，在执行时合并，不同JSP定义了同名变量正常
<%@ include %>指令是静态引入，涉及的JSP被翻译成一个servlet，在源文件合并，不同JSP定义了同名变量异常

JSP规范建议用.jspf（JSP fragments）作为静态引入文件的扩展名。
把一个.jsp命名为.jspf，只有用<@ include >指令时，jspf文件的内容才会被解析并执行其中的JSP指令和tag，而使用<jsp:include />和JSTL的<c:import />都没有用，jspf文件被当作纯文本文件处理了。

解决办法：

<jsp:forward>标签把请求转发给另外一个资源。只要页面有此标签，当前页面内容全部无法显示。

用<jsp:include>和<jsp:forward>标签引入或将请求转发给其它资源时，使用<jsp:param>向这个资源传递参数。
