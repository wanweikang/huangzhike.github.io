JSP：Java Server Pages

访问JSP时，其实是在访问Servlet，服务器把JSP翻译成Servlet，传递和创建9个对象。out和pageContext为JSP独有。

  • request
  • response
  • session
  • application
  • config
  • page
  • exception
  • out
  • pageContext

out对象：向客户端发送文本数据。

  • pageContext.getOut()方法返回，类型为JspWriter，和ServletResponse.getWriter返回的PrintWriter对象相似。
  • JspWriter相当于自带缓存的PrintWriter。

pageContext对象：JSP页面的运行环境。

  • 封装了对其他8大隐式对象的引用
  • 自身是一个域对象，可用来保存数据

  • 封装了一些常用操作，如：引入和跳转、检索其他域对象中的属性等

  • 四个域:

    • application域：应用程序范围，servletContext，对应常量PageContext.APPLICATION_SCOPE
    • session域：会话范围，session，对应常量PageContext.SESSION_SCOPE
    • resquet域：请求范围，request，对应常量PageContext.REQUEST_SCOPE
    • page域：页面范围，pageContext，对应常量PageContext.PAGE_SCOPE

从一个JSP跳转到另一个JSP，一般用<jsp:forward>标签。

request属性范围在一次服务器跳转中有效，只要是服务器跳转，request属性可以一直传递下去。

但是使用了超链接的方式，无法继续向下传递。

forward跳转是服务器跳转，地址栏不变。超链接是客户端跳转，地址栏改变。

session设置的属性不管如何跳转，都可以取得，新开一个浏览器，则无法取得设置的session对象的属性。

一个属性想让所有用户都可以访问，用application范围，过多的application属性影响到服务器性能。

  • JSP表达式：<%= %>。
    • 变量或表达式
    • 无分号，用于输出数据
  • JSP脚本片段:<% %>。
    • 翻译到servlet的service方法中
    • 可以定义变量，不能添加访问修饰符
    • 可以编写语句，不能定义方法，不能定义静态语句块
    • 编写多行Java代码，多个脚本片段可以相互访问，单个片段可不完整
    • 只能出现Java代码，不能出现模板元素
  • JSP声明：<%! %>。
    • 翻译到_jspService方法外
    • 定义JSP页面转换成的Servlet程序的静态代码块、成员变量和方法
    • JSP隐式对象的作用范围仅限于_jspService方法，在JSP声明中不能使用这些隐式对象
  • JSP注释:<%-- 注释 --%>，浏览器看不到注释。
  • HTML注释：<!-- 注释 -->，浏览器看得到注释。
  • JSP指令：不直接产生输出，只是说明如何处理JSP页面。
    • <%@ page %>指令
    • <%@ include %>指令
    • <%@ taglib %>指令

errorPage属性的设置值必须使用相对路径，/开头表示相对于当前Web应用的根目录(不是站点根目录)，否则表示相对于当前页面。

如果设置了JSP页面的errorPage属性，在web.xml设置的错误处理将不对该页面起作用。

将页面显式声明为错误处理页面后，可以在页面中使用exception对象。

  • JSP标签
    • <jsp:forward/>
    • <jsp:include/>
    • <jsp:param/>

<jsp:include>标签把另外一个资源的输出内容插入当前JSP的输出内容中，在JSP执行时引入，动态引入。

如果包含的是非*.jsp文件，则只是静态包含进来。

<jsp:include>标签是动态引入，涉及的JSP被翻译成不同的servlet，在执行时合并，不同JSP定义了同名变量正常

<%@ include %>指令是静态引入，涉及的JSP被翻译成一个servlet，在源文件合并，不同JSP定义了同名变量异常

建议用.jspf（JSP fragments）作为静态引入文件的扩展名，*.jspf只有用<@ include >指令时，文件的内容才会被解析并执行其中的JSP指令和tag，使用<jsp:include/>和JSTL的<c:import/>则被当作纯文本文件处理了。

解决办法：

<jsp:forward>标签把请求转发。只要页面有此标签，当前页面内容全部无法显示。

用<jsp:include>和<jsp:forward>标签引入或将请求转发给其它资源时，使用<jsp:param>向这个资源传递参数。
