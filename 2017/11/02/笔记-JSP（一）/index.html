[笔记]-JSP（一）

2017-11-02 14:26

访问JSP时，其实是在访问Servlet，服务器把JSP翻译成Servlet。

JSP引擎在调用JSP对应的_jspServlet时，会传递和创建9个对象。out和pageContext为JSP独有。

  • request
  • response
  • session
  • application
  • config
  • page
  • exception
  • out
  • pageContext

out对象：向客户端发送文本数据。

  • pageContext.getOut()方法返回，类型为JspWriter，和ServletResponse.getWriter返回的PrintWriter对象相似。
  • JspWriter相当于自带缓存功能的PrintWriter,设置page指令的buffer属性课调整缓存大小。
  • 满足如下条件之一，out对象才调用ServletResponse.getWriter方法，并通过返回的PrintWriter对象将out对象的缓冲区的内容写入缓冲区中：
    • 设置page指令的buffer属性关闭的out对象的缓存功能
    • out对象的缓冲区已满
    • 整个JSP页面结束

pageContext对象：JSP技术中最重要的一个对象，代表JSP页面的运行环境。

  • 封装了对其他8大隐式对象的引用
  • 自身是一个域对象，可用来保存数据

  • 封装了一些常用操作(提供管理所有域的入口)，如：引入和跳转其他资源、检索其他域对象中的属性等

  • 四个域:

    • application域:应用程序范围,servletContext,对应常量PageContext.APPLICATION_SCOPE
    • session域:会话范围,session,对应常量PageContext.SESSION_SCOPE
    • resquet域:请求范围,request,对应常量PageContext.REQUEST_SCOPE
    • page域:页面范围,pageContext,对应常量PageContext.PAGE_SCOPE

开发中，要想从一个JSP服务器端跳转到另一个JSP页面，一般用<jsp:forward>标签。

request属性范围在一次服务器跳转中有效，只要是服务器跳转，request属性可以一直传递下去。
但是使用了超链接的方式，属性无法向下继续传递。

forward跳转是服务器跳转,地址栏不变。超链接是客户端跳转,地址栏改变。

session设置的属性不管如何跳转，都可以取得。
如果新开了一个浏览器，则无法取得设置的session对象的属性，因为session只是保留了一个人的信息。
如果一个属性想让所有的用户都可以访问，则可以使用application范围。
过多的application属性影响到服务器的性能。
