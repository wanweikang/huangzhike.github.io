[笔记]-SQL

单条示例，不要联系上下文

约束：添加在列上用来约束列。

  • 唯一约束UNIQUE不可重复。

  • 非空约束NOT NULL不能为空。

  • 主键约束PRIMARY KEY，同时具备UNIQUE和NOT NULL约束。

  • 外键约束FOREIGN KEY。

  • INNER JOIN：返回记录当两个表有匹配。

  • LEFT JOIN：返回左表中所有的行，即使右表中没有匹配，用NULL值代替返回。

  • RIGHT JOIN：返回右表中所有的行，即使有在左表中没有匹配。

  • FULL JOIN：从左表和右表返回所有的行，如果没有匹配，用NULL值作为两侧缺失匹配结果。

  • SELF JOIN：用来连接表本身，如果表有两张表，暂时改名至少在一个表中的SQL语句。

  • CARTESIAN JOIN：返回来自两个或更多个联接的表的记录的集合的笛卡尔乘积。

INNER JOIN与JOIN是相同的。

LEFT JOIN称为LEFT OUTER JOIN。

RIGHT JOIN称为RIGHT OUTER JOIN。

FULL JOIN称为FULL OUTER JOIN。

自连接被用来联接表本身以作为两个表，暂时重命名，在SQL语句中至少有一个表。

MySQL不支持FULL JOIN，可以使用UNION ALL子句。

UNION操作符用于合并两个或多个SELECT语句的结果集，不返回任何重复的行。

UNION内部的SELECT语句必须拥有相同数量的列，相同的数据类型，相同的顺序。

UNION选取不同的值。

如果允许重复的值，请使用UNION ALL，用于合并两个SELECT语句，包括重复行的结果。

UNION结果集中的列名总是等于UNION中第一个SELECT语句中的列名。

SELECT INTO用于创建表的备份复件，从一个表中选取数据，然后把数据插入另一个表中。

GROUP BY语句用来与聚合函数(COUNT, SUM, AVG, MIN, or MAX)联合使用来得到一个或多个列的结果集。

SELECT id, COUNT(course) AS numcourse FROM student GROUP BY id;。

聚合函数作用于多行数据而只返回一个单个值，因此，在SELECT中出现的元素要么为聚合函数的输入值，要么为GROUP BY语句的参数，否则会出错：

SELECT id, COUNT(course) AS numcourse, score FROM student GROUP BY id;。

因为一个学生id对应多个分数，如果在SELECT语句中写上score，则无法判断应该输出哪一个分数。

SELECT id, COUNT(course) AS numcourse, AVG(score) AS avgscore FROM student GROUP BY id;。

HAVING语句通常与GROUP BY语句联合使用，用来过滤由GROUP BY语句返回的记录集。

WHERE关键字不能与聚合函数联合使用。

SELECT id, COUNT(course) AS numcourse, AVG(score) AS avgscore FROM student GROUP BY id HAVING AVG(score) >= 80;。

查找重复记录：

SELECT user_name, count(*) AS count FROM user_table GROUP BY user_name HAVING count > 1;。

求出每个省有多少人口和面积：

SELECT province, SUM(area) AS 't_area', SUM(population) AS 't_population' FROM city_info GROUP BY province;。

求出面积大于100和人口大于200的省份：

SELECT province FROM city_info GROUP BY province HAVING SUM(area) > 10 AND SUM(population) > 100;。

DELETE FROM：属于DML，可回滚。

TRUNCATE TABLE：属于DDL，不可回滚，保留表。

DROP TABLE：属于DDL，不可回滚，不保留表。

  • 分区

只支持水平分区，当表中存在主键或唯一索引时，分区列必须是唯一索引的组成部分，否则不能创建分区。

支持Range、List、Hash、Key、Columns。

对于查询大表的大量数据，分区是有用的，因为可能只需要查询对应的分区就好，但是对于通过索引获取几条记录的查询，B树只需要几次IO，已经足够，分区可能还会增加复杂度和IO（如遍历分区）。
