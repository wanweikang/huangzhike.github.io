package mmp.servlet.study;

public class ServletMdzz extends HttpServlet {
    // 容器在创建Servlet实例时，自动将初始化参数封装到ServletConfig对象中
    // 在调用servlet的init方法时，将ServletConfig对象传给servlet
    // 通过ServletConfig对象就可得到当前servlet的初始化参数
    private ServletConfig config;

    @Override
    public void init(ServletConfig config) throws ServletException {
        // 重写了init(ServletConfig config)方法，要调用父类的init方法，处理获取ServletContext对象的引用
        // 否则this.getServletContext();出现空指针异常
        super.init(config);
        this.config = config;
    }

    public void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {

        response.setContentType("text/html;charset=utf-8");
        // 获取指定的初始化参数
        String paramVal = this.config.getInitParameter("name");
        response.getWriter().print(paramVal);
        response.getWriter().print("<hr/>");
        // 获取Servlet所有初始化参数
        Enumeration<String> e = config.getInitParameterNames();
        while (e.hasMoreElements()) {
            String name = e.nextElement();
            String value = config.getInitParameter(name);
            response.getWriter().print(name + "=" + value + "<br/>");
        }
        ServletContext context = this.getServletContext();
        // 获取Web应用的初始化参数
        String contextInitParam = context.getInitParameter("url");
        response.getWriter().print(contextInitParam);
    }

    public void doPost(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        this.doGet(request, response);
    }
}

Servlet的整个生命周期内，Servlet的init()方法只调用一次。

多次请求Servlet，只创建一个Servlet实例，Web容器退出，实例才销毁。

每次请求Servlet都调用一次service()方法。

每次请求Servlet都创建一个新的HttpServletRequest对象和HttpServletResponse对象，然后作为参数传递给service()方法，service()方法再根据请求方式分别调用doXxx()方法。

WEB容器在启动时，为每个WEB应用程序创建一个对应的ServletContext对象，代表当前Web应用，application。

一个WEB应用的所有Servlet共享一个ServletContext对象，Servlet间可通过ServletContext实现通讯。

String data = "xxx";
ServletContext context = this.getServletConfig().getServletContext();
context.setAttribute("data", data); // 将data存储到ServletContext对象中
ServletContext context = this.getServletContext();
String data = (String) context.getAttribute("data");

JSP和Servlet都是单例多线程的，实例变量会引起问题。

并发访问同一个Servlet时，每个请求一个线程，并在这个线程上调用Servlet的service方法，service方法如果访问了同一个资源的话，可能引发线程安全问题。

不存在线程安全问题：

public void doGet(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    // i在方法内定义，局部变量，每个线程都有自己的i变量
    int i = 1;
    i++;
    response.getWriter().write(i + "");
}

存在线程安全问题：

// i定义成全局变量，多线程并发访问变量i时，存在线程安全问题
int i = 1;

public void doGet(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {
    i++;
    try {
        Thread.sleep(1000 * 4);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    response.getWriter().write(i + "");
}

int i = 1;

public void doGet(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    // Servlet不能用这种方式，假如有9999个人同时访问，后面的线程只能排队等候
    synchronized (this) { // 每个对象都有把锁，这里this指Servlet对象
        i++;
        try {
            Thread.sleep(1000 * 4);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        response.getWriter().write(i + "");
    }
}

实现了SingleThreadModel标记接口（Deprecated），将以单线程调用其service方法。

并不能真正解决问题，因为会创建多个Servlet实例对象。

标记接口：接口中没有定义任何方法和常量。

  • 实现了Cloneable接口，对象就可以被克隆。
  • 实现了Serializable接口，对象就可以被序列化。

基于Servlet生命周期的页面点击计数器：

  • init()方法中初始化一个全局变量。

  • 每次调用doGet()或doPost()方法时，增加全局变量。

  • 可以使用数据库表存储destroy()方法中的全局变量，下次初始化时，可以在init()方法读取。

  • 计算一个会话内一个页面的点量击，可以使用isNew()方法查看该会话内是否已点击过。

    public class PageHitCounter extends HttpServlet {
  private int hitCount;

  public void init() {
      // Reset hit counter.
      hitCount = 0;
  }

  public void doGet(HttpServletRequest request, HttpServletResponse response) 
          throws ServletException, IOException {
      hitCount++;
  }
  public void destroy() {}
}

基于过滤器生命周期的页面点击计数器：

  • init()方法中初始化一个全局变量。
  • 每次调用doFilter方法时，增加全局变量。
  • 可以使用数据库表存储过滤器的destroy()方法中的全局变量的值，下次初始化时，可以在init()方法读取。
public class SiteHitCounter implements Filter {
    private int hitCount;

    public void init(FilterConfig config) throws ServletException {
        // Reset hit counter.
        hitCount = 0;
    }

    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws java.io.IOException, ServletException {
        hitCount++;
        chain.doFilter(request, response);
    }
    public void destroy() {}
}
