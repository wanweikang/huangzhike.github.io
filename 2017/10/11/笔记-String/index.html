[笔记]-String

2017-10-11 19:45
  • null 表示 String 还没有 new ，对象的引用还没有创建，也没有分配内存空间。
  • “”、new String() 说明已经 new ，创建了对象的引用，分配内存空间。
  • 基本类型，==直接比较值，引用类型，==还比较地址。
  • Object类equals方法与==相同，但是String类重写了equals方法。
  • 所有对象都从 Object 类继承了 equals() 方法， 默认实现是 == 测试引用相同， 不测试内容，可自定义 equals() 方法。

超类 Object 中 equals() 方法：

  • “==”比较两个对象的的内存地址，若 object1.equals(object2) 为 true，equals1 和 equals2 实际上是引用同一个对象。

String、Math 等封装类都重写了 equals() 方法。

  • 检查空串使用if (str.length()==0)或if(str.equals(“”))
  • 对象域，使用 equals 方法 。
  • 类型安全的枚举，使用 equals 或== 。
  • 可能为 null 的对象域 : 使用 == 和 equals 。
  • 数组域 : 使用 Arrays.equals 。
  • 除 float 和 double 外的原始数据类型 : 使用 == 。
  • float 类型: 使用 Float.foatToIntBits 转换成 int 类型，然后使用==。
  • double 类型: 使用 Double.doubleToLongBit 转换成 long 类型，然后使用==。

在覆写 equals() 方法时，推荐使用 getClass 来进行类型判断，不是使用 instanceof。
instanceof 的作用是判断其左边对象是否为其右边类的实例，返回 boolean 类型的数据。

String 类为 final 型（不可被继承），每个修改 String 对象的操作，都创建了一个新的 String 对象。
创建字符串对象时，先检查字符串池中是否存在值相等的，有则不再创建，直接返回该对象的引用，没有则创建然后放入到字符串池中并返回引用。
直接赋值（String s=”a”），（String s = new String(“a”)）新建了一个 String 对象。
String 类不可改变，修改字符串，使用 StringBuffer 和 StringBuilder 。
StringBuffer （线程不安全，能同步访问，速度快）和 StringBuilder 类的对象能被多次修改，不产生新对象。
StringBuffer 不能用=初始化，必须产生 StringBuffer 实例，通过构造方法初始化。
不是所有的 String 字符串操作都比 StringBuffer 慢。
