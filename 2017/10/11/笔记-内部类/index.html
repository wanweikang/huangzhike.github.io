[笔记]-内部类

2017-10-11

类、接口和枚举类型都定义为顶层类型。类型还可以嵌套在其他类型中定义，包括四种。

  • 静态成员：定义为其他类型静态成员的类型。

    • 嵌套的接口、枚举和注解始终都是静态成员类型（就算不用static）。
    • 接口或注解中的嵌套类型也都隐式声明为静态类型。
    • 静态成员类型只能在顶层类型和其他静态成员类型中定义，不能在任何成员类、局部类或匿名类中定义。
    • 接口和注解的所有成员都隐式声明为public，嵌套在接口或注解中的静态成员类型不能声明为protected或private。
    • 静态成员类型和所在类的任何实例都不关联（即没有this对象）。
    • 静态成员类型只能访问所在类的静态成员。
    • 静态成员类型不能和任何一个外层类同名。

  • 非静态成员：没使用static声明的成员类型。只有类才能作为非静态成员类型。

    • 如果把静态成员类型比作类字段或类方法，那么非静态成员类可以比作实例字段或方法。
    • 只有类才能作为非静态成员类型。
    • 一个非静态成员类的实例始终关联一个外层类型的实例。
    • 非静态成员类的代码能访问外层类型的所有字段和方法（静态和非静态的都能访问）。

  • 局部类：代码块中定义的类，只在这个块中可见。接口、枚举和注解不能定义为局部类型。

    • 局部类往往在方法中定义，但也可以在类的静态初始化程序或实例初始化程序中定义。
    • 和局部变量一样，局部类只在定义它的块中有效。（在类的作用域中创建的局部类实例，在作用域之外仍能使用）
    • 局部类不能声明为public、protected、private或static。
    • 与成员类一样，局部类不能包含静态字段、方法。唯一例外是同时使用static和final声明的常量。
    • 和成员类一样，不能与任何外层类同名。
    • 接口、枚举类型和注解类型不能局部定义。
    • 局部类能使用同一个作用域中的局部变量、方法参数和异常参数，但必须声明为final。
    • 成员类能访问继承自超类的任何成员以及外层类定义的任何成员，局部类也一样。

  • 匿名类：一种局部类。接口、枚举和注解不能定义为匿名类型。

    • 除了使用static和final声明的常量之外，匿名类不能定义任何静态字段、方法。
    • 接口、枚举类型和注解类型不能匿名定义。
    • 和局部类一样，匿名类不能声明为public、private、protected或static。
    • 匿名类没有名称，所以没有构造方法。如果需要构造方法，用局部类。可以使用构造代码块代替构造方法。
class LinkedStack {
    // 静态成员接口，默认static
    public interface Linkable {}

    // 头节点
    private Linkable head;

    public void push(Linkable node) {}

    public Linkable pop() {}

    public Iterator<Linkable> iterator() {
        return new LinkedIterator();
    }

    // 非静态成员类
    protected class LinkedIterator implements Iterator<Linkable> {
        Linkable current;

        public LinkedIterator() {
            current = head; // 构造方法用了外层类的字段
            // 引用的外层类成员被成员类的同名成员遮盖时才用
            // this.current = LinkedStack.this.head; 
        }

        // 下面三个方法由Iterator接口定义
        public boolean hasNext() {
            return current != null;
        }

        public Linkable next() {
            if (current == null)
                throw new java.util.NoSuchElementException();
            Linkable value = current;
            current = current.getNext();
            return value;
        }

        public void remove() {
            throw new UnsupportedOperationException();
        }
    }
}

class LinkableInteger implements LinkedStack.Linkable {}

非静态内部类编译完后会隐含地保存着一个引用，指向创建它的外部类，静态内部类没有，意味着：

  • 它的创建不需要依赖于外部类。
  • 它不能使用任何外部类的非static成员变量和方法。
class OuterClass {

    public static String name = "cnm";

    static class InnerClass1 {

        public static String a = "a"; // 静态内部类中可以存在静态成员

        public void display() {
            System.out.println(name); // 静态内部类只能访问外部类的静态成员变量和方法
        }
    }

    class InnerClass2 {
        // 非静态内部类中不能存在静态成员，static，下面3个报错
        // static {}
        // static int a;
        // static void test () {};
        public String a = "a";

        // 非静态内部类中可以调用外部类的任何成员，static or not
        public void display() {
            System.out.println(name);
        }

        public OuterClass getOuterClass() {
            return OuterClass.this; // 生成对外部类对象的引用
        }
    }

    public InnerClass2 getInnerClass() {
        return new InnerClass2();
    }

    public void display() {
        System.out.println(InnerClass1.a); // 外部类访问静态内部类
        new InnerClass1().display(); // 可以直接创建静态内部类实例不依赖外部类

        OuterClass.InnerClass2 inner2 = new OuterClass().new InnerClass2(); // 非静态内部的创建依赖外部类
        inner2.getOuterClass().getInnerClass();
        inner2.display();
        System.out.println(inner2.a); // 需要非静态内部类的实例
    }
}

Java提供了两种方式实现多重继承：接口和内部类。

使用接口实现多重继承是普遍的方式，如果父类为抽象类或者具体类，只能通过内部类实现多重继承。

class Father {
    public int strong() {
        return 9;
    }
}

class Mother {
    public int kind() {
        return 8;
    }
}

class Son {
    // 内部类继承
    class Father_1 extends Father {
        public int strong() {
            return super.strong() + 1;
        }
    }

    class Mother_1 extends Mother {
        public int kind() {
            return super.kind() - 2;
        }
    }

    public int getStrong() {
        return new Father_1().strong();
    }

    public int getKind() {
        return new Mother_1().kind();
    }
}
interface B {
}

public class A {
    public B c() {
        // 局部内部类定义在方法里
        class C implements B {
            private String s;

            public String r() {
                return s;
            }
        }
        return new C();
    }

    public static void main(String[] args) {
        A a = new A();
        B b = a.c();
    }
}
// 局部类能访问的变量
class A { protected char a = 'a'; }
class B { protected char b = 'b'; }
public class C extends A {
    private char c = 'c'; // 局部类可见
    public static char d = 'd';
    public void createLocalObject(final char e) {
        final char f = 'f';
        int i = 0; // 没声明为final，局部类不能使用
        class Local extends B {
            char g = 'g';
            public void printVars() {
                // 这个类能访问
                System.out.println(g); // (this.g) g是这个类的字段
                System.out.println(f); // f是声明为final的局部变量
                System.out.println(e); // e是声明为final的局部参数
                System.out.println(d); // (C.this.d) d是外层类的字段
                System.out.println(c); // (C.this.c) c是外层类的字段
                System.out.println(b); // b是这个类继承的字段
                System.out.println(a); // a是外层类继承的字段
            }
        }
        Local l = new Local(); 
        l.printVars(); 
    }
}

如果创建了局部类的一个实例，执行完定义这个类的代码块后，实例不会自动消失。

因此，即便这个类在局部定义，这个类的实例能跳出定义它的地方。

代码块是局部变量的作用域，在之外无法访问。

这是词法作用域，局部类能使用局部变量，而且会从不复存在的词法作用域中创建变量值的副本。

public class Weird {
    // 静态成员接口
    public static interface IntHolder {
        public int getValue();
    }

    public static void main(String[] args) {
        IntHolder[] holders = new IntHolder[10];
        for (int i = 0; i < 10; i++) {
            final int fi = i;
            // 局部类
            class MyIntHolder implements IntHolder {

                public int getValue() {
                    return fi; // 使用前面定义的final变量
                }
            }
            holders[i] = new MyIntHolder();
        }
        // 局部类不在作用域中了，因此不能使用，但是在数组中保存有这个类的10个有效实例
        // 局部变量fi现在已经不在作用域中了，但仍然在那10个对象getValue()方法的作用域中
        for (int i = 0; i < 10; i++) {
            System.out.println(holders[i].getValue());
        }
    }
}

局部类MyIntHolder有时也叫闭包（closure）。

闭包是一个对象，它保存作用域的状态，并让这个作用域在后面可以继续使用。

abstract class Bird {

public class Test {

        bird.fly();

    public static void main(String[] args) {

            // 匿名内部类为局部内部类，所以局部内部类的所有限制同样对匿名内部类生效
            public int fly() {

            }

        });
    }
}
interface InnerClass {
    int getNumber();
}

public class OuterClass {
    // 方法的形参被匿名内部类使用，必须为 final
    public InnerClass getInnerClass(final int num, String str) {
        // 匿名内部类没有访问修饰符
        // new 匿名内部类，首先要存在，如果将 InnerClass 接口注释掉，编译会出错
        // new InnerClass() {} ，大括号内是接口的实现代码
        return new InnerClass() {
            // 匿名内部类没有构造方法，它连名字都没有
            int number = num + 3;

            public int getNumber() {
                return number;
            }
        }; // 分号不能省
    }

    public static void main(String[] args) {
        OuterClass out = new OuterClass();
        InnerClass inner = out.getInnerClass(2, "cnm");
        System.out.println(inner.getNumber()); // 5
    }
}

给匿名内部类传递参数的时候，若该形参在内部类中需要被使用，该形参必须要为final。

内部类编译后，产生一个class文件，与外部类并不是同一class 文件，只保留对外部类的引用。

public class OuterClass {
    public void display(final String name) {
        class InnerClass {
            void display() {
                System.out.println(name);
            }
        }
    }
}

看起来name参数是被内部类直接调用，其实编译后如下：

public class OuterClass$InnerClass {
    public InnerClass(String name) {
        this.InnerClass$name = name;
    }
    public void display() {
        System.out.println(this.InnerClass$name);
    }
}

内部类不是直接调用方法传递的参数，而是用自身的构造器对传入的参数进行备份，调用的实际上是自己的属性。

内部类中的属性和外部方法的参数两者外表上看是同一个东西，但在内部类中对属性的改变不会影响到外部的形参，为了保持一致性，规定使用final来避免形参的不改变。

拷贝引用，为了避免引用值发生改变，而导致内部类得到的值不一致，用final来让该引用不可改变。

成员内部类是依赖于外部类的，如果没有创建外部类的对象，则无法对Outter this&0引用进行初始化赋值，也就无法创建成员内部类的对象了。

如果在外部类没有人引用的时候，而成员内部类有人引用，外部类因为被内部类引用所以不会被回收。

匿名内部类中this表示当前对象，如果要使用外部类对象需要加上Outclass.this这才是外部对象。

静态内部类有特殊的地方：静态内部类不依赖于外部类的，可以在不创建外部类对象的情况下创建内部类的对象。

静态内部类不持有指向外部类对象的引用，反编译class文件，没有Outter this&0引用的。

public class Test {
    // 内部类只能访问局部final变量
    public void test(final int a) {
        final int b = 10;
        new Thread() {
            public void run() {
                System.out.println(b);
                System.out.println(a);
            }
        }.start();
        // test方法执行完毕之后，变量a的生命周期就结束了
        // 此时Thread对象的生命周期可能还没有结束，run方法中继续访问变量a就不可能了
        // Java采用了复制的方法，限定必须将变量a限制为final变量，不允许对变量a进行更改（对于引用类型的变量，是不允许指向新的对象）
        // 如果局部变量的值在编译期间就可以确定，则直接在匿名内部里面创建一个拷贝。
        // 如果局部变量的值无法在编译期间确定，则通过构造器传参的方式来对拷贝进行初始化赋值。
    }
}
public class ObjectTest {

    public static void main(String[] args) {
        Person p = new Person("Hi");
        change(p);
        System.out.println(p); // A
    }

    private static void change(Person per) {
        per.setName("A");
        per = new Person("B");
    }
}

class Person {
    // ...
}

在栈区，存在两个Person对象引用：p把对象的引用地址赋值给了per，此时p和per都指向了同一个存放在堆区(heap)Person对象；

per所引用的对象地址被重新赋值：per = new Person(“B”);

自此per就和p就一点关系都没有了：p所指向的对象和per所指向的对象是两个不同的对象了。

无论是基本数据还是对象，都是赋值操作，基本数据赋值直接在栈区进行，对象只是把对象的引用地址赋值，但对应同一个对象。

public static void changeValue(int x) {
    x = x * 2;
}

int num = 5;
changeValue(num);
System.out.println(num);

num作为参数传递给changeValue()方法时，将内存空间中num所指向的值，即”5”,传送给了changeValue()方法中的x变量。

这个x变量也在内存空间中分配了一个存储单元，把num的值5传送给了这个存储单元中。

在changeValue()方法中对x的一切操作都是针对x所指向的这个存储单元，与num所指向的那个存储单元没有关系了，这就是所谓的“值传递”。

值传递：传递的是存储单元中的内容，而非地址或者引用。
