疯狂java讲义

  • byte是8位、有符号整数（有正负数之分）

    • -128(-2^7) ~ 127(2^7-1)
    • 默认值0
    • byte类型用在大型数组中节约空间，因为byte占用空间只有int的四分之一

  • short是16位、有符号整数

    • -32768(-2^15)~32767(2^15 - 1)

  • int是32位、有符号整数

    • -2,147,483,648(-2^31)~2,147,485,647(2^31 - 1)
    • 整型变量默认为int类型。

  • long是64位、有符号整数

    • -9,223,372,036,854,775,808(-2^63)~9,223,372,036,854,775,807(2^63 -1)
    • 默认值0L

  • float是单精度、32位的浮点数

    • float在储存大型浮点数组的时候可节省内存空间
    • 默认值是0.0f
    • 浮点数不能用来表示精确的值，如货币
    • 默认浮点数是 double 类型，若想插入 float 类型的字面量， 加上 f 或 F：float f = 6.02e23 f;

  • double是双精度、64位的浮点数

    • 默认浮点数是 double 类型：double d = 6.02E23;
    • 默认值是0.0d
    • double类型同样不能表示精确的值

  • boolean默认值是false

  • char类型是一个单一的16位Unicode字符

    • 最小值是\u0000（即为0）
    • 最大值是\uffff（即为65,535）
    • 默认值是\u0000（即为0）

  • 8种基本类型

    • 4整型：long short int byte
    • 2浮点：float double
    • 1字符：char
    • 1真值：boolean

  • 8种包装类

    • Number子类
      • Integer
      • Byte
      • Short
      • Long
      • Float
      • Double
    • Object子类
      • Character
      • Boolean

  • null 关键字是特殊的字面量，可以赋值给属于任何引用类型的变量。

  • &&与||：短路

  • &与|：不短路

整数和浮点数可以相互转换。
每个字符对应 Unicode 编码中的一个数字， 字符与整数和浮点数之间也可以相互转换。
布尔值是唯一一种不能和其他基本类型转换的基本类型。

转换成范围更广的类型，是放大转换。
把 int 字面量赋值给 double 类型的变量和把字符字面量赋值给 int 类型的变量时， 执行放大转换。
另一种是缩小转换，缩小转换不安全。
把 13 000 转换成 byte 类型不合理， 因为 byte 类型只能介于 - 128 和 127 之间。

如果整数字面量（ int 类型） 的值落在 byte 和 short 的取值范围内，可以赋值给 byte 或 short 类型的变量。

浮点数转换成整数时，小数部分直接截掉。

大多数情况下， 字符类型和整数类型类似， 需要 int 或 long 类型的地方都可以使用字符。
字符类型没有符号， 即便字符和 short 类型都是 16 位， 也有差异：

  • 自动类型转换：转换前位数低于转换后，short为16位，可以自动转换为32的int。

    • byte,short,char—> int —> long—> float —> double

  • 不能对boolean类型进行类型转换。

  • 不能把对象类型转换成不相关类的对象。
  • 在把容量大的类型转换为容量小的类型时必须使用强制类型转换。

数组类型没有构造方法， 但创建数组时要指定长度。
每个元素都会自动初始化，boolean 类型是false， char 类型是\ u0000， 整数是0， 浮点数是 0.0， 引用类型是 null。

数组字面量在程序运行时， 而不是程序编译时， 创建和初始化。

一个数组中，数组元素类型是唯一的，不能存储多种数据类型的元素。
基本类型的数组不能放大转换。
因为数组可以放大转换成另一种数组类型， 所以编译时和运行时数组的类型并不总是一样。

new 创建多维数组时， 无需指定所有维度的大小， 只要为最左边的几个维度指定大小。

new 创建多维数组时，最好只用矩形数组，每个维度的数组大小相同。

数组作为函数的返回值

数组类型都实现了 Cloneable 接口，能调用 clone() 方法复制自己。
返回值必须校正。 在数组上调用 clone() 方法不会抛出 CloneNotSupportedException 异常：

clone() 方法是浅复制。 如果数组的元素是引用类型， 只复制引用。
任何数组都能被复制， 就算元素类型没有实现 Cloneable 接口也行。

  • System.arraycopy() ：把一个现有数组中的元素复制到另一个现有数组中。
  • java.util.Arrays 类中包含很多处理数组的静态方法。
  • 排序和搜索数组：sort() 和 binarySearch()。
  • equals() ： 比较两个数组的内容。
  • 把数组的内容转换成字符串： Arrays.toString()。
  • 处理多维数组的方法： deepEquals()、 deepHashCode() 和 deepToString()。
  • Arrays.copyOf() 方法产生的数组是浅拷贝，数组的 clone() 方法也是。
