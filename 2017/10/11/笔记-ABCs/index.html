** 疯狂java讲义**

  • byte是8位、有符号整数（有正负数之分）

    • -128(-2^7) ~ 127(2^7 - 1)
    • 默认值0
    • byte类型用在大型数组中节约空间，因为byte占用空间只有int的四分之一

  • short是16位、有符号整数

    • -32768(-2^15) ~ 32767(2^15 - 1)

  • int是32位、有符号整数

    • -2, 147, 483, 648(-2^31)~2, 147, 485, 647(2^31 - 1)
    • 整型变量默认为int类型。

  • long是64位、有符号整数

    • -9, 223, 372, 036, 854, 775, 808(-2^63)~9, 223, 372, 036, 854, 775, 807(2^63 - 1)
    • 默认值0L

  • float是单精度、32位的浮点数

    • float在储存大型浮点数组的时候可节省内存空间
    • 默认值是0.0f
    • 浮点数不能用来表示精确的值，如货币
    • 默认浮点数是double类型，若想插入float类型的字面量，加上f或F：float f = 6.02e23 f;

  • double是双精度、64位的浮点数

    • 默认浮点数是double类型：double d = 6.02E23;
    • 默认值是0.0d
    • double类型同样不能表示精确的值

  • boolean默认值是false

  • char类型是一个单一的16位Unicode字符

    • 最小值是\u0000（即为0）
    • 最大值是\uffff（即为65,535）
    • 默认值是\u0000（即为0）

  • 8种基本类型

    • 4整型：long, short, int, byte
    • 2浮点：float, double
    • 1字符：char
    • 1真值：boolean

  • 8种包装类

    • Number子类
      • Integer
      • Byte
      • Short
      • Long
      • Float
      • Double
    • Object子类
      • Character
      • Boolean

整数和浮点数可以相互转换。

每个字符对应Unicode编码中的一个数字，字符与整数和浮点数之间也可以相互转换。

布尔值是唯一一种不能和其他基本类型转换的基本类型。

转换成范围更广的类型，是放大转换。

把 int 字面量赋值给 double 类型的变量和把字符字面量赋值给 int 类型的变量时， 执行放大转换。

另一种是缩小转换，缩小转换不安全。

把 13 000 转换成 byte 类型不合理，因为 byte 类型只能介于 - 128 和 127 之间。

int i = 13;
byte b = i; // 不允许

如果整数字面量（ int 类型）的值落在byte和short的取值范围内，可以赋值给byte或short类型的变量。

int i = 13;
byte b = (byte) i; // int强制转byte
i = (int) 13.456; // double强制转int，得13

浮点数转换成整数时，小数部分直接截掉。

大多数情况下， 字符类型和整数类型类似， 需要 int 或 long 类型的地方都可以使用字符。
字符类型没有符号， 即便字符和 short 类型都是 16 位， 也有差异：

short s = (short) 0xffff; // 表示数字-1
char c = '\uffff'; // 一个Unicode字符
int i1 = s; // 把short类型转换成int类型，得-1
int i2 = c; // 把字符转换成int类型，得65535

  • 自动类型转换：转换前位数低于转换后，short为16位，可以自动转换为32的int。

    • byte,short,char—> int —> long—> float —> double

  • 不能对boolean类型进行类型转换。

  • 不能把对象类型转换成不相关类的对象。

  • 在把容量大的类型转换为容量小的类型时必须使用强制类型转换。

  • 转换过程中可能溢出或损失精度，例如：

int i =128;
byte b = (byte)i; // byte最大值为127

数组字面量在程序运行时，而不是程序编译时，创建和初始化。

// 数组引用和数组索引都可以是表达式

一个数组中，数组元素类型是唯一的，不能存储多种数据类型的元素。

数组作为函数的返回值

public static int[] reverse(int[] list) {
    int[] result = new int[list.length];

    for (int i = 0, j = result.length - 1; i < list.length; i++, j--) {
        result[j] = list[i];
    }
    return result;
}

数组类型都实现了Cloneable接口，能调用clone()方法复制自己。

返回值必须校正。

int[] data = { 1, 2, 3 };
int[] copy = (int[]) data.clone();

clone()方法是浅复制。

如果数组的元素是引用类型，只复制引用。

任何数组都能被复制，就算元素类型没有实现Cloneable接口也行。

  • System.arraycopy()：把一个现有数组中的元素复制到另一个现有数组中。
  • java.util.Arrays类中包含很多处理数组的静态方法。
  • 排序和搜索数组：sort()和binarySearch()。
  • equals()：比较两个数组的内容。
  • 把数组的内容转换成字符串：Arrays.toString()。
  • 处理多维数组的方法：deepEquals()、deepHashCode()和deepToString()。
  • Arrays.copyOf()方法产生的数组是浅拷贝，数组的clone()方法也是。
