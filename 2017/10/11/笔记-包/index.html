[笔记]-包

2017-10-11 19:45

包机制可以区别类名的命名空间。

  • 把功能相似或相关的类或接口组织在同一个包中，方便类的查找和使用。
  • 包采用了树形目录的存储方式。
  • 包限定了访问权限。

通常使用小写的字母来命名避免与类、接口名字的冲突。
如果在一个包中，一个类想要使用本包中的另一个类，该包名可以省略，用不同包中的类时，import该类。

引用类或接口时， 必须使用完全限定名称。三个例外：

  • java.lang 包始终可以使用简称引用（已自动导入）；
  • p.T 类型中的代码可以使用简称引用 p 包中定义的其他类型（已自动导入）；
  • 已经import 声明导入命名空间里的类型， 可以使用简称引用。

一个Java文件中最多只能有一个 public 顶层类，public 类的目的是供其他包中的类使用。
在一个类中， 声明为 public 的嵌套类或内部类数量不限。
如果 Java 文件中有 public 类， 文件名必须和这个类的名称相同。
一个文件中只定义一个类， 并使用类名命名文件， 是良好的编程习惯。
