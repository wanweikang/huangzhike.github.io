[笔记]-类、继承

2017-10-11 19:45

  • 每个类都有超类。 如果没使用 extends 子句指定超类， 超类是 java.lang.Object。

    • Object 是特殊的类：
      • Object 是唯一一个没有超类的类；
      • 所有 Java 类都从 Object 类中继承方法。

  • 创建类的实例时，一定会调用这个类的构造方法。

  • 创建子类的实例时，一定会调用超类的构造方法。
  • 每个构造方法都会调用超类的构造方法。
  • 如果构造方法的第一句没有 this() 或 super()，javac 编译器插入 super() ，调用超类的无参构造方法。
  • 如果超类没有无参数的可见构造方法，这种隐式调用会导致编译出错。
  • 如果类没有定义无参构造方法，它的所有子类必须定义显式调用超类构造方法，传入参数。
  • 实例字段在构造方法中初始化， 即使不创建类的实例， 类字段也关联在类身上， 类字段在首次加载类时初始化。

  • 静态初始化程序和类方法一样不能使用 this 关键字， 也不能使用类中的任何实例字段或实例方法。

  • PlaneCircle 类创建实例时：

    • 调用 PlaneCircle 类的构造方法；
    • 这个构造方法显示调用了 super(r) ，调用 Circle 类的一个构造方法；
    • Circle() 构造方法隐式调用 super() ，调用 Circle 的超类 Object 的构造方法；
    • 此时到达顶端，接下来开始运行构造方法；
    • 首先运行 Object 构造方法的主体；
    • 返回后，再运行 Circle() 构造方法的主体；
    • 最后，对 super(r) 的调用返回后，执行 PlaneCircle() 构造方法中余下的语句。

构造方法链在一起调用，只要创建对象，就会调用一系列构造方法，从子类到超类，向上到顶端 Object 类为止。
超类的构造方法始终在子类的构造方法的第一个语句中调用，Object 类的构造方法的主体始终最先运行，然后运行 Object 的子类的构造方法，一直向下，直到实例化的那个类为止。

以上都是实例字段，类字段也能被遮盖。
和字段一样， 类方法也能被子类遮盖， 但不能覆盖。
如果把类名当成方法名的一部分， 这两个方法的名称就不一样， 因此没有遮盖。
假如 PlaneCircle 自定义 PI 字段：public static final double PI = 3.14159265;
PlaneCircle 类中的代码可以通过 PI 或 PlaneCircle.PI 使用这个值， 还可以使用 super.PI 和 Circle.PI 引用旧值。
但是， PlaneCircle 继承的 c() 方法是在 Circle 类中定义的，就算 Circle.PI 被PlaneCircle.PI 遮盖了， 这个方法还是用 Circle.PI 。
方法覆盖和字段遮盖不一样。引用遮盖的字段， 只需把对象校正成适当超类的实例， 但不能使用这样调用覆盖的实例方法。

不要在父类构造器中调用被子类覆盖的方法。

编译器在编译时并不知道要调用哪个方法，在运行时使用“ 虚拟方法查找”，查找适用于对象的方法。
解释o.c() 时，检查变量 o 引用的对象的真正运行时类型， 然后找到适用于这个类型的 c() 方法。
不会直接使用关联在变量 o 表示的静态类型身上的 c() 方法， 如果这么做， 方法覆盖就不成立了。
实例方法默认使用虚拟查找。

  • 继承规则：两同两小一大

    • 方法名，形参列表相同
    • 子类方法返回值、抛出异常比父类小或相同，
    • 子类方法的访问权限比父类大或相同

  • 父类中声明为 public 的方法在子类中也必须为 public（不能缩小）。

  • 父类中声明为 protected 的方法在子类中要么声明为 protected，要么为 public，不能为 private。
  • 父类中声明为 private 的方法，不能被继承。
  • 子类拥有父类非 private 属性，方法。
  • Java的继承是单继承，一个子类只能继承一个父类。
  • extends：继承一个类。
  • implements：同时继承多个接口。
