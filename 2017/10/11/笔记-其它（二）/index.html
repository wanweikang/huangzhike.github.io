2017-10-11 19:45:17

如果不再使用的对象存在有效（但不再使用）的引用，仍然会发生内存泄露。
如果某个方法运行的时间很长，这个方法中的局部变量会一直保存对象的引用。

使用 HashMap 或类似的数据结构关联两个对象时，也可能会发生内存泄露。
就算有一个对象不再需要使用了，哈希表中仍然存有两个对象之间的关联，因此在回收哈希表之前，这两个对象一直存在。
如果哈希表的生命周期比其中的对象长得多，就可能导致内存泄露。

clone() 方法使用 Object 类的 clone() 方法：

  • 基本类型拷贝值，如 int、float 。
  • 实例对象拷贝地址引用。
  • String 字符串拷贝地址引用。修改时，从字符串池中重新生成新的字符串，原字符串对象不变。

不要返回一个引用可变对象的访问器方法，这样会破坏封装
