[笔记]-修饰符

2017-10-11 19:45

变量类型：局部变量，成员变量，类变量

  • 访问器方法命名：以“get”和“set”开头，boolean 类型，方法名以“is”开头。

  • static 变量和方法关联在类身上，不依赖实例，被所有实例共享，无论实例化多少对象，只有一份拷贝。

  • static 方法通过类调用，也可以通过实例对象访问。
  • static 方法只能调用 static 变量和方法。
  • static 方法不能引用 this、super。

实例有一个 this 引用指向当前对象， 但 static 方法不关联在实例身上，没有 this 引用， 因此不能访问实例字段。
实例方法可以使用 this 关键字， 但类方法不能使用。 因为类方法不关联在单个对象身上。

执行顺序应该为：静态代码块 > 构造代码块 > 构造函数。

构造代码不是在构造函数之前运行的，它是依托构造函数执行的。

  • final 不能修饰构造方法。
  • 父类的 private 方法不能被子类覆盖，所有 private 方法都隐式添加了 final 。
  • final 声明的类， 其中所有方法也都隐式 final 。
  • final 对象的引用不能指向不同的对象，但是值可以改变。

  • 如果字段同时使用 static 和 final ，就是编译时常量， javac 会将其内联化。

  • 修饰类

    • public：任何类都可以访问该类
    • default：相同包内的类可以访问该类
    • private：用于修饰内部静态类

  • 修饰成员变量和方法：

    • public：能访问这个类的地方访问这个成员
    • protected：包里的所有类都能访问（ 等于默认），子类能访问该成员（不管子类在哪个包中定义）
    • default：包内的类可以访问，包外部不能（不允许在包外部的子类中访问成员）
    • private：除了在类内部， 其他地方都不能访问

接口变量隐式声明为 public static final，接口方法默认 public。

假设 A 类使用 protected 声明了字段 x， 在另一个包中定义的 B 类继承 A 类。
B 类继承了 protected 声明的字段 x， 在 B 类的代码中可以访问当前实例的这个字段， 引用 B 类实例的代码也能访问这个字段。

• 子类继承超类中所有可以访问的实例字段和实例方法；
• 如果子类和超类在同一个包中定义， 子类继承所有非 private 实例字段和方法；
• 如果子类在其他包中定义， 它继承所有使用 protected 和 public 声明的实例字段和方法；
• private 字段和方法不会被继承，类字段和类方法也一样；
• 构造方法不会被继承（ 而是链在一起调用）。
