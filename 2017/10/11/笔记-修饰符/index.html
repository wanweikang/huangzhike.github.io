[笔记]-修饰符

2017-10-11

变量类型：局部变量，成员变量，类变量

class C {
    // 静态变量（类变量）
    private static double a;// 默认值：数值型是0，布尔型是false，引用类型是null

    private double b;// 成员变量，有默认值，同上
    // 实例变量可直接通过变量名访问，在静态方法及其他类中用完全限定名：ObejectReference.VariableName

    public void xxx() {
        // 访问修饰符不能用于局部变量，只在声明它的方法或块中可见，在栈上分配
        int a = 0; // 没有默认值，声明后必须初始化
        a += 1;
    }
}

访问器方法命名：以“get”和“set”开头，boolean 类型，方法名以“is”开头。

  • static变量和方法关联在类身上，不依赖实例，被所有实例共享，无论实例化多少对象，只有一份拷贝。
  • static方法通过类调用，也可以通过实例对象访问。
  • static方法只能调用static 变量和方法。
  • static方法不能引用this、super。

实例有一个this引用指向当前对象，但static方法不关联在实例身上，没有this引用，因此不能访问实例字段。

实例方法可以使用this关键字，但类方法不能使用。因为类方法不关联在单个对象身上。

class C {

    static {
        // JVM 加载类的时候会执行静态代码块，优先于主方法执行
        System.out.println("静态代码块"); // 静态代码块优先构造块执行，只执行一次
    }
    {
        System.out.println("构造块1"); // 每次实例化都执行构造块
    }

    public C() {
        System.out.println("构造方法");
    }

    {
        System.out.println("构造块2");
    }
}

public class A {
    static {
        System.out.println("主方法静态代码块");
    }

    public static void main(String[] args) {
        new C();
        new C();
    }
}
// 主方法静态代码块
// 静态代码块
// 构造块1
// 构造块2
// 构造方法
// 构造块1
// 构造块2
// 构造方法

执行顺序应该为：静态代码块 > 构造代码块 > 构造函数。

构造代码不是在构造函数之前运行的，它是依托构造函数执行的。

// final类不能被继承，没有子类
public final class Post {

    // final变量不可修改，声明时必须初始化
    public final static String DEAFUTL = "a";

    final int[] a = { 1, 2 };

    // final方法可以被子类继承，不能被子类方法覆盖，可以重载
    public final static void c() {
        a[1] = 3; // ok
    }
}
  • final不能修饰构造方法。
  • 父类的private方法不能被子类覆盖，所有private方法都隐式添加了final 。
  • final声明的类，其中所有方法也都隐式final。
  • final对象的引用不能指向不同的对象，但是值可以改变。
  • 如果字段同时使用static和final，就是编译时常量，javac会将其内联化。

修饰类

  • public：任何类都可以访问该类
  • default：相同包内的类可以访问该类
  • private：用于修饰内部静态类

修饰成员变量和方法：

  • public：能访问这个类的地方访问这个成员
  • protected：包里的所有类都能访问（ 等于默认），子类能访问该成员（不管子类在哪个包中定义）
  • default：包内的类可以访问，包外部不能（不允许在包外部的子类中访问成员）
  • private：除了在类内部， 其他地方都不能访问

接口变量隐式声明为public static final，接口方法默认public。

假设A类使用protected声明了字段x，在另一个包中定义的B类继承A类。

B类继承了protected声明的字段x，在B类的代码中可以访问当前实例的这个字段，引用B类实例的代码也能访问这个字段。

但是B类的代码中不一定能读取任何一个A类实例的受保护字段。

// A.java
package cnm.mdzz;
public class A {
    protected String x;
    public A(String x) {
        this.x = x;
    }
    public String getName() {
        return "A: " + x;
    }
}
// B.java
package cnm.mdzz.s;
import cnm.mdzz.A;
public class B extends A {
    public B(String x) {
        super(x);
    }
    public String getName() {
        return "B: " + x;
    }
}

如果把下面方法添加到B类中， 会导致编译出错， 因为B类的实例无法访问任何一个 A 类的实例：

public String examine(A a) {
    return a.name;
}

如果改成：

public String examine(B b) {
    return b.name;
}

就能编译通过，因为同一类型的多个实例可以访问各自的protected字段。

如果B类和A类在同一包中，任何一个B类的实例都能访问任何一个A类实例的全部受保护字段，因为使用protected声明的字段对同一个包中的每个类都可见。

  • 子类继承超类中所有可以访问的实例字段和实例方法；
  • 如果子类和超类在同一个包中定义，子类继承所有非private实例字段和方法；
  • 如果子类在其他包中定义，它继承所有使用protected和public声明的实例字段和方法；
  • private字段和方法不会被继承，类字段和类方法也一样；
  • 构造方法不会被继承（而是链在一起调用）。
