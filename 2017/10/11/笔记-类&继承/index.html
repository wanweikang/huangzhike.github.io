[笔记]-类&继承

2017-10-11
class Circle {
    public static final double PI = 3.14;
    protected double r;

    public Circle(double r) {
        this.r = r; 
        c(); // 实例方法，也可用this.c()
    }

    public Circle() {
        this(1.0); // 第一行
    }

    public double c() {
        return (PI * r * r);
    }
}

public class PlaneCircle extends Circle {
    // 自动继承了Circle类的字段和方法
    private final double cx, cy; 

    public PlaneCircle(double r, double x, double y) {
        super(r); // 调用超类的构造方法
        // super() 和 this() 在第一句用
        this.cx = x; 
        this.cy = y; 
    }
    // 用了继承的实例字段r
    public boolean isInside(double x, double y) {
        double dx = x - cx, dy = y - cy; // 到圆心的距离
        double distance = Math.sqrt(dx * dx + dy * dy); // 勾股定理
        return (distance < r); 
    }
}
// 声明变量p，存储PlaneCircle对象的引用
PlaneCircle p = new PlaneCircle(1.0, 0.0, 0.0);
Circle c = p; // 无需校正，赋值给Circle类型的变量
PlaneCircle p = (PlaneCircle) c; // 缩小转换要校正
boolean origininside = ((PlaneCircle) c).isInside(0.0, 0.0);

  • 每个类都有超类。如果没使用extends子句指定超类，超类是java.lang.Object。

    • Object是特殊的类：
      • Object是唯一一个没有超类的类；
      • 所有Java类都从Object类中继承方法。

  • 创建类的实例时，一定会调用这个类的构造方法。

  • 创建子类的实例时，一定会调用超类的构造方法。

  • 每个构造方法都会调用超类的构造方法。

  • 如果构造方法的第一句没有this()或super()，javac编译器插入super()，调用超类的无参构造方法。

  • 如果超类没有无参数的可见构造方法，这种隐式调用会导致编译出错。

  • 如果类没有定义无参构造方法，它的所有子类必须定义显式调用超类构造方法，传入参数。

  • 实例字段在构造方法中初始化，即使不创建类的实例，类字段也关联在类身上，类字段在首次加载类时初始化。

  • 静态初始化程序和类方法一样不能使用this关键字，也不能使用类中的任何实例字段或实例方法。

  • PlaneCircle类创建实例时：

    • 调用PlaneCircle类的构造方法；
    • 这个构造方法显示调用了super(r)，调用Circle类的一个构造方法；
    • Circle()构造方法隐式调用super()，调用Circle的超类Object的构造方法；
    • 此时到达顶端，接下来开始运行构造方法；
    • 首先运行Object构造方法的主体；
    • 返回后，再运行Circle()构造方法的主体；
    • 最后，对super(r)的调用返回后，执行PlaneCircle()构造方法中余下的语句。

构造方法链在一起调用，只要创建对象，就会调用一系列构造方法，从子类到超类，向上到顶端Object类为止。

超类的构造方法始终在子类的构造方法的第一个语句中调用，Object类的构造方法的主体始终最先运行，然后运行Object的子类的构造方法，一直向下，直到实例化的那个类为止。

假如有三个类A、B和C，都定义了一个名为x的字段，而且C是B的子类，B是A的子类。

在C类的方法中可以：

x // C类的x字段
this.x // C类的x字段
super.x // B类的x字段
((B) this).x // B类的x字段
((A) this).x // A类的x字段
super.super.x // 非法

如果c是C类的实例：

c.x // C类的x字段
((B) c).x // B类的x字段
((A) c).x // A类的x字段

以上都是实例字段，类字段也能被遮盖。

和字段一样，类方法也能被子类遮盖，但不能覆盖。

如果把类名当成方法名的一部分，这两个方法的名称就不一样，因此没有遮盖。

假如PlaneCircle自定义PI字段：public static final double PI = 3.14159265;

PlaneCircle类中的代码可以通过PI或PlaneCircle.PI使用这个值，还可以使用super.PI和Circle.PI引用旧值。

但是，PlaneCircle继承的c()方法是在Circle类中定义的，就算Circle.P被PlaneCircle.PI遮盖了，这个方法还是用Circle.PI。

方法覆盖和字段遮盖不一样。

引用遮盖的字段，只需把对象校正成适当超类的实例，但不能使用这样调用覆盖的实例方法。

class A {
    int i = 1;
    int f() { return i; }
    static char g() { return 'A'; }
}

class B extends A {
    // 遮盖A类的字段i
    int i = 2;

    // 覆盖A类的方法f
    int f() {
        return -i;
    }

    // 遮盖A类的类方法g()
    static char g() {
        return 'B';
    }
}

public class Override {
    public static void main(String args[]) {
        B b = new B();

        A a = (A) b; // 把b校正成A类的实例
        System.out.println(a.i); // A.i，1
        System.out.println(a.f()); // B.f()，-2
        System.out.println(a.g()); // A.g()，A
        System.out.println(A.g()); // A.g()
    }
}

不要在父类构造器中调用被子类覆盖的方法。

class A { 
    public A () { f(); }
    int f() { System.out.println("被覆盖"); }
}
class B extends A { 
    int i; 
    int f() { System.out.println(i); }
}
public class Override {
    public static void main(String args[]) {
        B b = new B(); // 调用父类构造方法，然后调用被覆盖的f方法，i未赋值
    }
}

编译器在编译时并不知道要调用哪个方法，在运行时使用“虚拟方法查找”，查找适用于对象的方法。

解释o.c()时，检查变量o引用的对象的真正运行时类型，然后找到适用于这个类型的c()方法。

不会直接使用关联在变量o表示的静态类型身上的c()方法，如果这么做，方法覆盖就不成立了。

实例方法默认使用虚拟查找。

  • 继承规则：两同两小一大

    • 方法名，形参列表相同
    • 子类方法返回值、抛出异常比父类小或相同
    • 子类方法的访问权限比父类大或相同

  • 父类中声明为public的方法在子类中也必须为public（不能缩小）。

  • 父类中声明为protected的方法在子类中要么声明为protected，要么为 public，不能为private。

  • 父类中声明为private的方法，不能被继承。

class A {
    private String f;
    private void c() {}
    public void p() { this.c(); }
    public String get() { return f; }
}

class B extends A {

    void c() {} // private 改default不算覆写

    public String get() {
        // return f + "b"; // 子类的get不能直接访问超类的私有f
        return super.get() + "b";
    }
}
// new B().p(); 
// 调用父类c方法，子类没有覆写父类，而是定义了一个新方法
  • 子类拥有父类非private属性，方法。
  • Java的继承是单继承，一个子类只能继承一个父类。
  • extends：继承一个类。
  • implements：同时继承多个接口。
class Singleton {
    // 将构造方法封装
    private Singleton () {
    }
}
public class S {
    public static void main(String[] args) {
        Singleton s = new Singleton(); // 无法实例化
    }
}
class Singleton {
    // 内部产生实例化对象
    static Singleton instance = new Singleton();
    // 将构造方法封装
    private Singleton () {
    }
    // static  类方法
    public static Singleton getInstance() {
        return instance;
    }
    public void print () {
        Systerm.out.println("OK")
    }
}
public class S {
    public static void main(String[] args) {
        Singleton s = Singleton.getInstance(); // 访问静态方法
        s.print();
    }
