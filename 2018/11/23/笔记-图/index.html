图G是一组由边E连接的节点（顶点V）。

一个顶点的度是其相邻顶点的数量。比如，A和其他三个顶点相连接，因此A的度为3。

如果图中不存在环，则称该图是无环的。

如果图中每两个顶点间都存在路径，则该图是连通的。

图可以是无向的（边没有方向）或是有向的。

图中每两个顶点间在双向上都存在路径，则该图是强连通的。

图还可以是未加权的或是加权的，加权图的边被赋予了权值。

图的实现

  • 邻接矩阵。

    • 用一个二维数组来表示顶点之间的连接。

    • 如果索引为i的节点和索引为j的节点相邻，则array[i][j] === 1，否则array[i][j] === 0。

    • 不是强连通的图（稀疏图）如果用邻接矩阵来表示，则矩阵中将会有很多0，浪费了空间。

    • 找给定顶点的相邻顶点，即使该顶点只有一个相邻顶点，也不得不迭代一整行。

    • 图中顶点的数量可能会改变，而二维数组不灵活。

    • 但要找出顶点v和w是否相邻，使用邻接矩阵会比较快。

  • 邻接表。

    • 邻接表由图中每个顶点的相邻顶点列表所组成。

    • 可以用列表（数组）、链表，甚至是散列表或是字典来表示相邻顶点列表。

  • 关联矩阵。

    • 在关联矩阵中，矩阵的行表示顶点，列表示边。

    • 使用二维数组来表示两者之间的连通性，如果顶点v是边e的入射点，则array[v][e] === 1；否则，array[v][e] === 0。

    • 关联矩阵通常用于边的数量比顶点多的情况下，以节省空间和内存。

import Dictionary from './dictionary';

export
default class Graph {
    constructor(isDirected = false) {
        // 是否有向
        this.isDirected = isDirected;
        // 数组存储图中所有顶点
        this.vertices = [];
        // 字典存储邻接表，顶点名作为键，邻接顶点列表作为值
        this.adjList = new Dictionary();
    }
    // 向图添加一个顶点
    addVertex(v) {
        if (!this.vertices.includes(v)) {
            // 将该顶点添加到顶点列表中
            this.vertices.push(v);
            // 在邻接表中，设置顶点 v 作为键对应的字典值为一个空数组
            this.adjList.set(v, []); // initialize adjacency list with array as well;
        }
    }
    // 添加顶点之间的边两个顶点作为参数
    addEdge(a, b) { 
        !this.adjList.get(a) && this.addVertex(a);
        !this.adjList.get(b) && this.addVertex(b);
        // 将b加入到a的邻接表中，添加了一条自顶点a到顶点b的边
        this.adjList.get(a).push(b);
        // 添加一条自b向a的边，上一步实现有向图，这一步实现无向图
        // commented to run the improved DFS with topological sorting
        this.isDirected !== true && this.adjList.get(b).push(a);
    }
    getVertices() {
        return this.vertices;
    }
    getAdjList() {
        return this.adjList;
    }
    toString() {
        let s = '';
        for (let i = 0; i < this.vertices.length; i++) {
            s += `$ {this.vertices[i]} - > `;
            const neighbors = this.adjList.get(this.vertices[i]);
            for (let j = 0; j < neighbors.length; j++) {
                s += `$ {neighbors[j]}`;
            }
            s += '\n';
        }
        return s;
    }
}

对图遍历：广度优先搜索（Breadth - First Search，BFS）和深度优先搜索（Depth - First Search，DFS）。

  • 图遍历：

    • 追踪每个第一次访问的节点，并且追踪有哪些节点还没有被完全探索。

    • 完全探索一个顶点要求查看该顶点的每一条边。

    • 对于每一条边所连接的没有被访问过的顶点，将其标注为被发现的，并将其加进待访问顶点列表中。

    • 为了保证效率，访问每个顶点至多两次。

  • 深度优先搜索

    • 数据结构：栈。

    • 将顶点存入栈中，顶点是沿着路径被探索的，存在新的相邻顶点就去访问。

  • 广度优先搜索

    • 数据结构：队列

    • 将顶点存入队列中，最先入队列的顶点先被探索。

  • 用三种颜色标注顶点的访问状态：

    • 白色：该顶点还没有被访问。

    • 灰色：该顶点被访问过，但并未被探索过。

    • 黑色：该顶点被访问过且被完全探索过（访问每个顶点最多两次）。

const Colors = {
    WHITE: 0,
    GREY: 1,
    BLACK: 2
};

// 初始化颜色地图
const initializeColor = vertices => {
    const color = {};
    vertices.forEach(v => color[v] = Colors.WHITE)
    return color;
};

广度优先搜索

  • 从顶点v开始的广度优先搜索算法

    • 创建一个队列Q。

    • 将v标注为被发现的（灰色），并将v入队列Q。

    • 如果Q非空：

      • 将u从Q中出队列；

      • 将标注u为被发现的（灰色）；

      • 将u所有未被访问过的邻点（白色）入队列；

      • 将u标注为已被探索的（黑色）。

import Queue from '../../data-structures/queue';

export const breadthFirstSearch = (graph, startVertex, callback) => {
    const vertices = graph.getVertices();
    const adjList = graph.getAdjList();
    const color = initializeColor(vertices);
    const queue = new Queue();
    // 进队列
    queue.enqueue(startVertex);
    // 循环队列
    while (!queue.isEmpty()) {
        // 出队列
        const u = queue.dequeue();
        const neighbors = adjList.get(u);
        // 未探索
        color[u] = Colors.GREY;

        neighbors.forEach(v => {
            const w = v;
            // 未访问
            if (color[w] === Colors.WHITE) {
                // 未探索
                color[w] = Colors.GREY;
                // 进队列
                queue.enqueue(w);
            }
        })

        // 已探索
        color[u] = Colors.BLACK;
        callback && callback(u);
    }
};

使用BFS寻找最短路径
给定一个图G和源顶点v，找出对每个顶点u，u和v之间最短路径的距离（以边的数量计）。
对于给定顶点v，广度优先算法会访问所有与其距离为1的顶点，接着是距离为2的顶点，以此类推。

export const BFS = (graph, startVertex) => {
    const vertices = graph.getVertices();
    const adjList = graph.getAdjList();
    const color = initializeColor(vertices);
    const queue = new Queue();
    // 距离
    const distances = {};
    // 前溯点
    const predecessors = {};
    // 起始顶点入队列
    queue.enqueue(startVertex);

    vertices.forEach(v => {
        // 到v的距离distances[v]
        distances[v] = 0;
        // 前溯点pred[v]，推导出到其他每个顶点v的最短路径
        predecessors[v] = null;
    })

    while (!queue.isEmpty()) {
        // 出队列
        const u = queue.dequeue();
        // 获得邻点
        const neighbors = adjList.get(u);
        // 未探索
        color[u] = Colors.GREY;

        neighbors.forEach(v => {
            // 如果未访问
            if (color[v] === Colors.WHITE) {
                // 设为未探索
                color[v] = Colors.GREY;
                // 设置 u 和 v 之间的距离（ u 是 v 的前溯点， d[u] 的值已经有了）
                distances[v] = distances[u] + 1;
                // 顶点 u 的邻点 v 时，则设置 v 的前溯点值为 u 
                predecessors[v] = u;
                // 进队列
                queue.enqueue(v);
            }
        })
        // 设为已探索
        color[u] = Colors.BLACK;
    }
    return {
        distances, predecessors
    };
};

深度优先搜索

  • 从第一个指定的顶点开始遍历图，沿着路径直到这条路径最后一个顶点被访问了，接着原路回退并探索下一条路径（先深度后广度）。

  • 深度优先搜索是递归的，使用栈来存储函数调用（递归调用所创建的栈）。

  • 访问顶点v步骤：

    • 标注v为被发现的（灰色）。

    • 对于v的所有未访问的邻点w，访问顶点w，标注v为已被探索的（黑色）。

// import Graph from '../../data-structures/graph';

export const depthFirstSearch = (graph, callback) => {
    const vertices = graph.getVertices();
    const adjList = graph.getAdjList();
    const color = initializeColor(vertices);

    vertices.forEach(v => {
        // 图中每个未被访问的顶点
        // 顶点、颜色数组以及回调函数
        color[v] === Colors.WHITE && depthFirstSearchVisit(v, color, adjList, callback);
    })
};

const depthFirstSearchVisit = (u, color, adjList, callback) => {
    // 访问 u 顶点时，标注为被发现
    color[u] = Colors.GREY;
    callback && callback(u);
    // console.log('Discovered ' + u);
    // 顶点 u 所有邻点
    const neighbors = adjList.get(u);

    neighbors.forEach(v => {
        // 对于顶点 u 的每一个未被访问过的邻点 w ，递归
        color[v] === Colors.WHITE && depthFirstSearchVisit(v, color, adjList, callback);
    })
    // 该顶点和邻点按深度访问之后，回退，该顶点已被完全探索，并将其标注为 black 
    color[u] = Colors.BLACK;
    // console.log('explored ' + u);
};

深度优先搜索遍历图G的所有节点，构建“森林”（有根树的集合）以及一组源顶点（根），并输出两个数组：发现时间和完成探索时间。

export const DFS = graph => {
    const vertices = graph.getVertices();
    const adjList = graph.getAdjList();
    const color = initializeColor(vertices);
    // 顶点u的发现时间d[u]
    const d = {};
    // 顶点u的完成探索时间f[u]
    const f = {};
    // 顶点u的前溯点p[u]
    const p = {};

    // 非对象的变量不能作为引用传递给其他方法
    // 变量作为引用传递：如果该变量在其他方法内部被修改，新值会在原始变量中反映出来

    // 追踪发现时间和完成探索时间
    // 变量值的范围只可能在图顶点数量的一倍到两倍之间
    const time = {
        count: 0
    };
    vertices.forEach(v => {
        f[v] = 0;
        d[v] = 0;
        p[v] = null;
    })

    vertices.forEach(v => {
        color[v] === Colors.WHITE && DFSVisit(v, color, d, f, p, time, adjList);
    })

    return {
        discovery: d,
        finished: f,
        predecessors: p
    };
};

const DFSVisit = (u, color, d, f, p, time, adjList) => {
    // console.log('discovered ' + u);
    color[u] = Colors.GREY;
    // 一个顶点第一次被发现时，追踪其发现时间
    d[u] = ++time.count;
    const neighbors = adjList.get(u);

    neighbors.forEach(v => {
        if (color[v] === Colors.WHITE) {
            // 当它是由引自顶点 u 的边而被发现的，追踪它的前溯点
            p[v] = u;
            DFSVisit(v, color, d, f, p, time, adjList);
        }
    })

    color[u] = Colors.BLACK;
    // 最后，当这个顶点被完全探索后，追踪其完成时间
    f[u] = ++time.count;
    // console.log('explored ' + u);
};

Dijkstra算法：计算从单个源到所有其他源的最短路径的贪心算法，用来计算从图的一个顶点到其余各顶点的最短路径。

// 计算顶点间的 minDistance ，就要搜索 dist 数组中的最小值，返回它在数组中的索引
const minDistance = (dist, visited) => {
    let min = INF;
    let minIndex = -1;
    for (let v = 0; v < dist.length; v++) {
        // 未访问的顶点&&该顶点到源顶点的距离小于上一个
        if (visited[v] === false && dist[v] <= min) {
            // 更新最小距离
            min = dist[v];
            // 更新该顶点索引
            minIndex = v;
        }
    }
    // 返回各顶点到源顶点的最小距离的顶点的索引
    return minIndex;
};
// src也是索引
export const dijkstra = (graph, src) => {
    const dist = [];
    const visited = [];
    const { length } = graph;
    for (let i = 0; i < length; i++) {
        // 首先把所有的距离初始化为无限大
        dist[i] = INF;
        // 未访问的顶点
        visited[i] = false;
    }
    // 把源顶点到自己的距离设为 0
    dist[src] = 0;
    // 找出到其余顶点的最短路径
    for (let i = 0; i < length - 1; i++) {
        // 从尚未访问的顶点中选出距离源顶点最近的顶点索引
        const u = minDistance(dist, visited);
        // 把选出的顶点标为已处理，以免重复计算
        visited[u] = true;
        for (let v = 0; v < length; v++) {
            // 没访问过的顶点&&两顶点相邻并且有距离&&到最近顶点u已经更新了距离&&到最近顶点u的距离+uv的边长<到顶点v的距离
            if (!visited[v] && graph[u][v] !== 0 && dist[u] !== INF && dist[u] + graph[u][v] < dist[v]) {
                // 如果找到更短的路径，则更新最短路径的值
                dist[v] = dist[u] + graph[u][v];
            }
        }
    }
    // 处理完所有顶点后，返回从源顶点到图中其他顶点最短路径的结果
    return dist;
};

Floyd-Warshall算法：计算图中所有最短路径的动态规划算法，可以找出从所有源到所有顶点的最短路径。

export const floydWarshall = graph => {
    const dist = [];
    const { length } = graph;
    // 首先把 dist 数组初始化为每个顶点之间的权值，因为 i 到 j 可能的最短距离就是这些顶点间的权值
    for (let i = 0; i < length; i++) {

        dist[i] = [];
        for (let j = 0; j < length; j++) {
            if (i === j) {
                dist[i][j] = 0;
            } else if (!isFinite(graph[i][j])) {
                dist[i][j] = Infinity;
            } else {
                dist[i][j] = graph[i][j];
            }
        }
    }
    // 通过 k ，得到 i 途径顶点 0 至 k ，到达 j 的最短路径
    for (let k = 0; k < length; k++) {
        for (let i = 0; i < length; i++) {
            for (let j = 0; j < length; j++) {
                // 判断 i 经过顶点 k 到达 j 的路径是否比已有的最短路径更短（核心）
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    // 如果是更短的路径，则更新最短路径的值
                    dist[i][j] = dist[i][k] + dist[k][j];
                }
            }
        }
    }
    return dist;
};

最小生成树（Prim算法和Kruskal算法）

在n个岛屿之间建造桥梁，用最低的成本实现所有岛屿相互连通，岛屿表示为图中的一个顶点，边代表成本。

Prim算法：求解加权无向连通图的MST问题的贪心算法。它能找出一个边的子集，使得其构成的树包含图中所有顶点，且边的权值之和最小。

const minKey = (graph, key, visited) => {
    // Initialize min value
    let min = INF;
    let minIndex = 0;
    for (let v = 0; v < graph.length; v++) {
        if (visited[v] === false && key[v] < min) {
            min = key[v];
            minIndex = v;
        }
    }
    return minIndex;
};
export const prim = graph => {
    const parent = [];
    const key = [];
    const visited = [];
    const {
        length
    } = graph;
    for (let i = 0; i < length; i++) {
        // 首先，把所有顶点（ key ）初始化为无限大（
        key[i] = INF;
        visited[i] = false;
    }
    // 选择第一个 key 作为第一个顶点，同时，因为第一个顶点总是MST的根节点，所以 parent[0] = -1 。
    key[0] = 0;
    parent[0] = -1;
    // 对所有顶点求MST
    for (let i = 0; i < length - 1; i++) {
        // 从未处理的顶点集合中选出 key 值最小的顶点
        const u = minKey(graph, key, visited);
        // 把选出的顶点标为 visited ，以免重复计算
        visited[u] = true;
        for (let v = 0; v < length; v++) {
            if (graph[u][v] && !visited[v] && graph[u][v] < key[v]) {
                // 如果得到更小的权值，则保存MST路径
                parent[v] = u;
                // 更新其权值
                key[v] = graph[u][v];
            }
        }
    }
    // 处理完所有顶点后，返回包含MST的结果
    return parent;
};

Prim算法和Dijkstra算法非常相似。

Prim算法用parent 数组保存MST的结果，用key数组保存权值最小的边，而在Dijkstra算法中，用dist数组保存距离。

可以修改Dijkstra算法，加入parent 数组，这样就可以在求出距离的同时得到路径。

Kruskal算法：求加权无向连通图的MST的贪心算法。

// 防止MST出现环路
const find = (i, parent) => {
    while (parent[i]) {
        i = parent[i]; // eslint-disable-line prefer-destructuring
    }
    return i;
};
const union = (i, j, parent) => {
    if (i !== j) {
        parent[j] = i;
        return true;
    }
    return false;
};
const initializeCost = graph => {
    const cost = [];
    const { length } = graph;
    for (let i = 0; i < length; i++) {
        cost[i] = [];
        for (let j = 0; j < length; j++) {
            if (graph[i][j] === 0) {
                cost[i][j] = INF;
            } else {
                cost[i][j] = graph[i][j];
            }
        }
    }
    return cost;
};
export const kruskal = graph => {
    const { length } = graph;
    const parent = [];
    let ne = 0;
    let a;
    let b;
    let u;
    let v;
    // 首先把邻接矩阵的值复制到 cost 数组，以方便修改且可以保留原始值
    const cost = initializeCost(graph);
    // 当MST的边数小于顶点总数减 1 时
    while (ne < length - 1) {
        // 找出权值最小的边
        for (let i = 0, min = INF; i < length; i++) {
            for (let j = 0; j < length; j++) {
                if (cost[i][j] < min) {
                    min = cost[i][j];
                    a = u = i;
                    b = v = j;
                }
            }
        }
        // 检查MST中是否已存在这条边，以避免环路
        u = find(u, parent);
        v = find(v, parent);
        // 如果 u 和 v 是不同的边，则将其加入MST
        union(u, v, parent) && ne++; 
        // 从列表中移除这些边，以免重复计算
        cost[a][b] = cost[b][a] = INF;
    }
    // 返回MST
    return parent;
};
