[笔记]-Redis

2018-01-10

整理自：The Little Redis Book

public class RedisJava {
    public static void main(String[] args) {
        // 连接本地服务
        Jedis jedis = new Jedis("localhost");
        System.out.println("连接成功");
        // 查看服务是否运行
        System.out.println("服务正在运行: " + jedis.ping());
        // 设置字符串数据
        jedis.set("xxx", "xxx");
        // 获取存储的数据并输出
        System.out.println("字符串为: " + jedis.get("xxx"));

        // 存储数据到列表中
        jedis.lpush("site-list", "Google");
        jedis.lpush("site-list", "Taobao");
        // 获取存储的数据并输出
        List<String> list = jedis.lrange("site-list", 0, 2);
        for (int i = 0; i < list.size(); i++) {
            System.out.println("列表项为: " + list.get(i));
        }

        // 获取数据并输出
        // keys命令需要一个模式，然后查找所有匹配的关键字
        // 这个命令不应该用在实际的产品代码
        Set<String> keys = jedis.keys("*");
        Iterator<String> it = keys.iterator();
        while (it.hasNext()) {
            String key = it.next();
            System.out.println(key);
        }
    }
}

关系型数据库里，可以认为引入了一个数据结构——表格。

Redis引入了5种不同的数据结构，只有一个是典型的关键字-值型结构。

Redis不仅仅是关键字-值型存储，Redis的5种数据结构中的每一个都至少有一个关键字和一个值。

  • 字符串（Strings)
set users:leto "{name: leto, planet: dune, likes: [spice]}"

get users:leto
  • 散列（Hashes）

散列数据结构很像字符串数据结构，区别在于，散列数据结构提供了额外的间接层：一个域（Field）。

hset users:goku powerlevel 9000

hget users:goku powerlevel

// 设置多个域
hmset users:goku race saiyan age 737
// 获取多个域
hmget users:goku race powerlevel
// 获取所有的域和值
hgetall users:goku
// 列出所有的域
hkeys users:goku
// 删除指定的一个域
hdel users:goku age
  • 列表（Lists）

列表数据结构维护了值的顺序，提供了基于索引的高效操作。

// 将一个新用户推入到列表的前端
lpush newusers goku

// 对列表调整，使得该列表只包含50个最近被推入的用户
ltrim newusers 0 50

// 获取最近的10个用户的详细资料
keys = redis.lrange('newusers', 0, 10)
redis.mget(*keys.map {|u| "users:#{u}"})
  • 集合（Sets）

集合数据结构常常被用来存储只能唯一存在的值。没有对值进行排序，提供了高效的基于值的操作。

sadd friends:leto ghanima paul chani jessica
sadd friends:duncan paul jessica alia

// 高效地（O(1)时间复杂度）识别出用户X是不是用户Y的朋友
sismember friends:leto jessica
sismember friends:leto vladimir

// 查看两个或更多的人是不是有共同的朋友
sinter friends:leto friends:duncan
  • 分类集合（Sorted Sets）

分类集合类似于集合数据结构，主要区分是标记（score），提供了排序（sorting）和秩划分（ranking）的功能。

// 秩分类的朋友名单
zadd friends:duncan 70 ghanima 95 paul 95 chani 75 jessica 1 vladimir

// 对于duncan的朋友，计算出标记（score）为90或更高的人数
zcount friends:duncan 90 100

// 获取chani在名单里的秩（rank）
zrevrank friends:duncan chani
// zrank：从高到低
// zrevrank：从低到高

大O表示法

  • 常数时间复杂度O(1)：最快速，无论处理5个元素还是5百万个元素，相同的性能。

    • sismember：一个值是否属于一个集合。许多Redis命令都具有O(1)的时间复杂度。

  • 对数时间复杂度O(log(N))：第二快速的，其通过使需扫描的区间不断皱缩来快速完成处理，使用“分而治之”的方式。

    • zadd：其中N是在分类集合中的元素数量。

  • 线性时间复杂度O(N)：在一个表格的非索引列里进行查找就需要O(N)次操作。

    • ltrim：其中N不是列表所拥有的元素数量，而是被删除的元素数量。

通过不同的关键字去查询相同的值。

如：通过电子邮件（当用户开始登录时）去获取用户的具体信息，或者通过用户id（在用户登录后）去获取。

  • 不实效的解决方法，其将用户对象分别放置到两个字符串值里去：

    set users:leto@dune.gov "{id: 9001, email: 'leto@dune.gov', ...}"
set users:9001 "{id: 9001, email: 'leto@dune.gov', ...}"

  • Redis没有提供将一个关键字链接到另一个的功能，可以使用散列数据，摆脱重复：

    set users:9001 "{id: 9001, email: leto@dune.gov, ...}"
hset users:lookup:email leto@dune.gov 9001
// 使用域来作为一个二级索引，然后去引用单个用户对象

// 通过id来获取用户信息
get users:9001

// 通过电子邮箱来获取用户信息
id = redis.hget(‘users:lookup:email’, ‘leto@dune.gov’)
user = redis.get(“users:#{id}”)


引用和索引（References and Indexes）

集合数据结构很常被用来实现索引：

sadd friends:leto ghanima paul chani jessica
// 这个集合里的每一个成员都是一个Redis字符串数据结构的引用
// 而每一个引用的值则包含着用户对象的具体信息

// 如果chani改变了她的名字，或者删除了她的帐号，chani也有她的朋友
sadd friends_of:chani leto paul


事务（Transactions）

Redis实际上是单线程运行的，这就是为什么每一个Redis命令都能够保证具有原子性。

当一个命令在执行时，没有其他命令会运行。
