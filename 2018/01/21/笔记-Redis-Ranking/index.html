[笔记]-Redis-Ranking

2018-01-21

扒自这篇文章。

第一次访问，查数据库，查出topN（使用SQL排序），丢给redis(使用sorted set数据类型)。

以后均访问redis，排序在redis。

每次积分变化的时候，到一定的变化数量，拿topN的最后一个判别，大于某个阈值则丢进redis排序。

比如现在第50名的积分是100，那80分以下的应该就没必要扔给redis了。

用户是会不断增加的，如果刚开始只有前50，后5千万人的积分大于第50名，就往redis加入这个用户（看起来很多，其实内存不大）。

    // 批量加入用户List
    @Override
    public long zaddList(String key, List<User> userList) {
        Jedis jedis = jedisPool.getResource();
        Long result = null;
        for (int i = 0; i < userList.size(); i++) {
            result = jedis.zadd(key, userList.get(i).getScore(), JSONArray.toJSONString(userList.get(i)));
        }
        jedis.close();
        return result;
    }

    // 单个用户加入 可用于更新（因为是Set）
    @Override
    public long zadd(String key, double score, User user) {
        Jedis jedis = jedisPool.getResource();
        Long result = jedis.zadd(key, score, JSONArray.toJSONString(user));
        jedis.close();
        return result;
    }

    // 拿榜单信息
    @Override
    public Set<String> zgetAll(String key, long start, long end) {
        Jedis jedis = jedisPool.getResource();
        Set<String> userSet = jedis.zrange(key, start, end);
        // System.out.println("userSet:" + userSet);
        jedis.close();
        return userSet;
    }

    // 拿最后一名的 start跟end必须标记最后一名的位置索引
    @Override
    public Set<String> getTopLast(String key, long start, long end) {
        Jedis jedis = jedisPool.getResource();
        Set<String> userSet = jedis.zrange(key, start, end);
        // System.out.println("userSet:" + userSet);
        jedis.close();
        return userSet;
    }
public String queryTopN() {
    List<User> userList = null;
    try {
        // 拿出redis的排行榜结果
        Set<String> resultSet = jedisClient.zgetAll("Toptest", (long) 0, (long) 20);
        // System.out.println("resultSet" + resultSet.toString());

        Iterator<String> iter = resultSet.iterator();
        if (resultSet.size() > 0) {
            // 字符串转为list
            userList = new ArrayList<>();
            while (iter.hasNext()) {
                User user = JSONObject.parseObject(JSONArray.toJSONString(iter.next()), User.class);
                // System.out.println("user" + user.toString());
                userList.add(user);
            }
            return JSON.toJSONString(userList);
        } else {
            // System.out.println("Toptest没查过");
            userList = userService.queryTopN();
            // 没查过就查一次再丢给redis
        }
    } 

    try {
        // 如果没有缓存就批量加入到redis缓存中
        jedisClient.zaddList("Toptest", userList);
    } catch (Exception e) {
        e.printStackTrace();
    }

    return JSON.toJSONString(userList);
}

修改了用户积分的时候就与缓存排行榜的最后一名比较，大于就丢进redis排序，小于就不丢进去。

所以只有第一次访问的时候是使用了数据库的SQL排序查询。

因为序列化，每个user都有独特的标记，所以要从redis中拿user去判断，而不是新建一个user（hash不一样）。
