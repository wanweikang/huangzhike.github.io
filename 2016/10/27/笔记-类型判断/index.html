[笔记]-类型判断

2016-10-27

2016-11-18重新整理更新

  • 基本类型：number、string、boolean、undefined、null
  • 引用类型：object、function、array、RegExp、Math、Date。
  • 值类型判断用typeof，引用类型判断用instanceof。
  • 基本类型保存值，引用类型保存地址。
  • 五种基本数据类型是按值访问的，可以操作保存在变量中的实际值。
  • 引用类型的值是按引用访问的，引用类型的值是保存在内存中的对象。
  • 当复制保存着对象的某个变量时，操作的是对象的引用。
  • 但在为对象添加属性是，操作的是实际的对象。
  • 引用类型的值，可以添加属性和方法，也可以改变和删除其属性和方法。
  • 但不能给基本类型的值添加属性，尽管不会报错。
  • 复制引用操作，两个变量实际上将引用同一个对象。因此，改变其中一个变量，就会影响另一个变量。
// 基本类型
var a = 1; // a变量存储的是数值1
var b = a;
a = 2;
b; // 1，保存副本，改变一个另一个不会影响
// 引用类型
var a = function() {}; //  a存储的是function的内存地址
var b = a; // 把匿名函数地址给了b
b.ha = 1; // 给b添加属性
a.ha; // 1，指向同一个对象

var R1 = [1,2,3]; // 存储的是内存地址
var R2 = R1;
R1.push(4);
R2; // [1, 2, 3, 4]，R1、R2指向同一个地址
R1 === R2; // true
var R1 = [1];
R1 === R2; // false，引用指向最终被引用的对象，不是引用本身

  • JavaScript共有六种数据类型：string number boolean null undefined object。

  • 简单基本类型（string、boolean、number、null、undefined）本身不是对象。

  • undefined类型只有一个值，即undefined。使用var声明变量但未初始化时，变量值就是undefined。

  • 对未初始化的变量执行typeof返回undefined，对未声明的变量执行typeof同样返回undefined。

  • 以下返回undefined：

    • 访问未修改的全局变量undefined。
    • 没有定义return表达式的函数隐式返回。
    • return表达式没有显式返回内容。
    • 访问不存在的属性。
    • 函数参数没有被显式的传递值。
    • 被设置为undefined值的变量。

  • null有时会被当作对象类型，typeof null返回字符串”object”。null是基本类型。

  • 内置对象：String Number Boolean Object Function Array Date RegExp Error。

  • 确定值类型：

    • typeof运算符
    • instanceof运算符
    • constructor属性
    • Object.prototype.toString方法

  • typeof 操作符返回下列字符串：

    • “undefined”–未定义或者是undefined；
    • “boolean”–布尔值；
    • “string”–字符串；
    • “number”–数值；
    • “object”–对象（正则，数组等）或者null；
    • “function”–函数。
typeof "1"; // "string"
typeof 1; // "number"
typeof NaN; // "number"
typeof true; // "boolean"
typeof !NaN // "boolean"
typeof undefined; // "undefined"
typeof a; // "undefined" (如果未声明)
typeof null; // "object"
typeof []; // "object"
typeof new Date(); // "object" 
typeof new Number(1.2); // "object" 
typeof function () {};  // "function"
typeof Math.abs; // "function"
typeof(Object); // "function"  
typeof(Function); // "function"
  • typeof操作符可以判断number、boolean、string、function、undefined。

typeof function和object都是对象，为何输出两种答案？

var obj = {a: 10};
var arr = [5];

这其实叫“语法糖”。

本质是：

var obj = new Object();
obj.a = 10;

var arr = new Array();
arr[0] = 5;

// Object和Array都是函数
typeof (Object); // function
typeof (Array); // function

所以，对象都是通过函数创建的。

对象是函数创建的，函数却又是一种对象。

  • typeof应用：检查一个没有声明的变量，而不报错，而不是检查对象的类型。
if (a) {
}
// ReferenceError: a is not defined

// 正确
if (typeof a === "undefined") {
}

  • 值类型通过typeof判断，string/number/boolean，引用类型，返回值只有object/function，如，typeof对array和object结果都是object。

  • instanceof确定对象是否为某个构造函数的实例。语法：a instanceof constructor。

var f= new F();
f instanceof F; // true
f instanceof Object; // true

var a = [];
a instanceof Array // true
a instanceof Object // true
// 如果存在继承，对象是多个构造函数的实例，instanceof都返回true
  • instanceof操作符判断构造函数的prototype属性是否在要检测对象的原型链上。即判断instanceof前面的对象是否是后面的对象的实例。
  • instanceof沿着实例的proto，同时沿着构造函数的prototype，如果两条线能找到同一个对象，返回true，否则false。
Object instanceof Object // true
Function instanceof Object // true
Function instanceof Function // true

[1, 2, 3] instanceof Array // true
({}) instanceof Object // true 
// JavaScript引擎把大括号解释为代码块，而不是对象
// 仅设置B.prototype为函数A本身，而不是A构造函数的实例
B.prototype = A;
new B() instanceof A; // false
new String("f") instanceof String; // true
new String("f") instanceof Object; // true

原始类型不能使用instanceof。

"" instanceof String // false
1 instanceof Number // false

constructor属性是对构造此对象的函数的引用。

[].constructor === Array;

constructor和instanceof比较不同上下文的对象（如不同frame）出错，每个frame都有自己的执行环境，跨frame实例化的对象不共享原型链。

var myFrame = document.createElement("iframe"); 
document.body.appendChild(myFrame); // 创建的iframe添加到body
var MyArray = window.frames[window.frames.length-1].Array;
var arr = new MyArray("1"); 
// 构造器不能跨frame
arr instanceof Array; // false
arr.constructor === Array; // false

Object.prototype.toString：取得对象的内部属性[[Class]]（[[]]表示外部不可直接访问的“内部属性”，返回类似”[object Array]”的字符串。配合call，取得对象的内部属性[[Class]]，然后把类型检测转化为字符串比较。

[[Class]] 的值只可能是下面字符串：Arguments, Array, Boolean, Date, Error, Function, JSON, Math, Number, Object, RegExp, String。

function isArray(obj) {
    Array.isArray ? Array.isArray(a) : Object.prototype.toString.call(a) === "[object Array]";
}

isArray([]); // true
// ES5中，对null和undefined调用Object.prototype.toString，返回值由Object变成了Null 和Undefined
// Object.prototype.toString返回标准格式字符串，通过slice截取指定位置
function is(type, obj) {
    var cls = Object.prototype.toString.call(obj).slice(8, -1);
    return obj !== undefined && obj !== null && cls === type;
}

is("String", "test"); // true
is("String", new String("test")); // true
  • 不用obj.toString()，虽然Array继承自Object，有toString方法，但可能被改写，而Object.prototype很少改。
  • 检测对象的类型，推荐使用Object.prototype.toString方法。

  • typeof运算符不能检测数组；
  • ES5的isArray函数是原生方法，但低版本浏览器不支持；
  • 对象自身的constructor属性和instanceof操作符虽然也能检测数组，但在frame中会产生问题；

function isNumber(n) {
    return !isNaN(parseFloat(n)) && isFinite(n);
}
