[笔记]-类型&类型转换

2016-10-27

2016-11-18重新整理

JavaScript中，字符串区分string和包装类型。

number、boolean和string都有包装对象。

包装对象用new创建：

var n = new Number(1); 
var b = new Boolean(true);
var s = new String("s");

虽然包装对象和原来的值一样，但类型已经变为object

typeof new Number(1); // "object"
new Number(1) === 1; // false

typeof new Boolean(true); // "object"
new Boolean(true) === true; // false

typeof new String("s"); // "object"
new String("s") === "s"; // false

如果没写new，Number()、Boolean()和String()被当做普通函数，转换为number、boolean和string类型（不是其包装类型）：

var n = Number("1"); // 1，相当于parseInt()或parseFloat()
typeof n; // "number"

var b = Boolean("false"); // true
var b = Boolean(""); // false
typeof b; // "boolean"

var s = String(1); // "1"
typeof s; // "string"

Function 是函数类型实例的构造函数（constructor）。

Object或String、Number等，都是内置类型实例的构造函数。

对于非原始类型的，通过ToPrimitive() 将值转换成原始类型。

基本类型与对象的转换

var strPrimitive = "I am a string";
strPrimitive.length; // 13
strPrimitive.charAt( 3 ); // "m"
// 引擎自动把字面量转换成 String 对象，所以可以访问属性和方法
// 这个临时的对象会被释放
"abc".charAt === String.prototype.charAt; // true

将对象转换为基本类型：

  • 调用对象的valueOf()方法取得对象的值，如果和上下文的类型匹配，则使用该值。
  • 如果valueOf()取不到值，则调用对象的toString()方法，如果上下文为数值型，则将此字符串转换为数值。

包装对象实例可以使用Object对象提供的原生方法，主要是valueOf方法和toString方法。

valueOf方法

new Number(123).valueOf(); // 123
new String("abc").valueOf(); // "abc"
new Boolean("true").valueOf(); // true

toString方法

new Number(123).toString(); // "123"
new String("abc").toString(); // "abc"
new Boolean("true").toString(); // "true"

包装对象与原始类型值混合运算，包装对象会转化为原始类型（调用自身的valueOf方法）。

new Number(123) + 123; // 246
new String("abc") + "abc"; // "abcabc"

三种包装对象还可以在原型上添加自定义方法和属性，供原始类型的值直接调用。

如，新增double方法，使字符串翻倍。

String.prototype.double = function () {
    return this.valueOf() + this.valueOf();
};

"abc".double(); // abcabc

转化为字符串型：用全局方法String()转换任意类型到string，或直接调用对象的toString()方法；

Array.prototype.toString、Boolean.prototype.toString、Date.prototype.toString、Error.prototype.toString、Function.prototype.toString、Number.prototype.toString

并非所有“对象”都有toString()方法。

null和undefined没有。

["a","b","c"].join(); // "a,b,c"
["a","b","c"].toString(); // "a,b,c"

new Boolean().toString(); // "false"
true.toString(); // "ture"
String(false); // "false"
String(true); // "true"

new Date().toString();
new Date().toLocaleString();
String(Date());     
Date().toString();   

(100).toString(); // "100"
(100).toString(2); // "1100100"
(100).toString(8); // "144"
(100).toString(16); // "64"
String(x); // 变量x转字符串
String(123); // 字面量123转字符串
String(100 + 23); // 表达式转字符串

将一个值加上空字符串可转换为字符串类型。

"" + 10 === "10"; // true
"value:" + null; // "value:null"
"value:" + undefined; // "value:undefined"

function Account(cash) {
    this.cash = cash;
    this.hasMoney = !!cash;
}

emptyAccount.cash; // 0
emptyAccount.hasMoney; // false
// 只要account.cash值大于0，account.hasMoney返回true

转化为数值型

全局方法Number()或ToNumber()把值转换成Number，推荐用parseInt()或parseFloat()来转换。

Number(false); // 0
Number(true); // 1
Number(new Date());   
new Date().getTime();  
Number("3.14"); // 3.14
Number(" "); // 0 （空字符串被转换为 0）
Number(""); // 0
Number("99 88"); // NaN
Number(null); // 0
Number(undefined); // NaN

使用一元加号操作符，可以把字符串转换为数字。

+"10"; // 10

var y = "5"; // y 是字符串类型
var x = + y; // x 是数字类型

+"ACB" // NaN （如果变量不能被转换，仍可转换为数字类型，但值是 NaN (Not a number)）

var y = "John"; // y是字符串类型
var x = + y; // x是数字类型，值是NaN

function toNum(n) {
    return +n;
}

+"010" === 10; // true
Number("010") === 10; // true
parseInt("010", 10) === 10; // 转换为整数

+"010.2" === 10.2; // true
Number("010.2") === 10.2; // true
parseInt("010.2", 10) === 10; // true

Infinity可以用于布尔运算。

Infinity是JavaScript最大的值（NaN除外），-Infinity是最小的值（NaN除外）。

isFinite(Infinity); // false

NaN（Not a Number），其类型是number，主要原因是字符串解析成数字出错。

5 - "x" // NaN，自动将字符串“x”转为数值，但x不是数字，得NaN
// 一些数学函数的运算结果会出现NaN
Math.acos(2); // NaN
Math.log(-1); // NaN
Math.sqrt(-1); // NaN
0 / 0; // NaN
typeof NaN; // "number"
Object.prototype.toString.call(NaN); // [object Number]
NaN === NaN; // false
Boolean(NaN); // false

NaN 与任何数（包括NaN）的运算，都是NaN。

NaN + 3; // NaN
NaN - 3; // NaN
NaN * 3; // NaN
NaN / 3; // NaN

NaN不等于自己，不全等于自己。

isNaN只对数值有效，如果传入其他值，被转成数值。

即，isNaN为true，可能不是NaN，而是一个字符串。

isNaN("Hello") // true
isNaN(Number("Hello")) // true
isNaN({}) // true
isNaN(["xzy"]) // true
isNaN(NaN); // true
isNaN("10"); // false
isNaN(true); // false
isNaN(null); // false
isNaN(23); // false
isNaN("32131sdasd"); // true 
undefined === undefined; // true

使用isNaN之前，先判断数据类型。

function myIsNaN(value) {
    return typeof value === "number" && isNaN(value);
}

判断NaN更可靠的方法是，利用NaN是JavaScript唯一不等于自身的值。

function myIsNaN(value) {
    return value !== value;
}

ES6中，isNaN()是Number的静态方法：Number.isNaN()

判断x是否是整数

// 将x转换成10进制，判断和本身是不是相等：
function isInteger(x) { 
    return parseInt(x, 10) === x; 
}

自动类型转换

5 + null; // 5，null被转换为 0
"5" + null; // "5null"，null被转换为"null"
"5" + 2; // 52，加法比较特殊，因为+同时是加法和连字符
"5" - 2; // 3，"5" 被转换为5
3 + undefined; // NaN   
3 + false; // 3
3 + true; // 4
"5" * "2"; // 10 "5" 和"2" 被转换为5和2
1 +  "2" + "2"; // 122
1 +  +"2" + "2"; // 32
1 +  -"1" + "2"; // 02
+"1" +  "1" + "2"; // 112
// 运算结果不能转换成数字的，返回 NaN
"A" - "B" + "2"; // NaN2
"A" - "B" + 2; // NaN
"a" * "sd"; // NaN
"A" - "B"; // NaN
1 * undefined; // NaN
// 无论运算，只要undefined参与运算，结果都是NaN
3 * "8"; // 24
"3" * "8"; // 24
3 * null; // 0，隐式转换null被转为0
3 * false; // 0，隐式转换false被转为0
3 * true; // 3，隐式转换true被转为1
3 * "8天"; // NaN，数学运算中，不纯的字符串没法隐式转换
false + true * null; // 0
  • 减号-，乘号*，肯定是进行数学运算，所以转化为number类型；
  • 加号+，可能是字符串拼接，也可能是数学运算，所以转化为number或string；
  • 一元运算，如+[]，只有一个操作数的，转化为number类型；
  • 与数字进行关系运算时，纯数字字符串被转为数字，null转为0（null不能进行和0的相等判定），true转为1，false转为0
null < 0.01; // true
null == 0; // false，null是对象，引用类型值，和基本类型值比
false == 0; // true
true == 1; // true

string和number比，string会被隐式转换为number

"66" < "8"; // true
66 < "8"; // false，有数字，就按数字比，"8"转为8
"66" < 8; // false

string和string进行关系运算，比较的是字符编码顺序

"a" < "b"; // true
"A" < "B"; // true
"A" < "a"; // true，大写字母在字符集里面是在小写字母前面
"1" < "A";  // true，数字在字母前端
"blank" < "blue"; // true，一位一位比，直到比出大小
"25678" < "3"; // true，string和string比，比的是字符编码顺序
