正则表达式（regular expression）

var  reg = /xyz/i; // 基本采用字面量法
var  reg = new  RegExp("xyz", "i"); // 构造函数
  • 字面量字符（literal characters）
  • 普通字符包括：字母（abcABC），数字（123），下划线（_）
  • 特殊字符（元字符，metacharacters）
  • 转义符：需要转义：( ) [ ] { } ^ $ * ? \ | + .
var reg = /\^a/
var reg = /\\m/; // 特殊字符要转义

/google\.com/ // 匹配google.com
/C:\\Windows/ // 匹配C:\Windows。
/Windows\d+/ // 匹配Windows后跟1个或更多数字

RegExp方法生成正则对象，转义要两个斜杠，因为字符串内部先转义一次。

/1\+1/.test("1+1"); // true
(new RegExp("1\\+1")).test("1+1"); // true

预定义特殊字符

  • .：单个字符，除换行\n 、行结束符（回车\r），相当于[^\n\r]，还包括行分隔符\u2028和段分隔符\u2029？
  • \w：单词字符，任意字母、数字和下划线，相当于[A-Za-z0-9_]。
  • \W：非单词字符，字母、数字和下划线以外的字符，相当于[^A-Za-z0-9_]。
  • \d：数字，相当于[0-9]。
  • \D：非数字字符，相当于[^0-9]。
  • \s：空白字符，匹配空格，包括制表符、空格符、断行符等，相当于[\t\n\v\f\r]。
  • \S：非空白字符，相当于[^ \t\n\x0B\f\r]。
  • \b：单词边界。
  • \B：非单词边界。
  • \0：NUL 字符。
  • \n：换行符。
  • \f：换页符。
  • \r：回车符。
  • \t：制表符。
  • \v：垂直制表符。
  • \xxx：以八进制数xxx规定的字符。
  • \xdd：以十六进制数dd规定的字符。
  • \uxxxx：以十六进制数xxxx规定的Unicode字符。
/c.t/; // 匹配c和t之间任一字符，只要在同一行，如cat、c2t、c-t，不匹配coot

/00\d/ // 可匹配"007"

/\d\d\d/ // 可匹配"010"

/\w\w/ // 可匹配"js"

/^js$/ // 整行匹配，只匹配"js"

/js/ // 可匹配"jsp"

分界

// ^：行的开头，^\d：数字开头
// $：行的结束，\d$：数字结束
/^abc/; // 是否以abc开始
/xyz$/; // 是否以xyz结尾
/^test$/.test("test"); // true
/^test$/.test("test test"); // false

// \b：单词分隔符
// \B：非单词分隔符
/\bworld/.test("hello-world"); // true
/\bworld/.test("helloworld"); // false
/\Bworld/.test("hello-world"); // false
/\Bworld/.test("helloworld"); // true
/\bhi\b/ // 精确查找hi
/\bhi\b.*\bLucy\b/ // 先是单词hi，然后是任意个任意字符（除换行），最后是单词Lucy
/\b\w{6}\b/ // 刚好6个字符的单词

通常，正则表达式遇到换行符\n就会停止匹配。

var html = "Hello\nworld!";
/.*/.exec(html)[0] // "Hello"

点字符.不匹配换行符，使用\s字符类（[\S\s]指代一切字符），就能包括换行符。

var html = "Hello\nworld!";
/[\S\s]*/.exec(html)[0] // "Hello\nworld!"

另一种写法（非捕获组）：

/(?:.|\s)*/.exec(html)[0] // "Hello\nworld!"

字符集[]：字符类（class），只匹配其中一个就行。如[xyz] 表示x、y、z之中任一个匹配。

/[abc]/.test("apple") // true

[…] // 在集合中的任一个字符
[^…] // 除集合中的任一个字符

脱字符^：如果方括号内第一个字符是[^]，表示除了字符类之中的字符，都可匹配。如[^xyz]表示除x、y、z之外。

/[^abc]/.test("bbc") // false

如果方括号内只有[^]，表示匹配一切字符，包括换行符，而点号.不包括换行符。

"abc efg\n123 456".match(/efg.*456/) // null
"abc efg\n123 456".match(/efg[^]*456/) // ["efg\n123 456"]

注意，脱字符只有在[]第一个位置才有特殊含义。

连字符-：表示字符的连续范围。

[abc]等于[a-c]，[0123456789]等于[0-9]，[A-Z]表示26个大写字母。

/a-z/.test("b") // false
/[a-z]/.test("b") // true

当连字号（dash）不在方括号中，只代表字面含义。只有在方括号中，才表示连续的字符序列。

以下都是合法的字符类简写。

[0-9.,]
[0-9a-fA-F]
[a-zA-Z0-9-]
[1-31] // 不代表1到31，只代表1到3

字符类的连字符必须在头尾两个字符中间，才有特殊含义，否则就是字面含义。[-9]表示连字符和9，而不是0到9。

连字符可以指定Unicode字符范围。

var str = "\u0130\u0131\u0132";
/[\u0128-\uFFFF]/.test(str) // true

不要过分使用连字符，设定一个很大的范围。如[A-z]，表面上是选中从大写的A到小写的z之间52个字母，但在ASCII编码之中，大写字母与小写字母之间还有其他字符。

  • 简单类：[abDF45]表示由abDF45六个字符组成的一个集合
  • 范围类：[a-z]表示小写字母集合。[A-Z]表示大写字母集合。[0-9]表示数字集合。[i-y] 表示小写字母i到y的集合
  • 负向类：[^abc] 表示不包含abc的任意字符集合，[^ab89CD]表示不包含ab89CD这六个字符的集合
  • 组合类：[a-zA-Z0-9] 表示大小写字母以及数字的结合

量词

  • ?：重复零次或一次
  • *：重复零次或多次(任意次)
  • +：重复一次或多次（至少一次）
  • {n}：重复n次
  • {n,m}：重复n~m次(中间不能有空格)
  • {n,}：至少重复n次 
/lo{2,5}k/.test("looook") // true
/t+est/.test("est") // false
/t*est/.test("est") // true

贪婪匹配：? + * 三个量词符，默认贪婪模式，匹配尽可能多的字符。

如，对于字符串”aaaa”，/a+/匹配”aaaa”，不会匹配”aa”。

在量词符后加一个问号，改为非贪婪模式，/a+?/只匹配”a”。

"aaaa".match(/a+?/) // ["a"]
"aaaa".match(/a+/) // ["aaaa"]

匹配数字后的0：

/^(\d+)(0*)$/.exec("102300"); // ["102300", "102300", ""] 
// \d+贪婪匹配，把0全部匹配了，0*只能匹配空字符串。

/^(\d+?)(0*)$/.exec("102300"); // ["102300", "1023", "00"]

\s+是贪婪模式，尽可能多匹配空格，匹配次数较少，性能更高

\s只表示一个空格，匹配次数多，性能相对低。

懒惰限定符

  • *?：重复任意次，但尽可能少重复
  • +?：重复1次或更多次，但尽可能少重复
  • ??：重复0次或1次，但尽可能少重复
  • {n,m}?：重复n到m次，但尽可能少重复
  • {n,}?：重复n次以上，但尽可能少重复 
// 中国的电话号码
/0\d\d-\d\d\d\d\d\d\d\d/ 
// 0开头，然后两个数字，然后一个连字号“-”，最后是8个数字
// 等于
/0\d{2}-\d{8}/

// 以任意个空格隔开的带区号的电话号码
/^\d{3}\s+\d{3,8}$/

/^\d{3}\-\d{3,8}$/ 
// 匹配"010-12345"，无法匹配"010 - 12345"，因为有空格

\s匹配一个空格（包括Tab等空白符），\s+表示至少有一个空格，如” “，”\t\t”等；

分枝条件

A|B // A或B
Java[S|s]cript // 匹配"JavaScript"、"Javascript"
/0\d{2}-\d{8}|0\d{3}-\d{7}/ // 010-12345678或0376-2233445
/\(?0\d{2}\)?[- ]?\d{8}|0\d{2}[- ]?\d{8}/ 
// 3位区号的电话号码，区号可用小括号括起来，也可不用，区号与本地号间可用连字号或空格间隔，也可没间隔。
/\d{5}-\d{4}|\d{5}/ // 5位数字，或连字号间隔的9位数字
/\d{5}|\d{5}-\d{4}/ // 5位数字，或9位数字的前5位

匹配分枝条件时，从左到右测试每个条件，如果满足了某个分枝，不再管其它条件。

  • 重复单个字符：直接在字符后加上限定符；
  • 重复多个字符，用小括号指定子表达式（分组）。
  • /(\d{1,3}\.){3}\d{1,3}/简单的IP地址匹配表达式，它也匹配 256.300.888.999。
  • 只能用冗长的分组，选择，字符类描述正确的IP地址：
/((2[0-4]\d|25[0-5]|[01]?\d\d?)\.){3}(2[0-4]\d|25[0-5]|[01]?\d\d?)/
// 关键是理解2[0-4]\d|25[0-5]|[01]?\d\d?

分组：正则还可以提取子串，用()表示要提取的分组

// 定义两个组
var re = /^(\d{3})-(\d{3,8})$/;
// 从匹配字符串中提取区号和本地号码
re.exec("010-12345"); // ["010-12345", "010", "12345"]
re.exec("010 12345"); // null

如果正则中定义了组，可以在RegExp对象上用exec()方法提取子串。

exec()方法匹配成功后，返回一个Array，第一个元素是原始字符串，后面的字符串表示匹配成功的子串。

exec()方法匹配失败返回null。

// 识别合法时间
var re = /^(0[0-9]|1[0-9]|2[0-3]|[0-9])\:(0[0-9]|1[0-9]|2[0-9]|3[0-9]|4[0-9]|5[0-9]|[0-9])\:(0[0-9]|1[0-9]|2[0-9]|3[0-9]|4[0-9]|5[0-9]|[0-9])$/;
re.exec("19:05:30"); // ["19:05:30", "19", "05", "30"]

有时正则也无法完全识别如”2-30”，”4-31”这样的非法日期：

var re = /^(0[1-9]|1[0-2]|[0-9])-(0[1-9]|1[0-9]|2[0-9]|3[0-1]|[0-9])$/;

电子邮件识别：

/^[\w-]+(\.[\w-]+)*@[\w-]+(\.[\w-]+)+$/;

[\w-]表示所有的字符，数字，下划线及减号，[\w-]+表示集合最少重复一次；

括号表示分组，修饰符星号 (*)表示重复零或多次。匹配任意字母，数字，下划线及中划线的集合，且至少重复一次。

@后部分，分组的修饰符 (+)，表示至少重复一次，后半部分至少有一个点号(.)，且点号后至少有一个字符，限制输入串中必须包含域名。

最后，脱字符(^)和美元符号($)限制，以…开始，且以…结束。

括号常见有三种作用：

一，括号标记子表达式，区别其他表达式，很多命令行程序帮助命令，h和help一样：

h(elp) // h必选，h后的elp可有可无

二，括号分组，正则执行完后，匹配的文本按规则填入分组，如，某数据库的主键格式：四个字符表示省份，四个数字表示区号，然后两位字符表示区县，如yunn0871cg表示云南省昆明市呈贡县，分离出区号和区县的两位字符代码：

var pattern = /\w{4}(\d{4})(\w{2})/;
var result = pattern.exec("yunn0871cg");

"city code = "+result[1]+", county code = "+result[2]; 
// city code = 0871, county code = cg

如果匹配成功，正则的exec方法返回一个数组，数组的第一个元素表示整个串，第一个元素为第一个分组，第二个元素为第二个分组，以此类推。

三，括号辅助引用，同一表达式中，后边式子引用前边匹配的文本。

var str = "hello, world";
var str = "fair enough";
var pattern = /['"][^'"]*['"]/;
// 如果输入：
var str = 'hello, world";

这就要引用，用斜杠加数字表示，\1表示第一个分组，\2表示第二个分组。

var pattern = /(['"])[^'"]*\1/;
// 单引号中不允许双引号，双引号中也不允许单引号
var pattern = /(['"])[^\1]*\1/;
// 第一个括号捕获a，第二个括号捕获c
"abcabc".match(/(.)b(.)/); // ["abc", "a", "c"]

组匹配，不宜用g修饰符，否则match方法不会捕获分组内容。

"abcabc".match(/(.)b(.)/g); // ["abc", "abc"]

\n引用括号匹配的内容，n是从1开始的自然数，表示对应的括号。

// \1表示第一个括号匹配内容a，\2表示第二个括号匹配内容b
/(.)b(.)\1b\2/.test("abcabc") // true

组匹配匹配网页标签。

/<([^>]+)>[^<]*<\/\1>/.exec("bold")[1] // "b"

捕获有属性的标签。

var html = "Helloworld";
var tag = /<(\w+)([^>]*)>(.*)<\/\1>/g;
var match = tag.exec(html);
match[1] // "b"
match[2] // "class="hello""
match[3] // "Hello"

var match = tag.exec(html);
match[1] // "i"
match[2] // ""
match[3] // "world"

非捕获组（Non-capturing group）：(?:x)，不返回该组匹配的内容，这个括号不计入匹配结果。

// 第一个括号是非捕获组，返回的第一个被捕获的组是第二个括号匹配的c
var m = "abc".match(/(?:.)b(.)/);
m[1] // "c"

分解网址。

var url = /(http|ftp):\/\/([^/\r\n]+)(\/[^\r\n]*)/;
url.exec("http://google.com/"); // ["http://google.com/", "http", "google.com", "/"]
// 正常匹配，第一个括号返回网络协议http

var url = /(?:http|ftp):\/\/([^/\r\n]+)(\/[^\r\n]*)/;
url.exec("http://google.com/"); // ["http://google.com/", "google.com", "/"]
// 非捕获匹配，返回结果不包括http

先行断言（Positive look-ahead）：x(?=y)，x在y前面才匹配，y不计入返回结果。

"abc".match(/b(?=c)/); // ["b"]
// b在c前所以匹配，括号对应的c不会返回

后行断言（Negative look-ahead）：x(?!y)，x不在y前面才匹配，y不计入返回结果。

"abd".match(/b(?!c)/); // ["b"]
// b不在c前所以匹配，括号对应的d不会返回

修饰符（modifier）

g修饰符：global，全局匹配。默认情况下，第一次匹配成功后就停止。global将匹配全部符合的结果，主要用于搜索和替换。

var r = /test/g;
// 等价于
var r = new RegExp("test", "g"); 

var  reg = /b/;
// 不加g修饰符，每次匹配都从字符串头部开始
reg.test("abba"); // true
reg.test("abba"); // true
reg.test("abba"); // true

var  reg = /b/g;
// 加g修饰符，每次匹配都从上一次匹配成功处开始
reg.test("abba"); // true
reg.test("abba"); // true
reg.test("abba"); // false

全局匹配可多次执行exec()搜索字符串。指定g后，每次exec()，会更新lastIndex属性，表示上次匹配到的最后索引：

var s = "JavaScript, VBScript, JScript and ECMAScript";
var re = /[a-zA-Z]+Script/g;
// 全局匹配:
re.exec(s); // ["JavaScript"]
re.lastIndex; // 10

re.exec(s); // ["VBScript"]
re.lastIndex; // 20

re.exec(s); // ["JScript"]
re.lastIndex; // 29

re.exec(s); // ["ECMAScript"]
re.lastIndex; // 44

re.exec(s); // null，直到结束仍没匹配到

全局匹配类似搜索，因此不能使用/^...$/，那样只会最多匹配一次。

i修饰符：ignorecase，忽略大小写。

m修饰符：multiline，多行模式，到达一行文本末还继续查找下一行。

  • 字符串的头部或尾部可能会有换行符。正则默认将换行符算入字符串的开头或结尾。
  • 加上m修饰符后会忽略字符串头部或尾部的换行符，即^和$会忽略换行符。
// 不加m修饰符，匹配失败，因为结尾不是world
/world$/.test("hello world\n") // false
// 加上后，换行符被忽略，匹配成功
/world$/m.test("hello world\n") // true

/^javascript/.test("java\njavascript"); // false
/^javascript/m.test("java\njavascript"); // true

修饰符多个一起用：var reg = /test/ig;。
