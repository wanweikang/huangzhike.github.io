[笔记]-三、类型判断

2016-10-27 12:11

基本类型和引用类型

  • 基本类型如数字，布尔值，在内存中有固定的大小，通过变量直接访问基本类型的数据。
  • 引用类型如对象，数组和函数，大小原则上不受限制，通过对其引用的访问来访问它们本身，指向存储对象的地址。

数据类型

  • JavaScript中一共有六种数据类型：string number boolean null undefined object。

  • 简单基本类型（string 、 boolean 、 number 、 null 和 undefined）本身并不是对象。

  • undefined 类型只有一个值，即 undefined。在使用 var 声明变量但未对其加以初始化时，这个变量的值就是 undefined。

  • 对未初始化的变量执行 typeof 操作符会返回 undefined 值，而对未声明的变量执行 typeof 操作符同样也会返回 undefined 值。

以下会返回 undefined：

  • 访问未修改的全局变量 undefined。
  • 由于没有定义 return 表达式的函数隐式返回。
  • return 表达式没有显式返回内容。
  • 访问不存在的属性。
  • 函数参数没有被显式的传递值。
  • 被设置为 undefined 值的变量。
  • null 有时会被当作一种对象类型，typeof null 返回字符串 “object” 。实际上， null 是基本类型。

内置对象
JavaScript中还有一些对象子类型，被称为内置对象。
String Number Boolean Object Function Array Date RegExp Error

JavaScript有三种方法，可以确定一个值到底是什么类型。

  • typeof 运算符
  • instanceof 运算符
  • constructor 属性
  • Object.prototype.toString方法

typeof 操作符返回下列字符串：

  • “undefined”–未定义或者是 undefined；
  • “boolean”–布尔值；
  • “string”–字符串；
  • “number”–数值；
  • “object”–对象（包括正则表达式，数组等等）或者 null；
  • “function”–函数。

typeof 操作符可以判断出 number、boolean、string、function 和 undefined

typeof 一个实际的应用（检查一个没有声明的变量，而不报错），而不是用来检查对象的类型。

instanceof 运算符

对于值类型，可以通过 typeof 判断，string/number/boolean，在引用类型时，返回值只有 object/function，如，typeof 对 array 和 object 的结果都是 object

语法：a instanceof constructor;
instanceof 运算符确定一个对象是否为某个构造函数的实例。

instanceof 操作符可以用来判断某个构造函数的 prototype 属性是否存在另外一个要检测对象的原型链上。也就是判断instanceof前面的对象是否是后面的类或对象的实例。

instanceof ：沿着实例的 proto，同时沿着构造函数的prototype，如果两条线能找到同一个对象，就返回true，否则返回false。

instanceof 比较内置类型

原始类型不能使用 instanceof 运算符判断类型。

对象原生toString

Object.prototype.toString的行为：取得对象的内部属性[[Class]]（[[]]用来表示外部不可直接访问的属性，称为“内部属性”），返回一个类似于”[object Array]”的字符串。
再配合call，取得对象的内部属性[[Class]]，然后把类型检测转化为字符串比较。

[[Class]] 的值只可能是下面字符串：Arguments, Array, Boolean, Date, Error, Function, JSON, Math, Number, Object, RegExp, String

为什么不用obj.toString()？
虽然Array继承自Object，有toString方法，但可能被改写，而 Object.prototype很少有人改

检测对象的类型，推荐使用 Object.prototype.toString 方法；

其它方法

ES5引入 Array.isArray()。

  • typeof运算符不能检测数组
  • ECMAScript 5 的 isArray函数是原生的检测方法，但低版本浏览器不支持
  • 对象自身的constructor属性和instanceof操作符虽然也能检测数组，但在frame中会产生问题
  • 对象原生toString检测也能检测数组

验证是否是数字

  • 判断Array使用Array.isArray(arr)

  • 判断null请使用 value === null

  • 判断全局变量是否存在用 typeof window.value === “undefined”

  • 函数内部判断某个变量是否存在用 typeof value === “undefined”
