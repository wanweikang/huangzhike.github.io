[笔记]-构造函数

2016-10-29

2016-11-15更新

构造函数：用来生成对象；通过new关键字调用；应该以大写字母开头。

var Person = function(name) {
    this.name= name || "jim";
    this.hello = function() {
        return this.name;
    }
};

var person = new Person; // new命令执行构造函数，所以后面可以带括号，也可以不带
var person1 = new Person(); // new执行构造函数，返回实例对象
person1.name; // jim

var person2 = new Person("a"); // 也可以接受参数

// person1与person2相互独立
person.hello === person2.hello; // false

// 没有new直接调用构造函数，构造函数是普通函数
var person3 = Person();
person.name // Uncaught TypeError: Cannot read property 'name' of undefined
// this这时代表全局对象
name // "jim"

保证构造函数与new命令一起使用：

  • 一个方法是在构造函数内部使用严格模式，use strict。严格模式中，函数内this不能指向全局对象，默认undefined，由于不允许对undefined添加属性，不加new调用会报错。
  • 另一个方法是在构造函数内部判断是否使用new命令，如果没有，则返回一个实例对象。
function Person(a) {
    if (!(this instanceof Person)) {
        return new Person(a);
    }
    this.say = a;
}

Person("jim").say // jim
(new Person("jim")).say // jim
  • 每个函数都有prototype（原型）属性，指向函数的原型（prototype）对象。
  • 默认所有原型（prototype）对象都有constructor（构造函数）属性，指向prototype属性所在函数。
Person.prototype.constructor === Person; // true
  • 实例对象person本身没有prototype属性，通过proto从构造函数Person的原型链上继承了constructor属性，指向构造函数Person：
person.prototype; // undefined
person.constructor === Person.prototype.constructor; // true
person.constructor === Person; // true
Object.getPrototypeOf(person) === Person.prototype; // true
person instanceof Person; // true
person instanceof Object; // true

构造函数内部return的如果是对象，返回return语句指定对象；否则，不管return语句，返回新创建的对象。

var Person = function (){
    return "jim";
};

(new Person()) === "jim"; // false

如果return跟this无关的新对象，返回这个新对象，而不是this对象。

var Person = function (){
    this.name= "a";
    return { name: "b" };
};

Person.prototype = {
    say: function() {
        console.log("p");
    }
};

(new Person()).name; // "b";

// 返回对象的原型没有指向Person.prototype
(new Person()).say; // "undefined"
// 构造函数的原型被指向到本来新创建的对象，而Person没有返回这个对象，因此创建的对象不能访问Person原型链上的属性

构造函数与普通函数区别：

  • 函数内部使用this，代表生成的实例对象。
  • new命令调用，让构造函数Person生成实例对象，赋给person。
  • 实例对象从构造函数继承了name属性。

原型链：person –> Person.prototype –> Object.prototype –> null

new的原理：new不是正常的函数调用，先创造一个空对象，作为上下文对象，将这个空对象传入apply。this指向新生成的空对象。

function Person(name) {
    this.name= name || "jim";
};

// 模拟一：
function New(Func) {
    var f = {'__proto__': Func.prototype}; // 创建实例，继承原型
    return function(){
        Func.apply(f, arguments); // 调用Func，this指向实例
        return f; // 返回实例
    }
}
var person = New(Person)("jim");

// 模拟二：
function _new (constructor, param) {
    var args = [].slice.call(arguments); // 获得传入参数并分割保存在数组中
    var constructor = args.shift(); // 返回第一个参数，即构造函数
    var context = Object.create(constructor.prototype); // 创建新对象，继承原型
    var result = constructor.apply(context, args); // 执行构造函数，this指向创建的新对象  
    return (typeof result === 'object' && result != null) ? result : context; // 返回新对象
}

var person = _new(Person, "jim");

构造函数缺点：

  • 所有实例对象都拥有构造函数内的属性，但实例之间无法共享属性。
  • 每个方法都要在每个实例上重建，性能低。
