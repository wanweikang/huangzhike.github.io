switch语句

switch进行相同判断，是===的比较，连类型一起比较

如果不写break，还将执行下面的case的语句，直到一个break。

switch语句部分和case语句部分，都可以使用表达式。

switch…case要求每一个case的最后一行必须是break语句，否则会接着运行下一个case。
这样不仅容易忘记，还会造成代码的冗长。
而且，switch…case不使用大括号，不利于代码形式的统一。
此外，这种结构类似goto语句，容易造成流程混乱，不符合面向对象编程原则。

遍历

有两种方式使用 in 操作符：

  • 单独使用 （单独使用时， in 操作符会在通过对象能够访问给定属性时返回 true ，无论该属性存在于实例中还是原型中。）
  • 在 for-in 循环中使用。

for in：可以用于遍历数组和对象

for(var key in arr|obj){}

  • 遍历数组时，key表示索引值
  • 遍历对象时，key表示属性名

forEach

回调函数中有2个参数，分别表示值和索引，这一点与jQuery的 $.each相反

  • forEach无法遍历对象
  • forEach在低版本浏览器无法使用（ES5）
  • forEach无法使用 break，continue 跳出循环，使用 return 时，和在 for 循环中使用 continue 一致
  • forEach可以添加第二参数，为一个数组，回调函数中的 this 指向这个数组。而如果没有第二参数，则 this 指向 window

do/while：不建议使用do/while的遍历数组

$.each：用来遍历数组和对象

$(selecter).each：用来遍历DOMList
