[笔记]-编写规范，可维护代码

2016-10-31 12:18

2016-11-21重新整理更新

  • 缩进

    • 早期语言规范，大多数选用Tab作为缩进标记，如Delphi、Microsoft C++等。
    • 近年来，空格缩进由于风格不变性，逐渐成为主流，如Python、Ruby甚至C#。
    • 两空格为Node.js缩进，不同于常见的四空格缩进， Node.js易写出深层的函数嵌套。
    • 我就喜欢Tab。

  • 引号

    • JavaScript最外层统一单引号。

    • HTML内id、class等属性的定义，使用双引号。

    • CSS内url() 、属性选择符、属性值等使用双引号。

  • 注释

    • HTML注释

      • 模块注释

      • 区块注释

    • JavaScript注释

      • 单行注释：必须独占一行。//后跟一个空格，缩进与下行被注释代码一致。
      • 多行注释：避免用/*...*/多行注释。使用多个单行注释。
      • 函数/方法注释：
        • 必须包含函数说明，有参数和返回值时必须使用注释标识；
        • 参数和返回值注释必须包含类型信息和说明；
        • 函数是内部函数，外部不可访问时，可以使用@inner标识；

    • CSS注释

      • 注释统一用/* */，（scss中也不要用//）；

      • //可位于代码行的末尾，与代码间隔一个空格。

  • 空格

    • JavaScript

      • one space before {；
      • one space between function parameters；
      • no space before (；

    • CSS

      • 以下不要空格：
        • 属性名后；
        • 多个规则的分隔符,前；
        • !important !后；
        • 属性值中(后和)前；
        • 行末不要有多余空格；
      • 以下要空格：
        • 属性值前；
        • 选择器>,+,~前后；
        • {前；
        • !important !前；
        • @else前后；
        • 属性值中的,后；
        • 注释/后和/前；

  • 换行:

    • JavaScript
      • 以下要空行：
        • 变量声明后（当变量声明在代码块的最后一行时，则无需空行）；
        • 注释前（当注释在代码块的第一行时，则无需空行）；
        • 代码块后（在函数调用、数组、对象中则无需空行）；
        • 文件最后保留一个空行；
      • 换行的地方，行末必须有,或运算符；
      • 以下不要换行：
        • 下列关键字后： else , catch , finally；
        • 代码块{前；
      • 以下要换行：
        • 代码块{后和}前；
        • 变量赋值后；
          CSS

  • 分号

    • JavaScript：分号表示语句的结尾，一些结构不需要在语句的结尾添加分号，主要以下三种：

      • for和while循环

      • 分支语句：if， switch， try

      • 函数声明语句

      • 以上三种情况用了分号不会出错，解释引擎把这个分号解释为空语句。

    • CSS
      • 每个属性声明末都要加分号。

    • 占用多行时，缩进2个空格：

  • 命名

    • Javascript

      • 变量命名：驼峰式命名（除了对象的属性，主要考虑到cgi返回数据）：var thisIsMyName;

      • 常量全大写，下划线连接：var MAX_COUNT = 10;；

      • 构造函数，大写第一个字母；
      • jQuery对象必须以$开头命名：var $body = $('body');；
      • 项目命名：全部小写，下划线分隔：my_project_name；
      • 目录命名：有复数结构时，复数命名法：scripts，styles，images；
      • JS文件命名：account_model.js
      • CSS, SCSS文件命名：retina_sprites.sCSS
      • HTML文件命名：error_report.HTML
      • 私有属性、变量和方法以下划线 _ 开头：var _privateMethod = {};
      • 函数，函数的参数，用Camel命名法function stringFormat(theBells) {}
      • 类名，用名词：function Engine(options) {}
      • 函数名，使用动宾短语：function getStyle(element) {}
      • boolean类型的变量用 is 或 has 开头：var isReady = false, hasMoreCommands = false;

    • CSS

      • class类名小写，中划线分隔：.element-content，以功能或内容命名，不以表现命名；
      • 唯一的id作为Javascript hook，驼峰式命名，避免无样式信息的class：#myDialog；
      • scss的变量、函数、混合、placeholder用驼峰命名：$colorBlack，@function pxToRem($px)，@mixin centerBlock，%myDialog；

  • 组件命名（Naming components）：Components最少以两个单词命名，通过-分离：

    • 点赞按钮：.like-button；
    • 搜索框：.search-form；
    • 文章卡片：.article-card；

  • 元素命名（Naming elements）：Elements是Components的元素，Elements类名尽可能仅有一个单词：

    • 多个单词（On multiple words）：两个或以上单词的Elements类名，不应用中划线和下划线，应直接连接：

    • 避免标签选择器（Avoid tag selectors）：尽可能用classname，标签选择器性能上稍弱，并且表意不明。

  • 变体命名（Naming variants）：Components和Elements的Variants的classname应带有前缀中划线-；

  • Components应该可以复用，避免设置以下属性：

    • Positioning (position, top, left, right, bottom)；
    • Floats (float, clear)；
    • Margins (margin)；
    • Dimensions (width, height)；
    • Fixed dimensions （固定尺寸）。
    • 头像logos 应设置固定尺寸（宽，高…）；
    • Define positioning in parents （应该在父元素中设置定位）。

  • 避免过分嵌套（Avoid over-nesting）：保持不超过一个嵌套。

  • 顾虑（Apprehensions）：

    • 中划线-可选择性的使用，只要将Components，Elements，Variants记在心上即可；
    • 有时候想不出两个单词：有些组件使用一个单词就能表意，其实可以使用后缀，使其意识更加明确；
      • 如块级元素：.alert-box，.alert-card，.alert-block；
      • 或行内元素：.link-button，.link-span；

  • 总结（Summary）

    • Components，以两个单词命名（ .screenshot-image ）；
    • Components的Elements，以一个单词命名（ .blog-post .title ）；
    • Variants，以中划线 - 为前缀（ .shop-banner.-with-icon ）；
    • Components，可以互相嵌套；
    • 使用继承更简单；

  • 一律使用小写字母

  • 省略外链URL的http/https协议，使URL成为相对地址，避免Mixed Content，其它协议（ftp等）不省略：

  • 使用最少的标签并保持最小复杂度。

  • HTML属性应该按照特定的顺序出现。

  • 避免选择器嵌套过多；

  • 避免选择器和Class、ID叠加使用；

  • 语义嵌套约束：如a不允许嵌套div （容错处理）。

    • <li>用于<ul>或<ol>下；
    • <dd>,<dt>用于<dl>下；
    • <thead>,<tbody>,<tfoot>,<tr>,<td>用于<table>下；

  • 严格嵌套约束：如a不允许嵌套（所有浏览器都不允许） 。

    • inline-Level元素，仅可包含文本或其它inline-Level元素;
    • <a>不可嵌套交互式元素<a>、<button>、<select>等;
    • <p>不可嵌套块级元素<div>、<h1>~<h6>、<p>、<ul>/<ol>/<li>、<dl>/<dt>/<dd>、<form>等。

  • HTML5 规范中 disabled 、 checked 、 selected 等属性不用设置值。

  • 媒体查询放在相关规则附近，不要打包放在单一样式文件中或文档底部。如果分开了，将来会被遗忘。

  • 其它：

    • 结构、样式、行为分离：文档和模板只包含 HTML ，样式放到样式表里，行为放到脚本里。
    • 统一缩进，不要使用 Tab 或者 Tab 、空格混搭。
    • 不允许空的规则；
    • 元素选择器用小写字母；
    • 去掉小数点前面的0；
    • 去掉数字中不必要的小数点和末尾的0；
    • 属性值’0’后面不要加单位；
    • 无前缀的标准属性应该写在有前缀的属性后面；
    • 不要在同个规则里出现重复的属性，如果重复的属性是连续的则没关系；
    • 不要在一个文件里出现两个相同的规则；
    • 用 border: 0; 代替 border: none; ；
    • 选择器不要超过4层（在sCSS中如果超过4层应该考虑用嵌套的方式来写）；
    • 发布代码中不要有 @import ；
    • 少用’*’选择器。
    • 链接的样式顺序：a:link -> a:visited -> a:hover -> a:active（LoVeHAte）

  • 声明块格式

    • 选择器分组时，保持独立的选择器占用一行；
    • 声明块的左括号 { 前添加一个空格；
    • 声明块的右括号 } 应单独成行；
    • 声明语句中的 : 后应添加一个空格；
    • 声明语句应以分号 ; 结尾；
    • 一般以逗号分隔的属性值，每个逗号后应添加一个空格；
    • rgb() 、 rgba() 、 hsl() 、 hsla() 或 rect() 括号内的值，逗号分隔，但逗号后不添加一个空格；
    • 对于属性值或颜色参数，省略小于 1 的小数前面的 0 （例如， .5 代替 0.5 ； -.5px 代替 -0.5px）；
    • 十六进制值应该全部小写和尽量简写，例如， #fff 代替 #ffffff ；
    • 避免为 0 值指定单位，例如，用 margin: 0; 代替 margin: 0px; ；

  • 声明顺序：相关属性应为一组

    • Positioning：可从正常文档流中移除元素，还能覆盖盒模型（box model）相关样式，因此排首位；
    • Box model：决定了组件的尺寸和位置；
    • Typographic：其他属性只影响组件的内部或不影响前两组属性；
    • Visual；
