[笔记]-十、函数参数

2016-10-26 18:45

定义函数的时候，参数在小括号里面，这个参数叫做形参。
调用函数的时候，参数传在小括号里面，这个参数叫做实参。
JavaScript不要求形参数量和实参数量一样多。
在Java中，同一个函数名，但是参数个数不一样，视为两个函数（重载）。
JavaScript没有重载。

定义的时候和调用的时候参数个数可以不一样

如果参数同名，则取最后那个。

创建函数时会有一个类数组 arguments 对象。arguments 有三个属性，即 callee（调用方法），length，和caller（调用函数的参考）。

arguments.callee是一个指针，指向拥有 arguments 对象的函数。该属性常用于解除函数执行与函数名之间的强耦合。
arguments.caller返回调用当前函数的函数的引用。

通过 var 关键字定义 arguments 或者将 arguments 声明为一个形式参数， 都将导致原来的 arguments 被覆盖。

arguments 变量不是一个数组（Array）。 尽管有数组类似的属性 length，但它不从 Array.prototype 继承，它是一个对象（Object）。
因此，无法对 arguments 使用标准的数组方法，比如 push, pop 或者slice以及forEach。

但 arguments 可以像数组一样，用在某些只用于数组的方法。比如，用在apply方法中，或使用concat方法完成数组合并。

虽然可以使用 for 循环遍历，但为了更好的使用数组方法，最好把它转化为一个真正的数组。

自动更新
改变形参的值会影响到 arguments 对象的值，反之亦然。

arguments 的 getters 和 setters 方法在严格模式（strict mode）不会被创建。
arguments.callee会显著影响 JavaScript 的性能。建议不要使用 arguments.callee 和它的属性。
在严格模式（strict mode），arguments.callee 会报错 TypeError，因为它已经被废除了。

arguments 可以模拟函数的重载，使得同一个函数，根据参数个数的不同，有不同的作用。
如果传进来一个参数，就得到这个数字的加1；如果是2个参数，那么返回两个数字的和。

函数是一个引用类型
基本类型：number、string、boolean、undefined、null
引用类型：object、function、array、RegExp、Math、Date。

值类型的类型判断用 typeof，引用类型的类型判断用 instanceof。

在jQuery中，“jQuery”或者“$”，是一个函数

基本类型保存值，引用类型保存地址

五种基本数据类型是按值访问的，可以操作保存在变量中的实际值。
引用类型的值是按引用访问的，引用类型的值是保存在内存中的对象。
当复制保存着对象的某个变量时，操作的是对象的引用。但在为对象添加属性是，操作的是实际的对象。
对于引用类型的值，我们可以为其添加属性和方法，也可以改变和删除其属性和方法。
但不能给基本类型的值添加属性，尽管这样做不会抛出错误。
复制引用操作，两个变量实际上将引用同一个对象。因此，改变其中一个变量，就会影响另一个变量。
ECMAScript中所有参数都是按值传递的，不可能通过引用传递参数。
当我们给函数传入参数时，实际就是把函数外部的值复制给函数内部的参数，这和把值从一个变量复制到另一个变量一样。

JavaScript的函数参数是传值传递（passes by value），在函数体内修改参数值，不会影响到函数外部。

函数内部修改原始类型和复合类型的参数值，都不会影响到函数外部。

虽然参数本身是传值传递，但是复合类型的变量属性值是传址传递（pass by reference）。
在函数体内修改复合类型变量的属性值，会影响到函数外部。

如果需要对某个变量达到传址传递的效果，可以将它写成全局对象的属性。
