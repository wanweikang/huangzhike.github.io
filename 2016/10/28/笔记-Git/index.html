[笔记]-Git

2016-10-28 12:18

最近换到了Hexo搭博客，要用到Git，再加上要管理在Github上的resume和作品，折腾一晚上顺便把Git基础操作给学了，其它命令等需要用再现学吧。

注：我主要看了廖雪峰的Git教程，网上又搜了一大堆补充，写了这个笔记。听说还有Git GUI，TortoiseGit，TortoiseHg，SVN什么的，不明觉厉。

注：创建、合并和删除分支非常快，Git鼓励使用分支完成任务，合并后再删掉分支，这和直接在master分支上效果一样，但更安全。

注：当Git无法自动合并分支时，必须先解决冲突。解决冲突后，再提交，合并完成。

多人协作模式：
segmentfault
首先用git push origin branch-name推送自己的修改；

如果推送失败，则因为远程分支比本地更新，先用git pull试图合并；

如果合并有冲突，则解决冲突，并在本地提交；

解决掉冲突后，再用git push origin branch-name推送就能成功。

如果git pull提示“no tracking information”，说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream branch-name origin/branch-name。

Git从远程的分支获取最新版本到本地有2个命令：

git fetch：从远程获取最新版本到本地，不会自动merge

更清晰的方式：

git pull：相当于从远程获取最新版本并merge到本地
