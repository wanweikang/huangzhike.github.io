[笔记]-函数参数

2016-10-28

2016-11-18重新整理更新

  • Java中，同函数名，参数个数不一样，视为两个函数（重载）。
  • JavaScript没有重载。
  • 定义函数时，参数在小括号里，叫形参。
  • 调用函数时，参数在小括号里，叫实参。
  • JavaScript不要求形参数量和实参数量一样多，定义和调用的参数个数可以不一样。
function sum(a, b) {
    console.log(a + b);
}

sum(10); // NaN，b为undefined。10+undefined就是NaN
sum(10, 20, 32, 23); // 30

如果参数同名，取最后那个。

function f(a, a) {
    console.log(a);
}

f(1, 2); // 2
f(1); // undefined
  • 创建函数时有一个类数组arguments对象。arguments有三个属性，callee（调用方法），length，和caller（调用函数的参考）。
  • arguments.callee指向拥有arguments对象的函数，常用于解除函数执行与函数名之间的强耦合。
  • arguments.caller返回调用当前函数的函数的引用。
var f = function(one) {
    console.log(arguments.callee === f); // true
}
  • var关键字定义arguments或将arguments声明为形参，都导致原来的arguments被覆盖。
  • arguments不是数组，尽管有数组类似的属性length，但它不从Array.prototype继承，它是一个对象（Object）。
  • 无法对arguments使用标准的数组方法，如push, pop或slice及forEach。
  • arguments可以用在某些数组方法。如，用在apply方法中，或用concat方法数组合并。
func.apply(obj, arguments);
Array.prototype.concat.apply([1,2,3], arguments);

虽然可以用for循环遍历，但为了使用数组方法，最好把转化为真正的数组。

Array.prototype.slice.call(arguments); 
// or 
var args = [];
for(var i = 0; i < arguments.length; i++) {
    args.push(arguments[i]);
}

改变形参的值会影响到arguments对象的值，反之亦然。

function foo(a, b, c) {
    arguments[0] = 2;
    a; // 2
    b = 4;
    arguments[1]; // 4
    var d = c;
    d = 9;
    c; // 3
}

foo(1, 2, 3);
  • arguments的getters和setters方法在严格模式（strict mode）不会被创建。
  • arguments.callee会显著影响性能，不建议使用arguments.callee和它的属性。
  • 严格模式（strict mode），arguments.callee会报错TypeError，因为它已经被废除了。
  • arguments可以模拟函数的重载，使同一个函数，根据参数个数的不同，有不同的作用。
  • 如果传进来一个参数，就得到这个数字的加1；如果是2个参数，那么返回两个数字的和。
sum(10); // 1
sum(3, 4); // 7

function sum(a, b) {
    switch(arguments.length) {
        // 如果实际参数的长度是1
        case 1:
            return ++a;
            break;
        case 2:
            return a + b;
            break;
    }
}

函数通常使用函数声明语法定义，也可以用函数表达式定义，还可以用Function构造函数。

Function构造函数接收任意数量的参数，但最后一个参数始终是函数体，前面的参数是新函数的参数：

var sum = new Function("num1", "num2", "return num1+num2");  // 不推荐

函数是一个引用类型。

var fn = function () {};
fn instanceof Object; // true
fn instanceof Function; // true

jQuery中，“jQuery”或“$”是一个函数。

typeof ($); // function
$.trim(" ABC "); // $或jQuery函数有trim属性，属性值是函数，截取前后空格

ECMAScript中所有参数都是按值传递的，不可能通过引用传递参数。

给函数传入参数，就是把函数外的值复制给函数内的参数，等于把值从一个变量复制到另一个变量。

var foo = {
    bar: function() {
        return this.baz;
    },
    baz: 1
}; 

(function() {
    return typeof arguments[0]();
})(foo.bar); // "undefined"

JavaScript函数参数传值传递（passes by value），在函数体内修改参数，不会影响到函数外部。

var p = 2; 

(function (p) {
    p = 3; // 修改原始类型的参数值
}(p);

p // 2

var a = [1, 2, 3];

(function (a) {
    a = [2, 3, 4]; // 修改复合类型的参数值
})(a);

a // [1, 2, 3]

函数内部修改原始类型和复合类型的参数值，都不会影响到函数外部。

虽然参数本身是传值传递，但是复合类型的变量属性值是传址传递（pass by reference）。

在函数体内修改复合类型变量的属性值，会影响到函数外部。

var o = { p:1 };

(function (obj) {
    obj.p = 2; // 修改对象的属性值
})(o);

o.p // 2

var a = [1, 2, 3];

(function f(a) {
    a[0]=4; // 修改数组的属性值
})(a);

a // [4, 2, 3]

如果需要对某个变量达到传址传递的效果，可以将它写成全局对象的属性。

var a = 1;

(function (p) {
    window[p]=2;
})("a");
// 变量a本来是传值传递，但是写成window对象的属性，就达到了传址传递
a // 2

设计一个函数，接收三个参数，返回前两个数字大的数字，与第三个数字的和。如sum(4, 7, 9);

function sum(a, b, c) { 
    return a >= b? a+c : b+c;
}

函数调用自身，就是递归（recursion）。

function fib(num) {
    if (num > 2) {
        return fib(num - 2) + fib(num - 1);
    } else {
        return 1;
    }
}
function factorial(num) {
    if (num<=1) {
        return 1;
    } else {
        return num*arguments.callee(num-1);
    }
}
var factorial = (function f(num) {
    if (num<=1) {
        return 1;
    } else {
        return num*f(num-1);
    }
});
function factorial(x) {
    return x == 0 ? 1 : x * factorial(x-1);
}

把多参数的函数变成单参数的函数。

function p(n) {
    return function (x) {
        return Math.pow(x, n);
    }
}

var pow2 = p(2);
var pow3 = p(3);

pow2(5); // 25
pow3(7); // 343
