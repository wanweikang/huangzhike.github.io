[笔记]-声明&提升

2016-10-28

2016-11-18重新整理

声明提升（Hoisting）：var表达式和function声明都被提升到当前作用域的顶部。

var a = "g";

(function test() {
    console.log(a); // undefined
    var a = "l";
    console.log(a); // l
})();
// 等于
var a = "g";

(function test() {
    var a = undefined;
    console.log(a);
    a = "l";
    console.log(a);
})();
a = 2;
var a;

console.log(a); // 2
// 等于
var a;
a = 2;

console.log(a);

没有块级作用域不仅导致var表达式从for循环内移到外部，if表达式看起来修改了全局变量a，实际在提升后，在修改局部变量。

function test(data) {
    if (false) {
        a = 1;
    } else {
        var a = 2;
    }
    for(var i = 0; i < 100; i++) {
        var b = i;
    }
}
// 等价于
function test(data) {
    var a, b, i; // 没有块级作用域，变量被移动到函数顶部
    if (false) {
        a = 1;
    } else {
        a = 2;
    }
    for(i = 0; i < 100; i++) {
        b = i;
    }
}
// 检查init是否已被初始化
if (! init) {
    var  init = {}; // 正常运行，var表达式被提升到作用域顶部
}

由于函数名的提升，在条件语句中声明函数是无效的。

if (false) {
    function f() {}
}

f(); // 不报错

要在条件语句中定义函数，只有使用函数表达式。

if (false) {
    var f = function () {};
}

f(); // undefined

函数首先被提升，然后是变量。

foo(); // 1

var foo=2;

function foo() {
    console.log(1);
}
// 函数声明会被提升，函数表达式不会
var foo = function() {
    console.log(3); 
};

尽管var foo出现在function foo() {}之前，但它是重复的声明（因此被忽略了），因为函数声明会被提升到普通变量之前。

尽管重复的var声明会被忽略掉，但出现在后面的函数声明是可以覆盖前面的。

foo(); // 3

function foo() {
    console.log(1);
}

var foo = function() {
    console.log(2);
};

function foo() {
    console.log(3);
}

foo (); // 2

JavaScript引擎将var a = 2;看作var a和a = 2，第一个是编译阶段，第二个是执行阶段。

无论作用域中的声明出现在什么地方，都将在代码被执行前先处理。

所有的声明（变量和函数）都会被“移动”到各自作用域的最顶端，这个过程被称为提升。

声明本身会被提升，而包括函数表达式的赋值在内的赋值操作不会提升。

注意避免重复声明，特别是var声明和函数声明混合在一起。

var foo = 1; 

function foo() {
    console.log(2);
}

foo(); // foo is not a function(…)

// 在var foo = 1之前，已把 function foo()预解析了，之后又定义了变量foo
foo(); // foo is not a function(…)

var foo = 1;

function foo() {
    console.log(2);
}

函数声明

foo(); // 正常，foo在代码运行前已经被创建
function foo() {} // 在执行前被 解析（hoisted），它存在于当前上下文的任意一地方，即使在定义的上面被调用

函数赋值表达式

foo(); // 出错：TypeError
var foo = function() {};  // 把一个匿名的函数赋值给变量 foo

命名函数的赋值表达式

// 将命名函数赋值给一个变量
var foo = function bar() {
    bar(); // 正常运行（函数名在函数内总是可见的）
}
bar(); // ReferenceError，bar函数声明外不可见，因为已把函数赋值给了foo（在IE8及以下在外部可见））

方法的赋值表达式

var test = obj.method; // 将一个方法赋值给一个变量
test(); // test 就像普通函数被调用；函数内的this不被指到obj对象
(function() {
    var a = b = 3;
})();

console.log(b); // 3
console.log(typeof a); // undefined

赋值从右到左，首先，是赋值表达式b = 0。这个表达式的返回值是0，赋给var定义的局部变量a。

var a = (b = 0);

所以b成了全局变量，而a是自执行函数的一个局部变量。
