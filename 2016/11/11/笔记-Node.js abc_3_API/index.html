Buffer(缓冲区)

  • JavaScript 语言自身只有字符串数据类型，没有二进制数据类型。
  • 但在处理像TCP流或文件流时，必须使用到二进制数据。
  • 因此在 Node.js中，定义了一个 Buffer 类，用来创建一个专门存放二进制数据的缓存区。
  • 需要在 Node.js 中处理I/O操作中移动的数据时，就有可能使用 Buffer 库。
  • 原始数据存储在 Buffer 类的实例中。 Buffer 类似整数数组，但对应于 V8 堆内存外的一块原始内存。

fs模块：文件系统模块，封装了文件的读取、写入、更名、删除、遍历目录、链接等 POSIX 文件系统操作。提供了异步如 fs.readFile()和同步如fs.readFileSync()方法。

  • 不传入文件编码时，回调函数的data参数将返回一个Buffer对象。

  • Node.js中，Buffer对象就是一个包含零个或任意个字节的数组（和Array不同）。

  • fs.open(path, flags, [mode], [callback(err, fd)]) ： POSIX open 函数的封装，与 C 语言标准库中的 fopen 函数类似。

  • fs.read(fd, buffer, offset, length, position, [callback(err, bytesRead,buffer)])： POSIX read 函数的封装，提供了更底层的接口。
  • fs.writeFile()：将数据写入文件。
  • fs.stat()：获取文件信息。

Stream是Node.js提供的仅在服务端可用的模块，目的是支持“流”这种数据结构。

  • Stream是抽象接口，对http服务器发起请求的request对象就是一个Stream。
  • 可以把数据看成数据流，敲键盘时把每个字符连起来，看成字符流。这个流从键盘输入到应用程序：标准输入流（stdin）。
  • 如果应用程序把字符一个一个输出到显示器上，这也可以看成是一个流：标准输出流（stdout）。
  • 流的特点是数据是有序的，而且必须依次读取，或者依次写入，不能像Array那样随机定位。
  • 有些流用来读取数据，从文件读取数据时，打开一个文件流，然后从文件流中不断读取数据。
  • 有些流用来写入数据，向文件写入数据时，把数据不断地往文件流中写进去。
  • 所有的 Stream 对象都是 EventEmitter 的实例：
    • data事件：当有数据可读时触发
    • end事件：没有更多的数据可读时触发
    • error事件：出错
    • finish事件：所有数据已被写入到底层系统时触发。
  • Stream 有四种流类型：
    • Readable：可读操作。
    • Writable：可写操作。
    • Duplex：可读可写操作.
    • Transform：操作被写入数据，然后读出结果。

Node.js 使用事件驱动模型，webserver一直接受请求而不等待任何读写操作（被称为非阻塞式IO或事件驱动IO）。

  • 事件驱动模型中，会生成一个主循环来监听事件，当检测到事件时触发回调函数。
  • 事件驱动的流程类似观察者模式，事件相当于主题(Subject)，所有注册到这个事件上的处理函数相当于观察者(Observer)。
  • Node.js 所有的异步 I/O 操作在完成时都会发送一个事件到事件队列。

  • Node.js里面的许多对象都会分发事件：

    • net.Server对象会在每次有新连接时分发一个事件，
    • fs.readStream对象会在文件被打开的时候发出一个事件。
    • 所有这些产生事件的对象都是 events.EventEmitter 的实例。

  • Node.js 有多个内置的事件，通过引入 events 模块，并实例化 EventEmitter 类来绑定和监听事件；

  • events 模块只提供了一个对象： events.EventEmitter，核心就是事件触发与事件监听器功能的封装。

EventEmitter 常用API

  • EventEmitter.on(event, listener) 为指定事件注册监听器，接受一个字符串 event 和一个回调函数 listener 。
  • EventEmitter.emit(event, [arg1], [arg2], […]) 触发 event 事件，传递若干可选参数到事件监听器的参数表。
  • EventEmitter.once(event, listener) 为指定事件注册单次监听器，监听器只触发一次，触发后立刻解除该监听器。
  • EventEmitter.removeListener(event, listener) 移除指定事件的某个监听器， listener 必须是该事件已经注册过的监听器。
  • EventEmitter.removeAllListeners([event]) 移除所有事件的所有监听器，如果指定 event ，则移除指定事件的所有监听器。
  • EventEmitter 定义了 error 事件，在遇到异常的时候通常会发射 error 事件。
  • 当 error 被发射时， EventEmitter 如果没有响应的监听器，Node.js 会把它当作异常，退出程序并打印调用栈。
  • 一般要为会发射 error事件的对象设置监听器，避免遇到错误后整个程序崩溃。

继承 EventEmitter

  • 大多数时候不会直接使用 EventEmitter ，而是在对象中继承它。包括 fs 、 net 、http 在内，只要支持事件响应的核心模块都是 EventEmitter 的子类。
