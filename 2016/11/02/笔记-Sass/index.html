Sass（预处理器）最初特征缩进敏感，第三代开始，放弃缩进式风格，完全兼容普通css代码，这一代Sass被称为SCSS（Sassy CSS）。Sass提供两种不同的语法：Sass（缩进语法）和 SCSS。Sass 的空白敏感语法通过缩进摆脱大括号、分号和其他。SCSS 更易学习。

  • 定义在任何形式的{…}块内变量只能在此规则块内使用（如@media或@font-face）。
  • 中划线声明的变量可以用下划线方式引用，反之亦然。
  • 反复声明一个变量，最后一处声明有效且覆盖前边。
  • !default用于变量：如果这个变量被赋值了，就用声明的值，否则就用这个默认值。

Sass有两种注释方式：

  • 标准CSS注释 /* comment */，会保留到编译后的文件。
  • 单行注释（静默注释）// comment，只保留在Sass源文件中，编译后被省略。

混合器：可重用的代码块。变量处理几处小小的样式类似。通过sass混合器实现大段样式的重用。

@mixin声明混合，可以传递参数，参数名以$符号开始，多个参数以逗号分开，也可以给参数设置默认值。

导入外部文件：@import "path/filename.scss";可以省略.sass或.scss后缀。基础的文件命名方法以_开头，如_mixin.scss，导入时可以不写下划线：@import "mixin"。编译时会将@import的scss文件合并生成一个CSS文件。另外，所有在被导入文件中定义的变量和混合器均可在导入文件中使用。

Sass部分文件：Sass局部文件的文件名以下划线开头。这样，Sass就不会在编译时单独编译这个文件输出css，而只用作导入。当@import一个局部文件时，可以省略文件名开头的下划线。导入themes/_night-sky.scss，只需@import``”themes/night-sky”;。

嵌套导入：跟原生css不同，Sass允许@import命令写在css规则内。这种导入方式下，生成对应的css文件时，局部文件被直接插入到css规则内导入它的地方。

原生CSS导入：由于Sass兼容原生CSS，所以也支持原生的CSS@import。导入的css文件不会合并到编译后的文件，而是以@import方式存在。造成浏览器解析css时额外下载，下列三种情况下等同于CSS的import命令：

  • 被导入文件的名字以.css结尾，如@import 'reset.css'；
  • 被导入文件的名字是一个URL地址（如http://www.Sass.hk/css/css.css），由此可用谷歌字体API提供的相应服务；
  • 被导入文件的名字是CSS的url()值。

因为Sass的语法完全兼容css，把原始css文件改名为.scss后缀，即可直接导入。

SASS允许在代码中使用算式，对数值型的Value(如：数字、颜色、变量等)进行加减乘除四则运算。注意运算符前后留一个空格，不然会出错。

@for循环有两种形式，分别为：@for $var from through 和@for $var from to 。区别是through表示包括end，而to则不包括end。

@each循环：@each $var in 。list和map表示list类型数据和map类型数据。

CSS 不要求字符串必须用引号包裹，甚至字符串中包含空格。font-family属性无论是否使用引号包裹，都可以正确解析。Sass 也不强制要求字符串必须被引号包裹。被引号包裹和没被包裹的一对字符串完全等同（如，’abc’ 严格等同于 abc）。但是，不强制要求字符串被引号包裹的毕竟是少数，所以，在 Sass 中字符串应该始终被单引号'包裹。

  • 如果颜色名不被引号包裹，会被解析为颜色值；
  • 大多数的语法高亮机制将会因未被引号包裹的字符串而失效；
  • 提高可读性；
  • 没有正当理由不去用引号包裹字符串。

CSS 中类似 initial 或 sans-serif 的专用名词无须引用起来。font-family: ‘sans-serif’ 是错误的，因为 CSS 希望获得的是一个标识符，而不是一个字符串。

如果字符串内包含了一个或多个单引号，使用双引号包裹整个字符串，避免使用过多的转义字符。

将一个单位添加给数字的时候，实际上是让该数值乘以1个单位。

最高级运算应该始终被包裹在括号中。

在逗号后面追加一个空格，不在前后括号和值之间加空格。

  • 冒号:之后添加空格；
  • 左开括号(要和冒号:写在同一行；
  • 如果键是字符串（99% 都是字符串），则使用引号包裹起来。
  • 每一个键值对单独一行；
  • 每一个键值对以逗号,结尾；
  • 最后一个键值对添加尾部逗号,，方便添加新键值对、删除和重排已有键值对；
  • 单独一行书写右闭括号 )；
  • 右闭括号)和分号;之间不用空格和换行。
  • 相关联的选择器写在同一行；不相关联选择器分行书写；
  • 最后一个选择器和左开大括号{中间书写一个空格；
  • 每个声明单独一行；
  • 冒号:后添加空格；
  • 所有声明的尾部都添加一个分号;；
  • 右闭大括号}单独一行；
  • 右闭大括号}添加空行。
  • 本地变量在其他任何变量之前声明，并和其他声明用空行分隔开；
  • 不需@content的混合宏在放在其他声明之前；
  • 嵌套选择器在新行声明；
  • 需要@content的混合宏在嵌套选择器后声明；
