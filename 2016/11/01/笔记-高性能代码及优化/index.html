[笔记]-高性能代码及优化

2016-11-01 18:20

全局变量：变量定义在全局作用域中，引擎默认不会将其回收，可能存活到进程退出，如果对象很大，比如在JavaScript中做模版渲染。
尽量选用局部变量而不是全局变量。局部变量的访问速度比全局变量的快，因为全局变量是window对象的属性，而局部变量放在函数的栈里。缓存重复使用的全局变量。

全局变量缺点：不利于内存释放；多人协作时容易混淆；在作用域链中容易被干扰；需要搜索更长的作用域链。

位运算比布尔运算或算数运算快，如取模，逻辑与和逻辑或可以用位运算替换

使用运算符时，尽量使用 +＝， －＝、 *＝、\=等运算符号，而不是直接进行赋值运算。

&& 和 ||
二元布尔操作符是短路的, 只有在必要时才会计算最后一项

把数字转换成字符串：应用 ""+1，效率最高。

性能上来说： ""+字符串 > String() > .toString() > new String()。

String() 属于内部函数，所以速度很快。
.toString() 要查询原型中的函数，所以速度略慢。
new String() 最慢。

浮点数转换成整型。

parseInt() 将字符串转换成 数字 ，而不是浮点数和整型之间的转换。
应该使用 Math.floor() 或者 Math.round() 。
Math 是内部对象，所以 Math.floor() 其实并没有多少查询方法和调用的时间，速度是最快的。

  • 使用数组和对象字面量，避免使用构造函数 new Array()， new Object()，new function () { … } 。
  • 避免使用eval()或者函数构造器，开销较大，每次调用，JavaScript引擎都要转换为可执行的代码。
  • 避免使用with()，with会把变量加入到全局作用域中，如果有同名变量，容易混淆，值也会被覆盖。
  • 如果是不断运行的代码，不应用setTimeout ，而用setInterval 。setTimeout每次要重新设置定时器。
  • 当JavaScript代码解析JavaScript代码时会双重解释，在eval("sum+=num1;")、new Function("sum+=num1;")和setTimeout("alert('hello world');", 1000)等传入字符串情况下遇到。

访问变量或数组是O(1) 操作。访问对象上的属性是O(n) 操作。对象的属性查找都要比访问变量或数组花费时间，因为必须在原型链中对该名属性搜索，属性越多，时间越长。需要多次用到对象属性，应将其存储在局部变量。

能形成作用域（scope）的有：一、函数调用。二、with语句，with 会创建自已的作用域，因此会增加其中执行代码的作用域的长度。三、全局作用域。
减少作用域链上的查找次数，如果需要访问变量或函数，需要遍历当前执行环境的作用域链，从这个作用域链一级一级遍历，直到全局执行环境。

循环事件绑定、私有属性、含参回调等一定要使用闭包时，并谨慎对待其中的细节。
循环绑定事件，我们假设一个场景：有六个按钮，分别对应六种事件，当用户点击按钮时，在指定的地方输出相应的事件。

不合理的使用闭包会导致内存泄漏。闭包的性能不如使用内部方法，更不如重用外部方法。
由于 IE 9 浏览器的 DOM 节点作为 COM 对象来实现， COM 的 内存管理 是通过引用计数的方式，引用计数有个难题就是循环引用，一旦 DOM 引用了闭包，例如 event handler，闭包的上层元素又引用了这个 DOM ，就会造成循环引用从而导致内存泄漏。

当函数执行完毕时，在函数内部所声明的对象 不一定 就会被销毁。引用( Reference )是 JAVASCRIPT 引擎垃圾回收中最关键的一个机制。

当代码执行完毕时，对象 val 和 bar() 并没有被回收释放， 判断对象是否正在使用中的标准，就是是否仍然存在对该对象的引用 。
JAVASCRIPT 的 引用 是可以进行 转移 的，那么就有可能出现某些引用被带到了全局作用域，但事实上在业务逻辑里已经不需要对其进行访问了，这个时候就应该被回收，
但是 JAVASCRIPT 引擎仍会认为程序仍然需要它。IE 下闭包引起跨页面内存泄露。

选择器

  1. 尽可能的使用 ID 选择器，因为它会调用浏览器原生方法 document.getElementById 查找元素。当然直接使用原生 document.getElementById 方法性能会更好；
  2. 在父元素中选择子元素使用 .find() 方法性能会更好, 因为 ID 选择器没有使用到 Sizzle 选择器引擎来查找元素；

DOM 操作

  1. 当要操作 DOM 元素的时候，尽量将其分离节点，操作结束后，再插入节点；
  2. 使用字符串连接 array.join 要比 .append() 性能更好；

事件

  1. 如果需要，对事件使用自定义的 namespace ，这样容易解绑特定的事件，而不会影响到此 DOM 元素的其他事件监听；
  2. 对 Ajax 加载的 DOM 元素绑定事件时尽量使用事件委托。事件委托允许在父元素绑定事件，子代元素可以响应事件，也包括 Ajax 加载后添加的子代元素；

链式写法

  1. 尽量使用链式写法而不是用变量缓存或者多次调用选择器方法；
  2. 当链式写法超过三次或者因为事件绑定变得复杂后，使用换行和缩进保持代码可读性；

其他

  1. 多个参数使用对象字面量存储；
  2. 不要将 CSS 写在 jQuery 里面；
  3. 正则表达式仅准用 .test() 和 .exec() 。不准用 “string”.match() ；

异步加载第三方内容
当无法保证嵌入第三方内容比如 Youtube 视频可以正常工作的时候，需要考虑用异步加载这些代码，避免阻塞整个页面加载。

避免使用 jQuery 实现动画

  1. 禁止使用 slideUp/Down() fadeIn/fadeOut() 等方法；
  2. 尽量不使用 animate() 方法；

AJAX优化
缓存 AJAX ：
异步 并不等于 即时 。
请求使用 GET ：
当使用 XMLHttpRequest 时，而URL长度不到 2K ，可以使用 GET 请求数据， GET 相比POST 更快速。
POST 类型请求要发送两个 TCP 数据包。先发送文件头。再发送数据。
GET 类型请求只需要发送一个 TCP 数据包。取决于 cookie 数量。
