[笔记]-高性能代码及优化-CSS选择器

2016-11-01 18:20

2016-11-22重新整理更新

  • CSS 选择器对性能的影响源于匹配。CSS 选择器从右到左匹配；
  • 子选择器#header > a。浏览器遍历所有 a 元素并确定其父元素 id 是否为header ；
  • 如果改为后代选择器#header a则开销更多，遍历所有 a 元素后向其上级遍历直到根节点；

  • 最右边选择符称为关键选择器。

    • 避免通用选择器

    • 避免 tag 或 class 选择器限制 id 选择器

    • 避免 tag 限制 class 选择器

    • 避免多层选择器，用 class 选择器替换，减少查找

    • 避免子选择器

    • 使用继承
