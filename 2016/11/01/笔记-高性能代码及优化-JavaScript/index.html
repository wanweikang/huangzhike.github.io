[笔记]-高性能代码及优化-JavaScript

2016-11-01 18:20:07

2016-11-22重新整理更新

  • 引擎默认不会回收全局变量，可能存活到进程退出，如果对象很大：在JavaScript中做模版渲染。
  • 尽量用局部变量。局部变量访问速度比全局变量快，因为全局变量是全局对象的属性，而局部变量在函数的栈里。
  • 缓存重复使用的全局变量。

  • 全局变量缺点：不利于内存释放；多人协作时容易混淆；在作用域链中易被干扰；需要搜索更长的作用域链。

  • 首次为变量赋值时务必用var，未声明直接赋值，默认作为全局变量。

  • 变量声明放代码块头部。

  • 函数作用域中所有变量声明尽量提到首部，用一个var声明。

  • 最小化语句数，语句数量也会影响执行速度。

  • 不要在循环内声明函数。

  • 声明提升

  • 不要使用封装基本类型

  • 原始操作符比函数调用快

  • length属性清空数组，截断数组；

  • 直接从数组中delete或remove元素，其实并没有删除，只是置为undefined；
  • 数组元素删除用splice；

  • 删除对象的属性用delete；

  • 用===取代==，==和!=操作符在需要时自动转换数据类型。===和!==不会，同时比较值和数据类型，比==和!=快；

  • 不要直接用undefined判断，用typeof和字符串’undefined’判断

  • 位运算比布尔运算或算数运算快，如取模，逻辑与和逻辑或可用位运算替换；

  • 使用运算符时，尽量用 +＝， －＝、 *＝、\=等，而不是直接赋值运算。

  • 自增和自减运算符，放在变量的前面或后面，返回的值不一样，容易出错。

  • 三元操作符替代if语句，用于赋值或返回

  • &&和||，二元布尔操作符是短路的, 只有必要时才计算后一项；

  • 把数字转换成字符串： ""+1效率最高，性能上： ""+字符串 > String() > .toString() > new String()。

    • String() 属于内部函数，速度很快。
    • .toString() 要查询原型中的函数，速度略慢。
    • new String() 最慢。

  • 浮点数转换成整型。

    • parseInt()将字符串转换成数字 ，而不是浮点数和整型之间的转换。
    • 应用Math.floor()或Math.round() 。Math是内部对象，Math.floor()并没有多少查询方法和调用的时间，速度是最快的。

  • 不要在内置对象的原型上添加方法；

  • 不要在不需要的地方加括号；

  • 使用数组和对象字面量，避免构造函数 new Array()， new Object()，new function () {} 。

  • 避免eval()或函数构造器，开销较大，每次调用，JavaScript引擎都要转换为可执行的代码。
  • 避免with()，with把变量加入到全局作用域中，如果有同名变量，容易混淆，值也会被覆盖。
  • 不断运行的代码，不应用setTimeout ，而用setInterval 。setTimeout每次要重新设置定时器。

  • 当JavaScript代码解析JavaScript代码时会双重解释，在eval("sum+=num1;")、new Function("sum+=num1;")和setTimeout("alert('hello world');", 1000)等传入字符串情况下遇到。

  • 不需要属性访问方法，因为所有的属性都是外部可见的。添加属性访问方法只是增加了一层重定向，没有意义。

  • 临时存储用于计算和查询的变量；

  • 根据字符串、数组的长度循环，通常长度不变，每次要额外查询a.length ，应预先缓存；

  • 现代浏览器引擎会自动优化，但还有旧浏览器。

  • 推荐使用for循环；

  • 推荐用 -=1 减值迭代；

  • 不要对数组for-in循环，它不是从 0到 length - 1 遍历, 而是所有对象及其原型链的键值。

  • for-in里一定要有hasOwnProperty判断；

  • 不要循环中使用try-catch：

  • 能形成作用域（scope）的有：

    • 一、函数调用。
    • 二、with语句，with 会创建自已的作用域，因此会增加其中执行代码的作用域的长度。
    • 三、全局作用域。

  • 访问变量或数组是O(1) 操作。访问对象上的属性是O(n) 操作。

  • 对象的属性查找要比访问变量或数组花时间，因为必须在原型链中对该名属性搜索，属性越多，时间越长。
  • 需要多次用到对象属性，应将其存储在局部变量。

  • 减少作用域链上的查找次数，访问变量或函数，需要遍历当前执行环境的作用域链，一级一级直到全局执行环境。

  • 谨慎对待循环事件绑定、私有属性、含参回调等使用闭包。

  • 循环绑定事件：有六个按钮，对应六种事件，点击按钮时，在指定地方输出相应事件。

  • 不合理的闭包会导致内存泄漏。闭包性能不如内部方法，更不如重用外部方法。

  • IE 9 的 DOM 节点作为 COM 对象来实现， COM 的 内存管理是通过引用计数，引用计数有个难题就是循环引用，一旦 DOM 引用了闭包，闭包的上层元素又引用了这个 DOM ，就会造成循环引用导致内存泄漏。IE 下闭包引起跨页面内存泄露。

  • 函数执行完时，函数内所声明对象不一定销毁。引用( Reference )是 JavaScript 引擎垃圾回收中最关键的机制。

  • JavaScript 的内存泄露处理

    • 给 DOM 对象添加的属性是一个对象的引用。

      • 解决方法：window.onunload事件写上：document.getElementById('myDiv').myProp = null;

      • 解决方法：onunload事件写上：document.getElementById('myDiv').myProp = null;

      • 解决方法：onunload事件中写上：element.detachEvent('onclick', doClick);

  • 除了用闭包访问内部变量，还可以使用回调函数处理业务。

    • 回调函数是后续传递风格( Continuation Passing Style , CPS )的技术，相比闭包好处也有很多。
    • 如果传入的参数是基础类型，回调函数中传入的形参是复制值，业务代码使用完后，更容易被回收。
    • 回调除了可以完成同步外，还可以用在异步编程中。
    • 回调函数通常也是临时的匿名函数，一旦执行完毕，自身的引用就会被解除，得到回收。

  • 只在构造器或对象的方法(包括创建的闭包)中使用this。

  • 面向对象书写方式：使用匿名函数进行封装，提高重用性

  • 使用匿名函数包裹代码

  • 使用一个 timer 完成多个动画。

  • 使用事件代理：在每个元素绑定事件本身就对性能有一定损耗。

    • 由于DOM Level2事件模型中所有事件默认传播到上层文档对象，借助这个机制在上层元素注册一个统一事件处理不同子元素。

  • AJAX优化

    • 缓存 AJAX；
    • 请求使用 GET ：
      • 当 XMLHttpRequest 时，而URL长度不到 2K ，可以用 GET 请求数据， GET 相比POST 更快速。
      • POST 类型请求要发送两个 TCP 数据包。先发送文件头。再发送数据。
      • GET 类型请求只需要发送一个 TCP 数据包。取决于 cookie 数量。

  • 异步加载第三方内容：无法保证嵌入第三方内容如 Youtube 视频可以正常工作时，异步加载这些代码，避免阻塞页面加载。

  • 避免用 jQuery 实现动画

    • 禁止使用 slideUp/Down() fadeIn/fadeOut() 等方法；
    • 尽量不用 animate() 方法；

  • 尽可能用 ID 选择器，它会调用浏览器原生方法 document.getElementById ，当然直接用原生 document.getElementById 方法更好；

  • 在父元素中选择子元素用 .find() 方法性能更好, 因为 ID 选择器没有用到 Sizzle 选择器引擎查找元素；

  • 当操作 DOM 元素时，尽量将其分离节点，操作结束后，再插入节点；

  • 使用字符串连接 array.join 比 .append() 性能更好；

  • 如果需要，对事件使用自定义的 namespace ，这样容易解绑事件，而不影响到此 DOM 元素的其他事件监听；

  • 对 Ajax 加载的 DOM 元素绑定事件时用事件委托。事件委托允许在父元素绑定事件，子代元素响应事件，也包括 Ajax 加载后动态添加的子代元素；

  • 尽量使用链式写法而不是用变量缓存或者多次调用选择器方法；

  • 当链式写法超过三次或者因为事件绑定变得复杂后，用换行和缩进保持代码可读性；
