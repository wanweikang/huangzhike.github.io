  • 引擎默认不会回收全局变量，可能存活到进程退出，如果对象很大：在JavaScript中做模版渲染。
  • 尽量用局部变量。局部变量访问速度比全局变量快，因为全局变量是全局对象的属性，而局部变量在函数的栈里。
  • 缓存重复使用的全局变量。
  • 全局变量缺点：不利于内存释放；多人协作时容易混淆；在作用域链中易被干扰；需要搜索更长的作用域链。

  • 把数字转换成字符串：""+1效率最高，性能上：""+字符串 > String() > .toString() > new String()。

    • String()属于内部函数，速度很快。
    • .toString()要查询原型中的函数，速度略慢。
    • new String()最慢。

  • 浮点数转换成整型。

    • parseInt()将字符串转换成数字，而不是浮点数和整型之间的转换。
    • 应用Math.floor()或Math.round()。Math是内部对象，Math.floor()并没有多少查询方法和调用的时间，速度是最快的。

  • 不要在内置对象的原型上添加方法；

  • 不要在不需要的地方加括号；

delete a.b;
// not good
delete(a.b);
  • 使用数组和对象字面量，避免构造函数new Array()，new Object()，new function () {}。
  • 避免eval()或函数构造器，开销较大，每次调用，JavaScript引擎都要转换为可执行的代码。
  • 避免with()，with把变量加入到全局作用域中，如果有同名变量，容易混淆，值也会被覆盖。
  • 不断运行的代码，不应用setTimeout，而用setInterval。setTimeout每次要重新设置定时器。
  • 当JavaScript代码解析JavaScript代码时会双重解释，在eval("sum+=num1;")、new Function("sum+=num1;")和setTimeout("alert('hello world');", 1000)等传入字符串情况下遇到。
// 如果传给setTimeout()和setInterval()一个字符串，将会类似eval转换：
setInterval('func()', 1000);  

// 传给setInterval()和setTimeout()时使用函数而不是字符串：
setInterval(func, 1000);

能形成作用域（scope）的有：

  • 一、函数调用。
  • 二、with语句，with 会创建自已的作用域，因此会增加其中执行代码的作用域的长度。
  • 三、全局作用域。
  • 访问变量或数组是O(1)操作。访问对象上的属性是O(n)操作。
  • 对象的属性查找要比访问变量或数组花时间，因为必须在原型链中对该名属性搜索，属性越多，时间越长。
  • 需要多次用到对象属性，应将其存储在局部变量。
  • 减少作用域链上的查找次数，访问变量或函数，需要遍历当前执行环境的作用域链，一级一级直到全局执行环境。
// 效率低
for (var i = 0; i < 10000; i++) {
    // 每次都遍历作用域链，直到全局环境
    var but1 = document.getElementById("but1");
}
// 避免全局查找
var doc = document; // 先保存全局对象
for(var i = 0; i < 10000; i++){
    // 只遍历了一次
    var but1 = doc.getElementById("but1");
}
function foo() {

  • 谨慎对待循环事件绑定、私有属性、含参回调等使用闭包。
  • 循环绑定事件：有六个按钮，对应六种事件，点击按钮时，在指定地方输出相应事件。
var btns = document.querySelectorAll('.btn'); // 6 elements
var output = document.querySelector('#output');
var events = [1, 2, 3, 4, 5, 6];
// Case 1：错误
for (var i = 0; i < btns.length; i++) {
    btns[i].onclick = function(evt) {
         // 传入参数是当前循环下标
        output.innerText += 'Clicked ' + events[i];
    };
}
// Case 2
for (var i = 0; i < btns.length; i++) {
    btns[i].onclick = (function(index) {
        // 函数调用传入的参数是基本类型对象，在函数体内的形参是复制值，当作局部变量定义在函数作用域内
        return function(evt) {
            output.innerText += 'Clicked ' + events[index];
        };
    })(i);
}
// 当某个元素被删除时，相应的事件监听函数、事件对象、闭包函数也随之被销毁回收
// Case 3
for (var i = 0; i < btns.length; i++) {
    btns[i].onclick = (function(event) {
        return function(evt) {
            output.innerText += 'Clicked ' + event;
        };
    })(events[i]);
}
  • 不合理的闭包会导致内存泄漏。闭包性能不如内部方法，更不如重用外部方法。
  • 函数执行完时，函数内所声明对象不一定销毁。引用(Reference)是JavaScript引擎垃圾回收中最关键的机制。
function foo() {
    var val = 'hello world';
    return function() {
        return val;
    };
}
global.bar = foo();
// 代码执行完时， val 和 bar() 没被释放， 判断对象正在使用的标准，是对该对象的引用

JavaScript的内存泄露

给DOM对象添加的属性是一个对象的引用。

解决方法：window.onunload事件写上：document.getElementById('myDiv').myProp = null;

var MyObject = {};
document.getElementById('myDiv').myProp = MyObject;

DOM对象与JS对象相互引用。

解决方法：onunload事件写上：document.getElementById('myDiv').myProp = null;

function Encapsulator(element) {
    this.elementReference = element;
    element.myProp = this;
}
new Encapsulator(document.getElementById('myDiv'));

给DOM对象用attachEvent绑定事件。

解决方法：onunload事件中写上：element.detachEvent('onclick', doClick);

function doClick() {}
element.attachEvent("onclick", doClick);

从外到内执行appendChild。即使调用removeChild也无法释放。

var parentDiv = document.createElement("div");
var childDiv = document.createElement("div");
document.body.appendChild(parentDiv);
parentDiv.appendChild(childDiv);

解决方法：从内到外执行appendChild：

var parentDiv = document.createElement("div");
var childDiv = document.createElement("div");
parentDiv.appendChild(childDiv);
document.body.appendChild(parentDiv);

反复重写同一属性造成内存大量占用。

相当于定义了5000个属性，解决方法：无。

for (i = 0; i < 5000; i++) {
    hostElement.text = "asdfasdfasdf";
}

闭包保留了指向它封闭作用域的指针，给DOM元素附加闭包时，很可能产生循环引用, 导致内存泄漏。

function foo(element, a, b) {
    element.onclick = function() { /* uses a and b */ };
}

可将代码重构为:

function foo(element, a, b) {
    element.onclick = bar(a, b);
}

function bar(a, b) {
    return function() { /* uses a and b */ }
}

除了用闭包访问内部变量，还可以使用回调函数处理业务。

  • 回调函数是后续传递风格( Continuation Passing Style , CPS )的技术，相比闭包好处也有很多。
  • 如果传入的参数是基础类型，回调函数中传入的形参是复制值，业务代码使用完后，更容易被回收。
  • 回调除了可以完成同步外，还可以用在异步编程中。
  • 回调函数通常也是临时的匿名函数，一旦执行完毕，自身的引用就会被解除，得到回收。
function getData(callback) {
    var data = 'some big data';
    callback(null, data);
}
getData(function(err, data) {console.log(data);});

只在构造器或对象的方法(包括创建的闭包)中使用this。

面向对象书写方式：使用匿名函数进行封装，提高重用性

(function ($) {
    function api() {}

    api.prototype = {
        init: function (domain, d) {
            this.domain = "//" + domain;
            this.d = d;
        },
        getList: function (id) {
            ...
        },
        ...
    }

    // 注册到全局
    window.api = new api;
})(jQuery);

var api = window.api;
api.init("api.shop.com", 10);
var list = api.getList(1);
