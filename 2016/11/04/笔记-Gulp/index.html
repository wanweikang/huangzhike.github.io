  • gulp是基于Node.js的自动任务运行器， 能自动化完成javascript/coffee/sass/less/html/image/css等文件的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的步骤。借鉴了Unix的管道（pipe），前一级的输出，直接变成后一级的输入。gulp 和 grunt 类似，相比grunt频繁IO操作，gulp的流操作更快更便捷。

安装nodejs -> 全局安装gulp -> 项目安装gulp以及gulp插件 -> 配置gulpfile.js -> 运行任务

  • npm（node package manager）nodejs的包管理器，用于node插件管理（包括安装、卸载、管理依赖等）

  • 使用npm安装插件：命令提示符执行npm install <name> [-g] [--save-dev]；

  • 以gulp-less为例（编译less文件），npm install gulp-less --save-dev，安装在node_modules的gulp-less目录下，还得本地安装gulp：npm install gulp --save-dev；全局安装了gulp，项目也安装了gulp。

  • 本地安装gulp插件，定位目录命令后提示符执行npm install --save-dev

  • 新建gulpfile.js文件（gulpfile.js是gulp项目的配置文件），在gulpfile.js文件中定义任务，放到项目目录根目录中。

  • 以上是node_modules的插件模块。因为之前没安装，得用npm安装相应模块：npm install gulp-uglify gulp-minify-css gulp-concat gulp-rename --save。安装完成之后，打开package.json文件，看到相关插件的一些版本信息。

  • gulp使用的是Nodejs中的stream(流)，首先获取到需要的stream，然后通过stream的pipe()方法把流导入，比如gulp的插件中，经过插件处理后的流可以继续导入到其他插件中，也可以把流写入到文件中。gulp以stream为媒介，它不需要频繁的生成临时文件。

  • gulp的使用流程：首先通过gulp.src()方法获取到想要处理的文件流，然后把文件流通过pipe方法导入到gulp的插件中，最后把经过插件处理后的流再通过pipe方法导入到gulp.dest()中，gulp.dest()方法则把流中的内容写入到文件中。

  • 运行gulp任务，只需切换到存放gulpfile.js文件的目录(cmd、Git Bash、Power Shell等)，在命令行中执行gulp命令，gulp后面可以加上要执行的任务名，如gulp task，如果没有指定任务名，则执行default默认任务。

  • 使用通配符，检查文件名和路径进行匹配。

  • 当有多种匹配模式时可以使用数组gulp.src(['js/*.js','css/*.css','*.html'])

  • watch方法路径不要用'./xx'，用 ./xx 开头作为当前路径开始，会导致无法监测到新增文件，直接省略掉 ./ 即可。'./images/*' === 'images/*'
