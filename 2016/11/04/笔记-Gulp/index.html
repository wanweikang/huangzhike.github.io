gulp是基于Node.js的自动任务运行器， 能自动化完成javascript/coffee/sass/less/html/image/css等文件的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的步骤。借鉴了Unix的管道（pipe），前一级的输出，直接变成后一级的输入。gulp 和 grunt 类似，但相比grunt频繁IO操作，gulp 的流操作更快更便捷。

安装nodejs -> 全局安装gulp -> 项目安装gulp以及gulp插件 -> 配置gulpfile.js -> 运行任务

npm（node package manager）nodejs的包管理器，用于node插件管理（包括安装、卸载、管理依赖等）

使用npm安装插件：命令提示符执行npm install <name> [-g] [--save-dev]；

以gulp-less为例（编译less文件），命令提示符执行npm install gulp-less --save-dev，将会安装在node_modules的gulp-less目录下，还得本地安装gulp：npm install gulp --save-dev；全局安装了gulp，项目也安装了gulp，全局安装gulp是为了执行gulp任务，本地安装gulp则是为了调用gulp插件的功能。

第一个gulp项目：本地安装gulp插件，定位目录命令后提示符执行npm install --save-dev

新建gulpfile.js文件（gulpfile.js是gulp项目的配置文件），在gulpfile.js文件中定义任务，放到项目目录根目录中。

以上是node_modules里的插件模块。因为之前没有安装过，所以得用npm安装相应的模块：npm install gulp-uglify gulp-minify-css gulp-concat gulp-rename –save。安装完成之后，打开package.json文件，看到相关插件的一些版本信息。

运行gulp任务，只需切换到存放gulpfile.js文件的目录(cmd、Git Bash、Power Shell等)，然后在命令行中执行gulp命令，gulp后面可以加上要执行的任务名，例如gulp task，如果没有指定任务名，则执行default的默认任务。

在gulp中，使用的是Nodejs中的stream(流)，首先获取到需要的stream，然后通过stream的pipe()方法把流导入到你想要的地方，比如gulp的插件中，经过插件处理后的流可以继续导入到其他插件中，也可以把流写入到文件中。gulp是以stream为媒介的，它不需要频繁的生成临时文件。

gulp的使用流程：首先通过gulp.src()方法获取到想要处理的文件流，然后把文件流通过pipe方法导入到gulp的插件中，最后把经过插件处理后的流再通过pipe方法导入到gulp.dest()中，gulp.dest()方法则把流中的内容写入到文件中。

当有多个任务时，通过任务依赖来控制任务的执行顺序。例如执行one,two,three这三个任务，可以定义一个空的任务，然后把那三个任务当做这个空的任务的依赖就行了：

//只要执行default任务，就相当于把one,two,three这三个任务执行了
gulp.task(‘default’,[‘one’,’two’,’three’]);
