[笔记]-Promise

2016-11-08

2016-11-16更新

整理自：http://es6.ruanyifeng.com/

ES6原生提供了Promise，Promise是一个对象，传递异步操作的消息。

Promise对象将异步操作以同步操作的流程表达，避免层层嵌套的回调。

  • 状态不受外界影响。Promise对象代表异步操作，有三种状态：Pending（进行中）、Resolved（已完成，Fulfilled）和Rejected（已失败）。只有异步结果，可以决定当前状态。
  • 一旦状态改变，就不会再变。Promise对象的状态改变，只有两种可能：从Pending变为Resolved和从Pending变为Rejected。就算改变已发生了，再对Promise对象添加回调函数，也会立即得到这个结果。这与事件（Event）不同，事件如果错过，再去监听，得不到结果。

Promise对象缺点：

  • 首先，无法取消Promise，一旦新建就会执行，无法中途取消。
  • 其次，如果不设置回调，Promise内部抛出的错误，不会反映到外部。
  • 第三，处于Pending状态时，无法得知目前进展（刚开始？将完成？）。如果事件反复发生，使用stream比Promise更好。

Promise对象是一个构造函数，生成Promise实例。

Promise构造函数接受一个函数为参数，该函数的两个参数是内置函数，resolve和reject。

  • resolve，将Promise对象的状态从“未完成”变为“成功”（Pending>Resolved），异步成功时调用，并将异步操作结果，作为参数传递；
  • reject，将Promise对象的状态从“未完成”变为“失败”（Pending>Rejected），异步失败时调用，并将异步操作报错，作为参数传递。

如果调用resolve和reject函数时带有参数，参数被传递给回调函数。

  • reject的参数通常是Error对象的实例，表示抛出的错误；
  • resolve的参数除了正常值外，还可能是Promise实例，异步的结果可能是一个值，也可能是另一个异步：
var pro1 = new Promise(function(resolve, reject) {
});

var pro2 = new Promise(function(resolve, reject) {
    // ...
    if (/* 异步成功 */) {
        resolve(p1);
    } else {
        reject(error);
    }
});
  • pro2的resolve方法参数是pro1，异步的结果返回另一个异步。
  • pro1状态会传递给pro2，pro1状态决定了pro2状态。
  • 如果pro1状态是Pending，pro2的回调等待pro1状态改变；
  • 如果pro1状态是Resolved或Rejected，pro2的回调立刻执行。

Promise对象实现Ajax：

// getJSON封装XMLHttpRequest对象，发出针对JSON数据的HTTP请求，返回Promise对象
var getJSON = function(url) {
    var promise = new Promise(function(resolve, reject) {
        var client = new XMLHttpRequest();
        client.open("GET", url);
        client.onreadystatechange = handler;
        client.responseType = "json";
        client.setRequestHeader("Accept", "application/json");
        client.send();
        // resolve和reject函数调用时，都带参数
        function handler() {
            if (this.status === 200) {
                resolve(this.response);
            } else {
                reject(new Error(this.statusText));
            }
        };
    });

    return promise;
};

Promise.prototype.then()：Promise实例有then方法，定义在原型对象Promise.prototype上。

then方法分别指定Resolved和Reject状态的回调函数。

then方法接受两个回调函数为参数。

  • 第一个Promise对象变为Resolved时调用；
  • 第二个Promise对象变为Reject时调用，可选。
  • 两个回调都接受Promise对象传出的值作为参数。
promise.then(function(value) {
    // success
}, function(value) {
    // failure
});

实例：

function timeout(ms) {
    return new Promise((resolve) => {
        setTimeout(resolve, ms, 'done');
    });
}
// ms后，Promise实例状态变为Resolved，触发then方法绑定的回调
timeout(100).then((value) => {
    console.log(value);
});

then方法返回新的Promise实例（不是原来Promise实例）。

可以采用链式写法。

getJSON("/posts.json").then(function(json) {
    return json.post;
}).then(function(post) {
    // ...
});
// 第一个回调完成以后，将返回结果作为参数，传入第二个回调

前一个回调，可能返回Promise对象（异步），后一个回调待该Promise对象的状态变化，才会调用。

getJSON("/post/1.json").then(function(post) {
    // 第一个then回调返回另一个Promise对象
    return getJSON(post.commentURL);
}).then(function funcA(comments) {
    // 第二个then回调等待这个Promise对象状态变化
    // 如果Resolved，调用funcA
    console.log("Resolved: ", comments);
}, function funcB(err){
    // 如果Rejected，调用funcB
    console.log("Rejected: ", err);
});

// 箭头函数写法
getJSON("/post/1.json").then(
    post => getJSON(post.commentURL)
).then(
    comments => console.log("Resolved: ", comments),
    err => console.log("Rejected: ", err)
);

Promise的链式调用需要每个过程返回Promise对象才能顺序执行。

gitPromise
    .then(() => git.add())  // 正确，箭头函数简写
    .then(() => {
        git.commit(); // 错误，函数返回 undefined，会立即执行下一过程
    })
    .then(() => {
        return git.log(); // 正确
    });

多个异步任务同时执行用Promise.all，顺序执行使用链式调用。

// Promise.all
Promise
    .all([jsBuildPromise, cssBuildPromise])
    .then(() => {
        ...
    });

// chain
jsBuildPromise
    .then(() => cssBuildPromise)
    .then(() => {
        ...
    });

Promise.prototype.catch是.then(null, rejection)的别名，指定错误的回调。

p.then((val) => console.log("fulfilled:", val))
    .catch((err) => console.log("rejected:", err));
// 等同于
p.then((val) => console.log("fulfilled:", val))
    .then(null, (err) => console.log("rejected:", err));
getJSON("/posts.json").then(function(posts) {
    // ...
}).catch(function(error) {
    // 处理前一个回调的错误
    console.log('发生错误！', error);
});
// getJSON方法返回Promise对象，如果该对象状态变为Resolved，调用then方法指定的回调
// 如果异步错误，状态Rejected，调用catch方法指定回调

如果Promise状态已经变成resolved，再抛出错误是无效的。

var promise = new Promise(function(resolve, reject) {
    resolve("ok");
    // 在resolve语句后，再抛出错误，不会被捕获
    throw new Error('test');
});
promise
    .then(function (value) { console.log(value) })
    .catch(function (error) { console.log(error) });
// ok

跟try…catch…不同，如果没用catch方法指定处理错误的回调，Promise对象抛出的错误不会传递到外层。

var err = function() {
    return new Promise(function(resolve, reject) {
        resolve(x + 2); // 报错，x没有声明
    });
};

err().then(function() {
    console.log('everything is great');
});
// err产生的Promise对象报错，但没catch，错误不会被捕获，也不会传递到外层，运行后没任何输出
var promise = new Promise(function(resolve, reject) {
    resolve("ok");
    setTimeout(function() { throw new Error('test') }, 0)
});
promise.then(function(value) { console.log(value) }); // ok
// Uncaught Error: test
// Promise指定在下一轮“事件循环”再抛出错误，由于没指定catch，冒泡到最外层，成了未捕获的错误

Promise对象的错误具有“冒泡”性质，一直向后传递，直到被捕获，错误总会被下个catch捕获。

Promise用catch方法捕获错误，而且过程内的错误不会阻塞后续代码执行。

new Promise(() => {
    f;  // not define error !
})
    .catch((err) => {
        console.log(err)  // 'f is not define'
    });

console.log('error test'); // 正常执行

catch方法返回的还是一个Promise对象，后面可接着调用then方法。

err().then(function() {
    return otherErr();
}).catch(function(error) {
    console.log('oh no', error);
}).then(function() {
    console.log('carry on');
});
// oh no [ReferenceError: x is not defined]
// carry on

catch方法中，还能再抛出错误。

err().then(function() {
    return otherErr();
}).catch(function(error) {
    console.log('oh no', error);
    y + 2; // 会报错，y没有声明
}).then(function() {
    console.log('carry on');
});
// oh no [ReferenceError: x is not defined]
// catch抛出一个错误，后面没catch，这个错误不会被捕获，也不会传递到外层

改写一下，结果不一样。

err().then(function() {
    return otherErr();
}).catch(function(error) {
    console.log('oh no', error);
    y + 2; // 报错，y没有声明
}).catch(function(error) {
    console.log('carry on', error);
});
// oh no [ReferenceError: x is not defined]
// carry on [ReferenceError: y is not defined]

Generator函数管理流程，遇到异步，通常返回Promise对象。

function getFoo() {
    return new Promise(function (resolve, reject) {
        resolve('foo');
    });
}

var g = function* () {
    try {
        var foo = yield getFoo(); // 异步操作getFoo返回Promise对象
        console.log(foo);
    } catch (e) {
        console.log(e);
    }
};

// 函数run处理这个Promise对象，并调用next方法
function run (generator) {
    var it = generator();

    function go(result) {
        if (result.done) return result.value;

        return result.value.then(function (value) {
            return go(it.next(value));
        }, function (error) {
            return go(it.throw(value));
        });
    }

    go(it.next());
}

run(g);
// 注意：实例化的Promise对象会立即执行
let promise = new Promise((resolve, reject) => {
    if (success) {
        resolve(a) // pending ——> resolved 参数将传递给对应的回调方法
    } else {
        reject(err) // pending ——> rejectd
    }
})


promise.then(
    () => {
        console.log('this is success callback')
    },
    () => {
        console.log('this is fail callback')
    }
)

// 建议不要使用then()的rejected回调，而是统一使用catch()来处理错误
promise
    .then(() => {
        console.log('this is success callback')
    }).catch(
    (err) => {
        console.log(err)
    }
)

// then() 和 catch() 都会返回一个新的Promise对象，可以链式调用

使用async/await，搭配Promise，可以通过编写形似同步的代码来处理异步流程。

await操作符用于等待一个Promise对象，它只能在异步函数async function内部使用。

// [return_value] = await expression;
// 其中 expression 是一个 Promise 对象或者任何要等待的值;

而await expression的执行结果有以下几种情况:

  • 若expression是Promise对象，并且其以值x被fulfilled，则返回值为x。
  • 若expression是Promise对象，并且其以异常e被rejected，则抛出异常e。
  • 若expression不是Promise对象，则会将expression处理成一个以expression值被fulfilled的Promise对象，然后返回这个Promise对象的最终值(即expression值)。

这种用法没太大意义，因此实际使用时还是尽量在await后跟一个Promise对象。

await在等待Promise对象时会导致async function暂停执行，直到Promise对象决议之后才会async function继续执行。

使用async function可以定义一个异步函数。

async function name([param[, param[, ... param]]]) {
    statements
}

async函数的返回值很特殊，不管在函数体内return了什么值，async函数的实际返回值总是一个Promise对象。

若在async函数中return了一个值x，不管x值是什么类型，async函数的实际返回值总是Promise.resolve(x)。

var fn1 = function () {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            resolve();
            console.log('fn1 is executed');
        }, 1000);
    });
};

var fn2 = function () {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            console.log('fn2 is executed');
            resolve();
        }, 1000);
    });
};

var fn3 = function () {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            console.log('fn3 is executed');
            resolve();
        }, 1000);
    });
};

(async function seq(fn1, fn2, fn3) {
    await fn1();
    await fn1();
    await fn1();
})(fn1, fn2, fn3);
(async function seq(...arr) {
    for (let value of arr) {
        await value();
    }
    return 'done';
})(fn1, fn2, fn3).then(function (value) {
    // 当所有await全部完成以后，return一个值 ，这里写的是’done’, 这个值会被then捕获到并且进行后续操作。
    // 因为return的仍然是一个promise对象。
    console.log(value);
});
const sleep = (timeout = 2000) => new Promise(resolve => setTimeout(resolve, timeout))

async function Prms(id) {
    console.log(`0`)
    await sleep(2000)
    console.log(`2000`)
    return id
}
// 两个请求串行处理多个await操作，先调用异步操作，再处理异步结果
(async (id) => {
    console.time('time')

    const a = await Prms('a')
    console.log(`a:${a}`)

    const b = await Prms('b')
    console.log(`a:${a}`)

    console.log(`b:${b}`)

    console.timeEnd('time') // 4001.122ms
})()
// 两个请求为并行
(async (id) => {
    console.time('time')
    const a = Prms('a')
    const b = Prms('b')

    const ra = await a
    console.log(`ra:${ra}`)
    const rb = await b
    console.log(`ra:${ra}`)

    console.log(`rb:${rb}`)

    console.timeEnd('time') // 2001.615ms
})()
