  • ES6原生提供了Promise对象。
  • Promise是一个对象，传递异步操作的消息。
  • 它代表了某个未来才会知道结果的事件（通常是一个异步操作），并且这个事件提供统一的API。
  • 有了Promise对象，可以将异步操作以同步操作的流程表达出来，避免了层层嵌套的回调函数。

  • Promise对象有以下两个特点。
    （1）对象的状态不受外界影响。Promise对象代表一个异步操作，有三种状态：Pending（进行中）、Resolved（已完成，Fulfilled）和Rejected（已失败）。只有异步结果，可以决定当前状态，其他操作都无法改变。
    （2）一旦状态改变，就不会再变。Promise对象的状态改变，只有两种可能：从Pending变为Resolved和从Pending变为Rejected。就算改变已经发生了，再对Promise对象添加回调函数，也会立即得到这个结果。这与事件（Event）不同，事件特点是，如果错过，再去监听，得不到结果。

  • Promise缺点：
    首先，无法取消Promise，一旦新建就会立即执行，无法中途取消。
    其次，如果不设置回调函数，Promise内部抛出的错误，不会反应到外部。
    第三，处于Pending状态时，无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）。如果事件反复发生，使用stream比Promise更好。

  • Promise对象是一个构造函数，生成Promise实例。

  • Promise构造函数接受一个函数为参数，该函数的两个参数是resolve和reject。它们是两个函数，由JavaScript提供。

  • resolve，将Promise对象的状态从“未完成”变为“成功”（从Pending变为Resolved），在异步操作成功时调用，并将异步操作的结果，作为参数传递；
  • reject，将Promise对象的状态从“未完成”变为“失败”（从Pending变为Rejected），在异步操作失败时调用，并将异步操作报错，作为参数传递。

  • 如果调用resolve函数和reject函数时带有参数，参数会被传递给回调函数。
    reject的参数通常是Error对象的实例，表示抛出的错误；
    resolve的参数除了正常值外，还可能是Promise实例，表示异步的结果可能是一个值，也可能是另一个异步，比如下面。

  • Promise.prototype.then()：Promise实例有then方法，定义在原型对象Promise.prototype上。then方法分别指定Resolved状态和Reject状态的回调函数。

  • then方法返回新的Promise实例（不是原来Promise实例）。可以采用链式写法。

  • Generator函数管理流程，遇到异步，通常返回Promise对象。

// 2. Promise 的链式调用需要每一个过程返回一个 Promise 对象才能保证顺序执行
gitPromise
.then(() => git.add()) // 正确，箭头函数简写
.then(() => {
git.commit(); // 错误，函数返回 undefined，会立即执行下一过程
})
.then(() => {
return git.log(); // 正确
});

// 3. Promise 需要调用 catch 方法来捕获错误，而且过程内的错误不会阻塞后续代码执行
new Promise(() => {
f; // not define error !
})
.catch((err) => {
console.log(err) // ‘f is not define’
});

console.log(‘error test’); // 正常执行
