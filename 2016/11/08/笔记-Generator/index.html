[笔记]-Generator

2016-11-08

2016-11-16更新

整理自：http://es6.ruanyifeng.com/

  • Generator函数是ES6提供的一种异步编程方案；
  • function命令与函数名之间有一个星号；
  • 函数体内部使用yield语句，定义遍历器的每个成员，即不同的内部状态。
  • 调用与普通函数一样，调用后，函数并不执行，返回一个指向内部状态的指针对象，也就是遍历器对象（Iterator Object）。
  • 调用遍历器对象的next方法，使得指针移向下一个状态。
  • 每次调用next方法，内部指针就从函数头部或上一次停下来的地方开始执行，直到下一个yield语句（或return语句）为止。
  • Generator函数是分段执行的，yield命令暂停执行，next方法恢复执行。
function* helloWorldGenerator() {
    // 该函数有三个状态：hello，world和return语句（结束执行）
    yield 'hello';
    yield 'world';
    return 'ending';
}

const hw = helloWorldGenerator();

hw.next() // { value: 'hello', done: false }
hw.next() // { value: 'world', done: false }
hw.next() // { value: 'ending', done: true }
hw.next() // { value: undefined, done: true }
// 共调用了四次next方法

第一次调用，开始执行，直到第一个yield语句。next方法返回一个对象，value属性就是当前yield语句的值hello，done属性值false，表示遍历没完成。

第二次调用，从上次yield语句停下的地方，一直执行到下一个yield语句。

第三次调用，从上次yield语句停下的地方，一直执行到return语句（如果没有return语句，就执行到函数结束）。next方法返回的对象的value属性，就是return后面的表达式的值（如果没有return语句，则value属性的值为undefined），done属性的值true，表示遍历已经结束。

第四次调用，已经运行完毕，next方法返回对象的value属性为undefined，done属性为true。以后再调用next方法，返回的都是这个值。

yield语句后面的表达式，只有调用next方法、指向该语句时才会执行，等于手动的“惰性求值”（Lazy Evaluation）。

function* gen{
    yield  123 + 456;
}
// yield后面的表达式123 + 456，不会立即求值，只会在next方法将指针移到这一句时，才会求值。
  • yield语句与return语句都能返回语句后表达式的值。
  • 区别在于遇到yield，函数暂停执行，下一次再从该位置继续向后执行。
  • 一个函数里面，只能执行一个return语句，但可以执行多个yield语句。
  • 正常函数只能返回一个值，因为只能执行一次return；
  • Generator函数可以返回一系列的值，因为可以有任意多个yield。

Generator函数不用yield语句，就变成了单纯的暂缓执行函数。

function* f() {
    console.log('执行了！')
}
// 只有调用next方法时，函数f才会执行
const generator = f();

setTimeout(function () {
    generator.next()
}, 2000);

yield语句不能用在普通函数中，否则报错。

const arr = [1, [[2, 3], 4], [5, 6]];
// forEach方法的参数是一个普通函数，但是在里面使用了yield语句。
const flat = function* (a){
    a.forEach(function(item){
        if (typeof item !== 'number'){
            yield* flat(item);
        } else {
            yield item;
        }
    }
};

for (var f of flat(arr)){
    console.log(f);
}

// 一种方法是用for循环
const flat = function* (a){
    const length = a.length;
    for(var i =0;i<length;i++){
        var item = a[i];
        if (typeof item !== 'number'){
            yield* flat(item);
        } else {
            yield item;
        }
    }
};

for (var f of flat(arr)){
    console.log(f);
}
// 1, 2, 3, 4, 5, 6

任意一个对象的Symbol.iterator属性，等于该对象的遍历器函数，调用该函数会返回该对象的一个遍历器。

// 遍历器本身也是一个对象，它的Symbol.iterator属性执行后，返回自身
function* gen(){
}

const g = gen(); // gen是Generator函数，调用它生成遍历器g
// 遍历器g的Symbol.iterator属性是一个遍历器函数，执行后返回它自己
g[Symbol.iterator]() === g // true

yield语句本身没有返回值，或者说返回undefined。

next方法可以带一个参数，该参数就会被当作上一个yield语句的返回值。

// 定义了一个可以无限运行的Generator函数f
function* f() {
    for(var i=0; true; i++) {
        // 如果next方法没有参数，每次运行到yield语句，变量reset的值总是undefined
        var reset = yield i;
        // 当next方法带一个参数true时，当前的变量reset就被重置为这个参数（即true）
        if(reset) { i = -1; } // i会等于-1，下一轮循环就会从-1开始递增
    }
}

var g = f();

g.next() // { value: 0, done: false }
g.next() // { value: 1, done: false }
g.next(true) // { value: 0, done: false }

Generator函数从暂停状态到恢复运行，上下文状态（context）是不变的。

function* foo(x) {
    var y = 2 * (yield (x + 1));
    var z = yield (y / 3);
    return (x + y + z);
}

var a = foo(5);

a.next() // Object{value: 6, done: false}
a.next() // Object{value: NaN, done: false}
// 第二次运行next方法不带参数，y的值等于2 * undefined（即NaN），除以3是NaN，返回对象的value属性也是NaN。
a.next() // Object{value: NaN, done: false}
// 第三次运行Next方法不带参数，z等于undefined，返回对象的value属性等于5 + NaN + undefined，即NaN。

如果向next方法提供参数，返回结果就不一样了。

function* foo(x) {
    var y = 2 * (yield (x + 1));
    var z = yield (y / 3);
    return (x + y + z);
}

var it = foo(5);

it.next() // { value: 6, done: false }
// 第一次调用next方法时，返回x+1的值6；
it.next(12) // { value: 8, done: false }
// 第二次调用next方法，将上一次yield语句的值设为12，y等于24，返回y / 3的值8；
it.next(13) // { value: 42, done: true }
// 第三次调用next方法，将上一次yield语句的值设为13，z等于13，x等于5，y等于24，return语句的值等于42。

注意，由于next方法的参数表示上一个yield语句的返回值，所以第一次使用next方法时，不能带有参数。

for…of循环自动遍历Generator函数，不需要next方法。

function* foo() {
    yield 1;
    yield 2;
    yield 3;
    yield 4;
    yield 5;
    return 6;
}
// 使用for...of循环，依次显示5个yield语句的值。
for (let v of foo()) {
    console.log(v);
}
// 1 2 3 4 5
// 一旦done为true，for...of循环中止，且不包含该返回对象，return语句返回的6，不包括在for...of循环中

利用generator函数和for…of循环，实现斐波那契数列。

function* fibonacci() {
    let [prev, curr] = [0, 1];
    for (;;) {
        [prev, curr] = [curr, prev + curr];
        yield curr;
    }
}

for (let n of fibonacci()) {
    if (n > 1000) break;
    console.log(n);
}
// 使用for...of语句时不需要使用next方法

ES6以前，异步方法大概有四种：回调函数，事件监听，发布/订阅，Promise对象。

异步：所谓”异步”，简单说就是一个任务分成两段，先执行第一段，然后转而执行其他任务，等做好了准备，再回过头执行第二段。

比如，有一个任务是读取文件进行处理，任务的第一段是向操作系统发出请求，要求读取文件。

然后，程序执行其他任务，等到操作系统返回文件，再接着执行任务的第二段（处理文件）。

这种不连续的执行，就叫做异步。

相应地，连续的执行就叫做同步。由于是连续执行，不能插入其他任务，所以操作系统从硬盘读取文件的这段时间，程序只能干等着。

为什么Node.js约定，回调函数的第一个参数，必须是错误对象err（如果没有错误，该参数就是null）？

原因是执行分成两段，在这两段之间抛出的错误，无法捕捉，只能当作参数，传入第二段。

JavaScript语言对异步编程的实现，就是回调函数。

回调函数本身并没有问题，问题在多个回调函数嵌套。

// 读取A文件之后，再读取B文件
fs.readFile(fileA, function (err, data) {
    fs.readFile(fileB, function (err, data) {
        // ...
    });
});
// 如果依次读取多个文件，就会出现多重嵌套。

Promise解决了这个问题。允许将回调函数的横向加载，改成纵向加载。

// 采用Promise，连续读取多个文件
var readFile = require('fs-readfile-promise'); // fs-readfile-promise模块返回Promise版本的readFile函数
// Promise提供then方法加载回调函数，catch方法捕捉执行过程中抛出的错误
readFile(fileA)
    .then(function(data){
        console.log(data.toString());
    })
    .then(function(){
        return readFile(fileB);
    })
    .then(function(data){
        console.log(data.toString());
    })
    .catch(function(err) {
        console.log(err);
    });
// Promise最大问题是代码冗余

“协程”（coroutine），多个线程互相协作，完成异步任务。
第一步，协程A开始执行。
第二步，协程A执行到一半，进入暂停，执行权转移到协程B。
第三步，（一段时间后）协程B交还执行权。
第四步，协程A恢复执行。上面流程的协程A，就是异步任务，因为它分成两段（或多段）执行。

// 读取文件的协程写法
function asnycJob() {
    // ...其他代码
    var f = yield readFile(fileA);
    // ...其他代码
}
// 函数asyncJob是协程，奥妙在yield命令。表示执行到此处，执行权将交给其他协程
// yield命令是异步两个阶段的分界线
// 协程遇到 yield 命令就暂停，等到执行权返回，再从暂停的地方继续往后执行
// 优点是写法像同步操作

Generator函数是协程在ES6的实现，可以交出函数的执行权（即暂停执行）。

Generator函数就是一个封装的异步任务，或者说是异步任务的容器。

异步操作需要暂停的地方，都用yield语句注明。

function* gen(x){
    var y = yield x + 2;
    return y;
}
// Generator函数执行不会返回结果，返回的是指针对象。
var g = gen(1); // 调用Generator函数，返回一个内部指针（即遍历器）g
g.next() // { value: 3, done: false }
// 调用指针g的next方法，移动内部指针（即执行异步任务的第一段），指向第一个遇到的yield语句
g.next() // { value: undefined, done: true }
// 执行到x + 2为止

Generator函数内部部署错误处理代码，捕获函数体外抛出的错误。

function* gen(x){
    try {
        var y = yield x + 2;
    } catch (e){
        console.log(e);
    }
    return y;
}

var g = gen(1);
g.next();
g.throw（'出错了'）// 出错了
// 最后一行，Generator函数体外，使用指针对象的throw方法抛出的错误，被函数体内的try ...catch代码块捕获
// 出错代码与处理错误代码，实现分离

虽然Generator将异步表示得很简洁，但流程管理却不方便（何时执行第一阶段、何时执行第二阶段）。

var fetch = require('node-fetch');

function* gen(){
    var url = 'https://api.github.com/users/github';
    var result = yield fetch(url);
    console.log(result.bio);
}
// enerator函数封装了一个异步操作，先读取一个远程接口，然后从JSON格式的数据解析信息
// 非常像同步操作，除了yield命令

// 执行方法如下
var g = gen(); // 首先执行Generator函数，获取遍历器对象
var result = g.next(); // 然后使用next 方法，执行异步任务第一阶段

// 由于Fetch模块返回的是Promise对象，要用then方法调用下一个next方法
result.value.then(function(data){
    return data.json();
}).then(function(data){
    g.next(data);
});
