  • PPI（Pixels Per Inch）：屏幕每英寸的像素数量，即像素密度（Screen density）
  • 物理像素（physical pixel）
  • 设备独立像素（density-independent pixel）：密度无关像素，代表一个可由程序使用的虚拟像素（如css像素）
  • 设备像素比（device pixel ratio）：dpr = 物理像素 / 设备独立像素
  • javascript中，通过window.devicePixelRatio获取当前设备的dpr

  • css中，通过-webkit-device-pixel-ratio，-webkit-min-device-pixel-ratio和 -webkit-max-device-pixel-ratio进行媒体查询

  • 以iphone6为例：设备宽高为375×667，可理解为css像素；dpr为2，物理像素应该×2，为750×1334

  • 普通屏幕下，1个css像素 对应 1个物理像素（1:1）
  • retina屏幕下，1个css像素对应 4个物理像素（1:4）
  • window.innerWidth度量的是浏览器窗口的宽度。是css pixels。将页面放大到200%，window.innerWidth变为了刚才的一半
  • PC端：正常情况下，一css像素等于一设备像素。放大200%时，一个css像素等于四个设备像素。（宽2倍 高2倍）

  • 缩放的作用就是改变一个css像素可以容纳设备像素的多少。

  • retina图片高清问题

    • 解决方案：两倍图片（@2x），200×300（css pixel）img标签，提供400×600的图片。

    • 如：图片大小，400×600;

  • 不同的dpr下，加载不同的尺寸的图片。通过css媒体查询或javascript条件判断都可以。

  • 缺点就是浪费流量，因为在JavaScript执行之前，就已经在获取低解析度的图片了

  • 通过 viewport + rem 实现

  • 设计师们在真实的iphone5上做的标注，而iphone5分辨率是640，在开发只需要按照320的标准来。为了节省时间，不至于每次都需要将标注取半，可以将整个网页缩放比例，模拟提高分辨率。为不同的设备设置不同的meta即可:

  • 设计师想要的retina屏幕border: 1px;是1物理像素，不是css像素，但是实际border: 1px;是指css像素，即2物理像素，所以显粗，而1物理像素应该对应0.5css像素，但不是所有手机都支持border: 0.5px;。

  • 元素scale：利用 :before 或 :after 做 border ，并 transform scale 缩小一半，原先的元素相对定位，新的 border 绝对定位

    • 单条 border

    • 四条 border：可能会遮住里面的元素，导致里面的触屏事件都不在好使了，文字也选择不了

    • 使用时结合 JS 代码，判断是否 Retina 屏

  • 然而，页面scale带来一些问题：

    • 字体大小会被缩放
    • 页面布局会被缩放(如 div的宽高等)

  • 动态改变根节点html的font-size，可以通过css做，也可以通过javascript做。

  • 方法一：纯css，支持calc函数的动态计算，不支持的用css媒体查询断点，优雅降级。

  • 方法二：脚本动态计算（和淘宝flexible.js有些不一样），前提：
    1、initial-scale 为 1；
    2、在项目css中(注意不要被公共的base、common之类的影响了，资源加载顺序也是蛮重要的)，把html的fontSize设为 50px（或者加上媒体查询）, 避免加载未完成时候样式错乱；

  • 一般来说，640的稿子量出来的字体是data-dpr=”2”的，750（ip6）也是data-dpr=”2”，data-dpr=”1”和data-dpr=”3”的按倍数计算，同时可以微调。

  • 640px宽的屏幕，可能是6401dpr，也可能是3202dpr，不管哪一种，页面的布局（使用rem单位）长得都是一样的，字体怎么度量？以前是@medie，但现在通过[data-dpr=”x”]实现，问题是，怎么知道640的页面是1dpr还是2dpr？只能通过经验，一般640，750都是2dpr的，因此640的稿子量出来的字体大小是属于data-dpr=”2”的。

  • 如果需要适应pad，有两个方案，一个是定宽；另一个是在html元素上再加一个data-device，因为640*1会在pad中出现。

  • 图片需要3倍高清：

    • css实现时，和font-size处理一样，通过[data-dpr=”x”]设置不同的图片
    • js实现可以通过html的data-dpr属性值来设置
    • 首屏图片最好用css，使用background，如果需要img，则只能内嵌js代码
    • 非首屏图片的话img和background都可以用，在lazyload里实现

  • 小数点问题：

    • 导致 width - backgroundSize = 1，会出现截边问题，需要设置background-size: cover。因为只相差1px，对background-position影响不大，因此依然使用rem

  • 使用icofont建议：

    • 把字符资源当成一般图片资源即可
    • 需要共享使用外链，不需要使用内嵌
    • 外链时，字符文件用gzip压缩
    • 只需要trueType（.ttf）格式
    • 一个字符资源不要包含太多图标，测试时，3个5k，10个7k

  • 不需要使用rem的属性（为以后的自动计算工具做准备）

    • font-size
    • border系，除了border-radius
  • 综合：
    1、meta:viewport, 还是initial-scale为 1；
    2、320px屏幕下，把页面根元素html的字体大小设置为50px；
    3、鉴于我们拿到的设计图目前是640px宽的基准，这样我们就不用每次自己除以2了，直接在PS中量就好；
    4、宽度什么的最好还是用百分比处理；涉及到高度、字体大小之类的则用rem。
