三种迭代器

  • ArrayList#iterator

  • ArrayList#listIterator

  • ArrayList#spliterator

关于Spliterator，这是JDK8加的，就是将元素分割为多份，多线程并行处理，对于ArrayList这种随机存取的还好，LinkedList这种的话，分割操作本身的性能应该就不太好吧。

ArrayList$Itr#remove

所以遍历时如果要移除List的元素，不能直接调用List#remove方法，而是要通过Iterator移除，否则会抛异常。

类似for...each这种方式遍历其实是个语法糖，反编译类文件后就会发现其实还是调用了迭代器。

不过我还是不喜欢用迭代器哈，麻烦。

可以看出迭代器还是多了挺多步骤的，所以只是简单的迭代的话，用普通的for (; ; ) { }会更加快些。

ListIterator其实加了个ListIterator#hasPrevious和ListIterator#previous，即向前迭代的功能，这里不贴代码了。
