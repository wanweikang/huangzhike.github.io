    public void load() {
        if (loaded) {
            return;
        }
        loaded = true;
        // ...
        // 初始化temp目录，java.io.tmpdir
        initDirs();
        // 初始化命名服务属性，JNI
        initNaming();
        // Set configuration source
        ConfigFileLoader.setSource(new CatalinaBaseConfigurationSource(Bootstrap.getCatalinaBaseFile(), getConfigFile()));
        // ${catalina.home}/conf/server.xml
        File file = configFile();
        // 实例化Digester，用来解析XML节点，创建Java对象或执行对象的某个方法
        Digester digester = createStartDigester();
        // conf/server.xml
        try (ConfigurationSource.Resource resource = ConfigFileLoader.getSource().getServerXml()) {
            InputStream inputStream = resource.getInputStream();
            InputSource inputSource = new InputSource(resource.getURI().toURL().toString());
            inputSource.setByteStream(inputStream);
            // 放入Catalina实例，顶级
            digester.push(this);
            // 解析并创建Server及下级的一系列对象
            digester.parse(inputSource);
        } catch (Exception e) {
            // ...
        }
        // Catalina和上面创建的Server互为对方的属性
        getServer().setCatalina(this);
        // 设置StandardServer的CatalinaHome
        getServer().setCatalinaHome(Bootstrap.getCatalinaHomeFile());
        // 设置StandardServer的CatalinaBase
        getServer().setCatalinaBase(Bootstrap.getCatalinaBaseFile());
        // 重定向日志流
        initStreams();
        // Start the new server
        try {
            // 初始化StandardServer，LifecycleBase#init
            getServer().init();
        } catch (LifecycleException e) {
            // ...
        }
        // ...
    }

    @Override
    protected void initInternal() throws LifecycleException {
        // LifecycleMBeanBase#initInternal
        super.initInternal();
        if (engine != null) {
            // 初始化StandardEngine
            engine.init();
        }
        // Initialize any Executors
        for (Executor executor : findExecutors()) {
            if (executor instanceof JmxEnabled) {
                ((JmxEnabled) executor).setDomain(getDomain());
            }
            // 初始化线程池
            executor.init();
        }
        // Initialize mapper listener
        mapperListener.init();
        // Initialize our defined Connectors
        synchronized (connectorsLock) {
            for (Connector connector : connectors) {
                // 初始化Connector
                connector.init();
            }
        }
    }

总结：

server.xml其实已经描述了Tomcat的整个结构。

  • Server：代表一个Tomcat服务器。

  • Service：代表一个服务，一个服务器可以有多个服务，Server可以包含多个Service。Service包含了多个Connector和一个Engine（顶级的Container）。

  • Connector：代表一个连接器，主要是监听一个端口，处理客户端连接，一个服务可以有多个连接器，如HTTP，HTTPS，或者不同端口。

  • Container：代表一个容器，按层级有Engine，Host，Context，Wrapper四种，通过组合形成上下级关系。一个服务只有一个容器。

    • Engine：引擎，下面可以有多个Host，一个Service下面只有一个Engine。

    • Host：主机，代表一个域名。下面可以有多个Context。

    • Context：Web应用，下面可以有多个Wrapper，一个web.xml一个Context。

    • Wrapper：一个Wrapper包装一个Servlet。

以默认配置为例，webapps就是一个Host，名称为localhost，webapps下的每个目录都是一个Context。

每个Container（容器）都有一个Pipeline（管道），每个Pipeline至少都有一个Valve（阀门），以职责链的形式来处理请求。

每个Pipeline都有一个内置的Valve，通过Pipeline#setBasic设置，最后一个执行，调用下一层的Pipeline。

其它的Valve是通过Pipeline#addValve添加的。

Connector收到请求并生成Request和Response后，通过Pipeline传入，并调用处理，到达Servlet，最后回复响应。
