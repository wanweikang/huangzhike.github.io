[笔记]-J.U.C-ReentrantLock

2019-04-20 20:51:17

非公平LOCK：

  • 首先CAS设置同步state由0为1，成功就设置锁的持有线程为自己，一般别的线程释放锁后锁空闲并且没有竞争就会成功。

  • 失败就执行AQS的acquire方法，调用子类重写的tryAcquire方法再次尝试获取。

  • 如果再次失败，addWaiter方法CAS循环直到成功进入AQS同步队列队尾，然后调用acquireQueued，自旋。

    • 如果前驱节点是头节点，返回中断状态；

    • 如果当前线程是非中断状态，parkAndCheckInterrupt会阻塞，直到被前驱节点unpark，或者中断，返回线程的中断状态并复位，继续自旋。

      • 如果是被前驱unpark，走第一步；

      • 如果是被中断，再次parkAndCheckInterrupt不起作用，直接返回中断状态true并复位，继续自旋，由于之前已经复位，parkAndCheckInterrupt会再次阻塞。

  • acquireQueued直到获取锁了才返回，如果在等待过程中被中断过，则自己产生一个中断（之前的被清除了）。

公平LOCK：

  • 少了直接的CAS抢锁操作，而是调用AQS的acquire方法，里面调用重写的tryAcquire方法，tryAcquire方法调用hasQueuedPredecessors，只有没有前继节点时才CAS设置同步state，成功就设置锁的持有线程为自己，失败就addWaiter入队列，调用acquireQueued，同非公平锁。

UNLOCK：

  • 调用AQS的release释放锁，release调用了子类重写的tryRelease方法，设置同步state，成功则调用unparkSuccessor唤醒后继节点。
