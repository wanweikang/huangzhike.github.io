  • BootstrapClassLoader：加载核心，包括：sun.boot.class.path（%JAVA_HOME%\jre\lib\，%JAVA_HOME%\jre\classes），-Xbootclasspath；

  • ExtentionClassLoader：加载扩展，包括：java.ext.dirs（%JAVA_HOME%\jre\lib\ext\）；

  • AppClassLoader：加载当前应用的classpath，包括：java.class.path；

继承关系：

AppClassLoader -> URLClassLoader -> SecureClassLoader -> ClassLoader

ExtClassLoader -> URLClassLoader -> SecureClassLoader -> ClassLoader

父加载器（ParentClassLoader）不是父类（Superclass）。

AppClassLoader的父加载器是ExtClassLoader，ExtClassLoader的父加载器是BootstrapClassLoader。

双亲委派（Parent Delegation Model）：

类加载器加载类时，先检查该类是否已被该类加载器加载，有就直接返回，没有就递归向上委托父类加载器加载类，如果都没被加载，则按照递归链依次向下从各自路径加载类返回。

  • 防止重复加载类；

  • 保证系统的内置类不会被覆盖。

避开双亲委托：

参考了这篇文章。

双亲委托加载类是单向的，顶层的类加载器，无法访问底层的类加载器所加载的类。

系统核心类是由BootstrapClassLoader加载的，有些情况是需要由BootstrapClassLoader加载应用类路径下的类，而不是由AppClassLoader来加载。

比如java.sql.Driver，是一个标准，该接口的实现是由数据库厂商完成，根据SPI加载。

按照双亲委托，BootstrapClassLoader加载该实现时，由于自己就是顶级的类加载器，但是实现不在自己负责的目录下，所以会抛ClassNotFoundException异常。

解决办法：线程上下文类加载器（Thread Context ClassLoader）。

  • Thread#getContextClassLoader

  • Thread#setContextClassLoader

    public static <S> ServiceLoader<S> load(Class<S> service) {

CustomClassLoader

/**
 * 使用自定义类加载器时，假设加载的是Xxx.class，执行到native方法ClassLoader#defineClass1时
 * 由于Xxx继承了Object.class，所以会递归调用该自定义类加载器的loadClass方法，加载java.lang.Object.class
 */
public class CustomClassLoader extends ClassLoader {
    /**
     * 自定义类加载器时，如果不想遵守双亲委派模型，重写loadClass方法，否则只需重写findClass方法
     * findClass是从自定义路径加载类
     * ClassLoader#findClass默认抛出ClassNotFoundException
     */
    @Override
    public Class findClass(String name) {
        byte[] data = classToByte(name);
        //
        return defineClass(name, data, 0, data.length);
    }

    /**
     * .class文件转字节流数组
     */
    private byte[] classToByte(String name) {
        FileInputStream fis;
        byte[] data = null;
        try {
            File file = new File(name + ".class");
            fis = new FileInputStream(file);
            ByteArrayOutputStream baos = new ByteArrayOutputStream();
            int ch;
            while ((ch = fis.read()) != -1) {
                baos.write(ch);
            }
            data = baos.toByteArray();
        } catch (IOException e) {
            e.printStackTrace();
        }
        return data;
    }
}

    protected Class<?> loadClass(String name, boolean resolve) throws ClassNotFoundException {
        synchronized (getClassLoadingLock(name)) {
            // 首先检测类是否已经被加载，里边是个native实现
            Class<?> c = findLoadedClass(name);
            if (c == null) {
                long t0 = System.nanoTime();
                try {
                    if (parent != null) {
                        // 有父加载器，委托给父加载器加载
                        c = parent.loadClass(name, false);
                    } else {
                        // 委托给BootstrapClassLoader加载，里边是个native实现
                        c = findBootstrapClassOrNull(name);
                    }
                } catch (ClassNotFoundException e) {
                    // ClassNotFoundException thrown if class not found
                    // from the non-null parent class loader
                }
                if (c == null) {
                    long t1 = System.nanoTime();
                    // 还没找到，查找自己定义的目录（继承了URLClassLoader）
                    c = findClass(name);
                    // this is the defining class loader; record the stats
                    sun.misc.PerfCounter.getParentDelegationTime().addTime(t1 - t0);
                    sun.misc.PerfCounter.getFindClassTime().addElapsedTimeFrom(t1);
                    sun.misc.PerfCounter.getFindClasses().increment();
                }
            }
            if (resolve) {
                // 解析class，里边是个native实现
                resolveClass(c);
            }
            return c;
        }
    }
