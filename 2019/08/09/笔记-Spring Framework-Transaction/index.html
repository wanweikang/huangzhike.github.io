事务的一些注意事项：

参考了这篇文章：https://blog.csdn.net/bntX2jSQfEHy7/article/details/79040349 ，文章不错，就是标题太。。。我还以为在看微信传销号呢。。。

@Transactional注解可用于接口、接口方法、类以及类方法上，但建议不要在接口方法上声明@Transactional注解，这样只有在基于接口的JDK代理时才会生效，最好在具体类的方法上使用@Transactional注解。原因是被子类重写的方法不能继承父类的注解，所以如果是基于类的CGLIB代理时，不会被拦截（貌似现在可以了？AbstractFallbackTransactionAttributeSource#AbstractFallbackTransactionAttributeSource）。

@Transactional注解在类级的声明中，会使得所有方法都有事务，不需要使用事务的方法，不要放置事务，比如查询方法，否则对性能有影响。

假设某类有方法A和方法B，方法A没有@Transactional注解，方法B有@Transactional注解，方法A内部调用了方法B，此时事务不会生效。

因为AOP生成的代理对象，不会拦截方法A，所以直接调用了原本实例的方法A，方法A直接调用了自身的方法B，完全没走AOP的拦截，所以事务不会生效。

假设某类有方法A和方法B，都有@Transactional注解，方法A内部调用了方法B，此时方法B的事务是跟随着方法A的。

原因还是一样，方法A内部调用的方法B是直接调用实例对象的方法，没有被拦截。

解决方法： @EnableAspectJAutoProxy(exposeProxy = true)

@Transactional注解只被应用到public方法上，否则将被忽略，也不会抛异常。

同样的道理，只有外部调用时，才走AOP的拦截，而内部的相互调用是不会被拦截的，为了避免这一点，所以不支持非public方法。
